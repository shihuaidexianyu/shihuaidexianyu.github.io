顺序处理器实现 (SEQ)

SEQ 的基本思想

SEQ 是一种严格顺序执行的 Y86-64 处理器实现。一条指令必须在一个时钟周期内完成取指、译码、执行、访存、写回与 PC 更新六个阶段，然后下一条指令才开始。这种实现方式在结构上最直观，因为它直接体现了 ISA 语义是如何映射到硬件行为的。

SEQ 由组合逻辑与存储元件共同构成。组合逻辑负责对当前状态进行计算，例如指令译码、ALU 运算、地址生成与条件判断；存储元件负责在时钟边沿更新寄存器文件、条件码寄存器、程序计数器和状态寄存器。

六阶段执行模型

SEQ 的六阶段如下：

graph LR
    A["取指 Fetch"] --> B["译码 Decode"]
    B --> C["执行 Execute"]
    C --> D["访存 Memory"]
    D --> E["写回 Write Back"]
    E --> F["PC 更新 PC Update"]

取指阶段从 PC 指向的位置读取指令字节，解析出 icode、ifun、寄存器编号与常量，并计算顺序下一条指令地址 valP。译码阶段从寄存器文件读取源操作数。执行阶段通过 ALU 计算算术结果、有效地址或栈指针新值，并在需要时设置条件码。访存阶段读取或写入数据存储器。写回阶段把 valE 或 valM 写回寄存器文件。最后，PC 更新阶段根据指令类型与条件判断结果，选择新的程序计数器。

这种阶段划分并不意味着六个阶段跨越六个周期。在 SEQ 中，六个阶段属于同一周期中的连续组合逻辑路径。

典型指令的六阶段语义

算术逻辑指令 OPq rA, rB 的行为最典型。取指出 icode:ifun 与 rA:rB，得到 valP = PC + 2；译码时读出 valA = R[rA] 和 valB = R[rB]；执行时计算

valE=valBOPvalA

并更新条件码；访存阶段为空；写回时把 valE 写回 R[rB]；最后令 PC = valP。这类指令展示了“寄存器读 - ALU 运算 - 条件码更新 - 寄存器写回”的基本路径。

写内存指令 rmmovq rA, D(rB) 在取指阶段额外读出 8 字节位移 valC，并令 valP = PC + 10；译码时读取源值 valA = R[rA] 和基址 valB = R[rB]；执行阶段计算有效地址

valE=valB+valC

访存阶段执行

M8​[valE]←valA

写回阶段为空，PC 最终更新为 valP。

读内存指令 mrmovq D(rB), rA 与之对称。执行阶段仍先形成有效地址 valE = valB + valC，访存阶段读取

valM=M8​[valE]

写回阶段执行

R[rA]←valM

栈类指令的关键在于 %rsp 既参与地址形成，又在写回阶段更新。pushq rA 的执行阶段用加法实现减法：

valE=R[%rsp]+(−8)

访存阶段把 valA 写入地址 valE，写回阶段把 %rsp 更新为 valE。popq rA 则在译码阶段同时把旧 %rsp 读到 valA 与 valB，执行阶段算出 valE = valB + 8，访存阶段从旧栈顶地址 valA 读取 valM，写回阶段同时执行

R[%rsp]←valE R[rA]←valM

call Dest 的核心是“把返回地址压栈，再跳转到目标地址”。其返回地址正是 valP。因此执行阶段先计算新栈顶 valE = R[\%rsp] - 8，访存阶段把 valP 写入 M_8[valE]，写回阶段更新 %rsp，最后令 PC = valC。ret 则相反：访存阶段从旧栈顶地址读出返回地址 valM，写回阶段恢复 %rsp，PC 更新阶段选择 valM 作为新地址。

条件跳转 jXX Dest 在执行阶段求值

Cnd=Cond(CC,ifun)

PC 更新阶段据此二选一：若条件成立则 PC = valC，否则 PC = valP。

条件传送 cmovXX rA, rB 与普通 rrmovq 使用同一类编码，不同之处在于写回是否真正发生。由于 SEQ 的写口必须来自 valE 或 valM，而不能跨阶段直接拿 valA，因此执行阶段把 valA 通过 ALU“透传”为 valE，写回阶段仅在 Cnd 成立时执行 R[rB] \leftarrow valE。irmovq 的实现与此类似，只不过透传对象换成 valC。

SEQ 的控制逻辑

取指阶段首先决定一条指令是否需要寄存器说明符与立即数。以 HCL 形式表示，need_regids 对 rrmovq/cmovXX、OPq、pushq、popq、irmovq、rmmovq、mrmovq 为真；need_valC 对 irmovq、rmmovq、mrmovq、jXX 与 call 为真。instr_valid 则用于判断 icode 是否属于已定义指令集合。

译码与写回阶段的关键是寄存器端口选择。srcA 表示需要从寄存器文件哪个端口读出 valA，srcB 表示哪个端口读出 valB；dstE 和 dstM 分别表示 ALU 结果与内存读出值应写回到哪个寄存器。其设计体现了一条通用原则：源寄存器由译码阶段决定，目的寄存器由指令语义决定，而写回时只允许 valE 或 valM 作为数据来源。

执行阶段需要决定 aluA、aluB 与 alufun。对于 OPq，aluA = valA，aluB = valB，alufun = ifun。对于访存指令，aluA = valC，aluB = valB，ALU 统一使用加法形成有效地址。对于 call、pushq、ret、popq，ALU 也统一使用加法，通过输入 -8 或 8 来改变 %rsp，从而避免额外设计减法控制路径。只有 OPq 需要设置条件码，因此 set_cc 只在 icode = IOPQ 时为真。

访存阶段需要决定三件事：是否读内存、是否写内存，以及地址和写入数据来自哪里。mrmovq、popq 和 ret 需要读内存；rmmovq、pushq 和 call 需要写内存。地址通常来自 valE，但 popq 和 ret 的读地址来自旧栈顶 valA。写入数据对 rmmovq 与 pushq 取自 valA，对 call 取自 valP。

最后，PC 更新逻辑体现指令控制流语义。若指令为 call，则新 PC 取 valC；若为条件跳转且条件成立，则新 PC 也取 valC；若为 ret，则新 PC 取 valM；其余情况一律取 valP。

SEQ 的时钟语义与局限

SEQ 的整个时钟周期遵循“先计算，后统一更新”的原则。周期开始时，PC、寄存器文件、条件码与内存保持旧值；组合逻辑根据这些旧值传播并产生所有中间信号；在周期末的时钟边沿，所有状态元件同时接收新值。

这一设计带来一个重要性质：为了完成一条指令，不需要在同一周期内重新读取由该指令自己刚刚写入的状态。例如 pushq 不会先把 %rsp 写成新值，再读出新 %rsp 作为访存地址；相反，执行阶段直接计算 valE = old\_rsp - 8，然后把同一个 valE 同时作为内存写地址和 %rsp 的新值。这种“从不回读本周期新状态”的做法使组合逻辑与状态更新分离得非常清晰。

SEQ 的根本缺点是时钟周期过长。因为任意一条指令都必须在一个周期内穿过取指、寄存器文件、ALU、数据存储器和写回等整条最长路径，所以时钟周期必须迁就最慢指令。真实处理器无法接受这种低吞吐量实现，因此需要引入流水线。
