项目分析与因子分析

Summary

这一讲的核心就两件事：项目分析（这道题好不好？）和因子分析（这张卷子到底测了几个维度？）。项目分析里，CTT 的经典指标（难度P、区分度D）是基础必考；IRT 的 a/b/c 参数和信息函数是进阶重点，理解”不同题目在不同能力区间有不同区分力”是关键。因子分析则要分清 EFA（探索）和 CFA（验证），记住样本量要求和旋转方法的选择原则。整章内容环环相扣，考试时特别喜欢考概念辨析和适用条件。

项目分析的目标与基本框架

项目分析是对测验题目质量进行系统评估的过程，其目的在于筛选、修订或删除不合适的项目，从而提高测验的信度、效度与解释质量。项目分析既包括定性分析，也包括量化分析。定性分析关注测验内容取样是否恰当、题目表述是否清楚、内容是否符合测量目标；量化分析则依据作答数据考察题目的难度、区分度以及备选答案是否发挥应有功能。项目分析的核心功能不是简单计算指标，而是服务于测验编制中的决策：保留哪些题目、修改哪些题目、删除哪些题目。

flowchart LR
    A[项目分析] --> B[定性分析]
    A --> C[量化分析]
    B --> B1[内容取样是否适当]
    B --> B2[题目表达是否清楚]
    B --> B3[内容效度与编写质量]
    C --> C1[题目难度]
    C --> C2[题目区分度]
    C --> C3[选项功能]
    C --> C4[IRT参数表现]
    C --> D[项目决策]
    D --> D1[保留]
    D --> D2[修改]
    D --> D3[删除]

项目分析主要适用于最大作为测验，即以能力上限表现为目标的测验，如学业选拔测验、能力测验等。在典型作为测验中，例如人格、兴趣等测验，项目“难度”不再表示会不会做，而更接近“通俗性”或“被 endorsing 的比例”。因此，同一指标在不同测验类型中的心理含义并不完全相同。

题目难度

难度的含义

题目难度通常用通过率或答对率表示，记作 P。

其基本思想是：答对该题的人越多，题目越容易；答对的人越少，题目越难。因此，在心理测量中，“难度值大”并不表示题目更难，而是表示题目更容易。这一点与日常语言直觉相反，必须明确区分。

Trap

考试重灾区！ 日常我们说“这道题难度高”，意思是题很难；但在心理测量里，P 值越大表示答对的人越多，题越容易。这个反向关系一定要刻进DNA，考卷上见到“难度高”先想清楚问的是日常语义还是测量学定义，不然直接送分。

对于二分计分项目，难度定义为：

P=NR​

其中，P 为题目难度，R 为答对该题的人数，N 为参加测验的总人数。该公式用于估计某道题的通过率。其结果范围在 0 到 1 之间。P 越接近 1，题目越容易；P 越接近 0，题目越困难。

  • 该指标反映的是群体层面上的项目容易程度，而不是某个个体主观感受到的难易。
  • 它不能单独说明题目是否优质，因为过易与过难都可能削弱区分功能。

在早期手工分析中，也常用高低分组法近似估计难度：

P=2PH​+PL​​

其中，PH​ 表示高分组在该题上的通过率，PL​ 表示低分组在该题上的通过率。

高低分组法通过按总分排序并抽取上下各 27% 的极端被试，构造出在测验总水平上差异较大的两组，以总分作为目标特质水平的近似指标，进而考察某一项目是否能够区分高水平与低水平被试。

该方法利用两端样本近似整体情况，在无条件处理全部数据时具有实用价值，但在现代数据分析中通常不如全样本比例直接。

非二分计分项目的难度

当项目不是对错二分，而是具有多个得分等级时，难度不能直接用答对率表示，而应根据平均得分与满分的比例估计：

P=Xmax​Xˉ​

其中，Xˉ 为所有被试在该项目上的平均分，Xmax​ 为该项目的最高得分。该公式用于多级评分项目，如 0、1、2 分题目。结果仍然表示该题目相对容易程度。P 越大，说明群体平均得分越接近满分，该项目越容易。

它不能区分“项目容易”与“评分标准宽松”这两种来源，解释时必须结合题目内容与评分规则。

猜测校正

在是非题或选择题中，答对有可能来自猜测，因此原始通过率会夸大项目的真实容易程度。为校正猜测，可使用校正通过率公式：

PC​=1−K1​P−K1​​

等价地，也可写为：

PC​=1−K1−PK​

其中，PC​ 为校正后的通过率，P 为实际通过率，K 为选项数目。

  • 该公式解决的是“随机猜测抬高得分”问题，适用于存在明确猜测概率的客观题。
    • 其逻辑是先扣除猜中的机会成分，再对剩余比例重新缩放。结果代表在控制纯猜测影响后，项目的相对通过程度。
  • 该指标不能消除部分知识基础上的“排除式猜测”，也不能替代对选项功能本身的分析。

Example

来道具体例题，搞懂猜测校正到底在干什么： 例如，某五择一题共有 80 名学生作答，其中 32 人答对，则原始通过率为：

P=8032​=0.40

由于 K=5，猜测率为 51​=0.20。代入校正公式：

PC​=1−0.200.40−0.20​=0.800.20​=0.25

校正后难度为 0.25。这表明若排除随机猜测成分，该题真正反映掌握水平的通过率低于表面上的 0.40。该结果适用于分析题目本身难度，不宜直接当作学生真实能力水平的绝对指标。

难度水平的确定

难度并不存在脱离测验目的的统一最优值。

  • 其取值应根据测验用途、测验性质和题型特征综合确定。
  • 选拔测验通常要求项目难度与录取率或选拔比例相匹配；
  • 标准参照测验则主要判断是否达到既定标准，因此一般不像常模参照测验那样强调难度分布；
  • 选择题至少应明显高于随机猜测率，否则题目基本失去实际意义。
  • 理论上，难度接近 0.50 时最有利于区分个体差异，但在实际测验中，不可能也没有必要让所有项目都保持在 0.50。

难度的等距转换

原始难度值 P 只是比例意义上的顺序信息，不是天然的等距尺度，直接用于进一步统计分析会受到限制。同时，P 越大题目越容易，也容易造成语义上的误读。因此，在样本足够大、测验编制较好且总分近似正态分布的条件下，可以将难度值视为正态曲线下的累积面积，并通过标准正态分布表转换为等距的标准分数。

转换后的标准分数通常为 Z 分数，其均值为 0、标准差为 1。为避免出现负数和小数，也可进一步线性变换为：

Z′=Z+5

其中，Z′ 的均值为 5，标准差为 1。还可转换为：

Δ=13+4Z

其中，Δ 分数的均值为 13，标准差为 4。这些转换的目的在于把原本非等距的难度比例映射为便于比较和运算的量尺。

Note

难度等距转换速查：

  • Z 分数：均值 0，标准差 1（可能有负数）
  • Z′=Z+5：均值 5，标准差 1（全为正，好读）
  • Δ=13+4Z：均值 13，标准差 4（ETS常用）

记住：这是对”难度值”做标准化，不是给被试分数转标准分，别搞混了。

这里的“标准化”是对难度指标进行量尺变换，不等于对被试原始分数进行标准分转换，两者不可混同。

    A[项目作答结果] --> B[计算通过率 P]
    B --> C{是否需控制猜测}
    C -- 是 --> D[计算校正通过率 Pc]
    C -- 否 --> E[保留 P]
    D --> F[作为正态面积转换]
    E --> F
    F --> G[Z 分数]
    G --> H[Z' = Z + 5]
    G --> I[Δ = 13 + 4Z]

难度对测验的影响

  • 项目难度影响测验总分的分布形态。
    • 若项目整体偏易，总分会集中在高端，形成负偏态；
    • 若项目整体偏难，总分会集中在低端，形成正偏态。
  • 项目难度影响测验信度。
    • 过难或过易的项目都会使得作答结果趋于集中，缩小总分全距，从而降低个体差异的可观测程度，使信度下降。
  • 项目难度与区分度密切相关
    • 当 P 接近 0 或 1 时，大多数人作答结果相同，项目几乎不能区分个体；当 P 接近 0.50 时，区分潜力通常较高。

题目“容易”并不等于“差”，题目“困难”也不等于“好”。在完整测验中，需要根据测量目标配置不同难度水平的项目，以覆盖目标特质范围。过度追求所有项目都在中等难度，会损害内容代表性与测量范围。

区分度

区分度的含义

区分度又称鉴别力，通常记作 D，是指项目对被试心理品质差异的区分能力。理想的项目应使高水平被试更可能答对、低水平被试更可能答错。区分度因此成为评价项目质量与筛选项目的重要依据。它所测量的不是题目的内容范围，而是项目对个体差异的敏感性。

鉴别指数法

最常见的区分度计算方法是极端组鉴别指数法：

D=PH​−PL​

其中，D 为鉴别指数，PH​ 为高分组通过率，PL​ 为低分组通过率。高低分组通常依据总分排序后取上下各 27% 作为极端组。

该公式解决的是“该题能否把高能力者与低能力者分开”的问题。

  • D 越大，说明高分组比低分组更容易通过该题，项目区分度越好。
  • 若 D 接近 0，说明项目几乎不能区分个体；
  • 若 D 为负，说明低分组反而比高分组更容易答对，常提示题目或答案存在严重问题。

常见评价标准为：

  • D≥0.40 时项目很好；
  • 0.30≤D≤0.39 时良好，修改后更佳；
  • 0.20≤D≤0.29 时尚可，但仍需修改；
  • D≤0.19 时较差，通常应淘汰。

该阈值属于经验性标准，不能脱离题型、样本性质和测验目的机械套用。

Note

鉴别指数 D 的评价标准（经验值，眼熟即可，别机械套用）：

  • D≥0.40：优秀
  • 0.30∼0.39：良好，修一下更好
  • 0.20∼0.29：凑合，建议修改
  • D≤0.19：拉胯，建议淘汰

相关法

区分度还可通过项目分数与外部效标分数或测验总分之间的相关来估计。根据变量类型不同，可采用积差相关、点二列相关或二列相关。其共同逻辑是：若某题得分与总体能力或相关效标高度一致，则该题具有较强区分作用。

（1）积差相关

积差相关适用于项目分数与效标分数都为连续变量的情形。其核心思想是考察项目得分与总分或外部标准之间的线性关系。若用项目与总分相关来表示区分度，则该相关越高，说明项目越能反映整体能力差异。

需要注意的是，当某题在总分中占比较大时，项目与总分的相关会被人为抬高，因此应从总分中扣除该项目分数，再计算修正后的题总相关。否则容易高估项目区分度。

Trap

算项目与总分的积差相关时，一定要把这题的得分从总分里扣掉再算！不然这题在总分里占着权重，相关会被人为抬高，区分度就虚高了。这个小细节考试经常挖坑。

（2）点二列相关

点二列相关适用于项目为 0/1 记分，而效标变量为连续变量的情况。其公式为：

rpb​=St​Xˉp​−Xˉt​​pq​​

效标分数就是用来和该题作相关比较的连续分数，最常见的是测验总分，也可以是外部标准分数。

这里，rpb​ 为点二列相关系数，Xˉp​ 为答对该题者在效标上的平均分，Xˉt​ 为全体被试效标平均分，St​ 为全体被试效标分数标准差，p 为该题通过率，q=1−p。

该系数适用于二分项目，前提是连续变量分布单峰且大体对称。通常认为题总相关大于 0.30 的题目具有较好的区分度。

点二列相关衡量的是项目与连续能力指标之间的一致程度。它能说明项目是否有助于区分高低水平个体，但不能直接说明题目内容是否代表目标领域，也不能单独证明题目有效。

（3）二列相关

二列相关适用于本质上连续但被人为二分的变量。例如某题原本有 0 到 6 分的连续得分，却为了分析方便把“高于 3 分”与“不高于 3 分”分成两类。

二列相关试图恢复这一人为二分之前的连续性信息，因此其使用条件比点二列更严格：被二分前的变量应近似正态分布，连续变量本身也应呈单峰对称分布。该系数可用于估计这类二分项目的区分度。

flowchart LR
    A[区分度估计] --> B[鉴别指数法]
    A --> C[相关法]
    B --> B1[高低分组]
    B --> B2[D = P_H - P_L]
    C --> C1[积差相关<br/>项目与效标均连续]
    C --> C2[点二列相关<br/>项目二分 效标连续]
    C --> C3[二列相关<br/>连续变量被人为二分]

>[!note]
> 
> 三种相关方法怎么选？速查表记一下：
> - **积差相关**：项目分数 & 效标分数 **都是连续变量**
> - **点二列相关**：项目 **0/1 二分**，效标 **连续**
> - **二列相关**：变量**本质连续，但被人为切成了二分**（比如把0-6分切成及格/不及格）

难度与区分度的关系

Intuition

难度和区分度的关系可以这样理解：如果一道题所有人都对或所有人都错，那它根本没法区分谁强谁弱，区分度自然为0。只有当难度适中（P≈0.5）时，才有可能把高水平者和低水平者彻底拉开。所以难度决定了区分度的天花板。 难度直接限制区分度的上限。当 P=0 或 P=1 时，所有人作答一致，区分度必为零。一般而言，中等难度项目具有更高的区分潜力。

  • 当 P=0.5 时，区分度最大值可达 1；
  • 随着 P 偏离 0.5，最大可达区分度逐渐下降。

由此可见，难度不是与区分度彼此独立的两个指标，而是相互约束的。

项目选择时，通常将难度与区分度联合考虑，而不是单看某一个指标。一个适宜的项目应既不过易不过难，又能有效区分不同水平个体。

区分度与信度的关系

区分度越高，通常意味着测验信度越高。

其逻辑基础在于：高区分度项目更能反映被试真实能力差异，因此多个高区分度项目组成的测验，其总分更能稳定而准确地表示个体差异。

从数学角度看，区分度常用题总相关表示，而题总相关越高，通常越有利于提升内部一致性信度。

但“区分度高”不等于“测验一定可靠”。信度还受到测验长度、样本异质性、误差来源等因素影响。区分度只是内部一致性提升的重要条件之一，而非充分条件。

区分度的相对性

区分度不是脱离样本与方法的固定属性。

  1. 极端组划分标准不同，所得区分度会不同，通常采用上下 27% 只是经验上较优的选择。
  2. 样本越同质，个体差异越小，区分度往往越低，因此同一项目在不同样本中可能表现不同。
  3. 采用不同公式会得到不同区分度估计。
  4. 样本容量影响统计稳定性，实际解释时应结合显著性检验与置信程度，而不是只看单个数值。

项目反应理论在项目分析中的应用

IRT 的基本思想

项目反应理论通过数学模型刻画被试潜在能力与项目正确作答概率之间的关系。

其核心工具是项目特征曲线，简称 ICC。

  • 潜在特质水平 θ 是指个体在某一目标心理特质上的位置或高低程度。它不能被直接观察，只能通过项目反应加以估计。
  • ICC 则描述了当 θ 变化时，个体答对某项目的概率如何变化。

与经典测验理论只用总分统计项目不同，IRT 把项目参数与被试能力置于同一潜在量尺上，因此可以更细致地描述项目性能。

Intuition

CTT 时代我们看的是”这道题全班有多少人做对”；IRT 时代我们看的是”一个能力值为 θ 的人，碰到这道题有多大的概率做对”。想象一下，IRT 给每个人都安了一个”能力值 GPS”，也给每道题贴了”难度、区分度、猜测率”三个标签，然后在同一把尺子上看它们怎么互动——这就是 ICC 曲线在干的事。

Important

重点： IRT 在项目分析中的核心不只是“给题目打分”，而是通过项目特征曲线把题目难度、区分度与猜测因素统一纳入概率模型中。这样得到的参数具有更明确的心理测量含义，也使测量精度可以随能力水平而变化。

三参数 Logistic 模型

三参数 Logistic 模型（3PM），其参数包括难度 b、区分度 a 与猜测率 c。模型可写为：

P(θ)=c+(1−c)1+e−a(θ−b)1​

其中，P(θ) 表示能力水平为 θ 的被试答对该题的概率；θ 为被试潜在特质水平；a 为项目区分度参数，反映曲线斜率；b 为项目难度参数，反映曲线拐点位置；c 为猜测参数，表示极低能力水平下仍可能答对的下限概率。

Note

这是一经过变换的sigmoid曲线。其标准形式为y=1+e−x1​

从这个角度来看，参数c负责平移（上移）和放缩曲线纵轴尺度（负相关），a负责放缩曲线横轴尺度（正相关），b负责平移曲线（右移）。

该公式用于在潜在特质量尺上估计项目性能。结果不是一个固定通过率，而是一条随能力变化的概率曲线。它不能脱离模型拟合与样本条件单独解释，也不能简单与 CTT 中的通过率或题总相关一一对应。

参数的心理含义

难度参数 b 对应 ICC 曲线拐点的横坐标。曲线越向右，说明需要更高的特质水平才有较高答对概率，因此项目越难。最右侧项目要求的心理特质水平最高。这里的“难度”不再是群体中的通过率，而是潜在能力量尺上的位置参数。

区分度参数 a 由曲线在拐点附近的陡峭程度表示。曲线越陡，说明在某一能力区间内，能力稍有变化就会显著影响答对概率，因此项目越能区分相近能力者。区分度越高，项目提供的信息量通常越大，预测误差越小。课件特别指出，当 Pi​(θ) 接近 0.5 时，项目信息量最大。

猜测参数 c 表示项目特征曲线最小的概率下限。对于极低能力被试，若仍可能因猜测而答对，则曲线不会从 0 起始。对 K 选一选择题，c 常接近 K1​。猜测参数会降低项目在低能力区间的信息量，因为低能力者的反应不再完全由能力决定。

Note

IRT 三参数速记口诀：

  • b（难度）：曲线拐点在哪，越右越难
  • a（区分度）：曲线陡不陡，越陡区分越好
  • c（猜测率）：曲线最低点到哪，选择题里通常接近 1/K
flowchart LR
    A[潜在能力 θ] --> B[项目特征曲线 ICC]
    B --> C[难度 b<br/>曲线位置]
    B --> D[区分度 a<br/>曲线斜率]
    B --> E[猜测率 c<br/>曲线下限]
    C --> F[越右越难]
    D --> G[越陡区分越好]
    E --> H[低能力区仍可能答对]

IRT 图解的解释原则

比较多条 ICC 时，曲线位置决定相对难度，曲线斜率决定相对区分度，曲线下限决定猜测影响。

  • 若两条曲线位置相同但其中一条更陡，则它们难度相近，但更陡者区分度更高。
  • 若一条曲线明显更靠右，则它对更高特质水平者更有用。

举例子：某些项目只适合测量低特质水平个体，另一些项目只适合测量高特质水平个体。

由此可见，项目好坏不是绝对的，而是取决于测量目标范围。

Intuition

你可以把 ICC 曲线想象成”能力-答对概率”的地图。b 是地图上的”关卡位置”（在哪一关需要这个能力），a 是”关卡敏感度”（能力差一点是不是就过不去了），c 是”新手保护”（完全不会的人也有基础概率蒙对）。这样记三个参数的含义就很形象了。

Important

重点： 不同项目在不同特质区间具有不同效用。难度较低的项目更适合区分低水平个体，难度较高的项目更适合区分高水平个体。项目“有用”不是全区间统一的，而是依赖于其信息主要集中在哪一段能力区间。

信息函数与测量精度

在 IRT 中，信息表示项目或测验降低不确定性的能力。项目信息函数描述项目在不同能力水平上的区分效能。信息越高，说明该项目在该能力区间对个体差异越敏感，测量误差越小，测量精度越高。

项目信息函数通常由项目反应函数的斜率推导而来，因此区分度越高、猜测越少的项目通常能提供更多信息。

这一定义与 CTT 中“整个测验只有一个整体信度系数”的思想不同。IRT 认为测量精度会随个体所在特质区间而变化，因此同一份测验对不同被试可能具有不同精度。这也是自适应测验成立的理论基础：系统可以根据被试当前能力估计，动态选择在该能力区间信息量最大的项目，以达到预定精度。

Intuition

CTT 给你一个统一的信度系数，比如 α=0.85，意思是”这份卷子整体还行”。但 IRT 更精细：它告诉你”能力中等的人测得很准，能力极高的人测得不太准”。就像医院里的精密仪器，对不同体征的病人精度不一样。自适应测验就是靠这个原理，老是给你推送”刚刚好能测出你水平”的题。

flowchart TD
    A[项目反应函数 IRF] --> B[求导与曲线形态]
    B --> C[项目信息函数 IIF]
    C --> D[信息高]
    D --> E[测量误差低]
    E --> F[测量精度高]
    C --> G[信息集中于特定特质区间]

IRT 的不变性

IRT 的魅力在于其不变性。

  • 对被试而言，只要项目反应函数已知，潜在特质可以由任何一组已校准项目反应估计出来，这体现了跨题目比较的可能性。
  • 对项目而言，理论上项目反应函数不依赖于某一特定样本群体，而是具有相对稳定的参数结构，这为跨人群比较提供基础。
  • 不同样本只是在估计中帮助更精确地确定曲线不同部分，而不应改变项目参数所代表的本质关系。

这里的“不变性”是模型理想性质，而非经验上自动成立的事实。它依赖于模型拟合良好、单维性与参数稳定等前提。若模型不合适或样本差异导致项目功能改变，则不变性会受到挑战。

信度理论模型的扩展视角

信度并不是只有一个单一框架，不同理论对“测量稳定性与精确性”的理解并不完全相同。

  • 经典测验理论认为观察分数由真分数与误差分数组成，优点是简单易用，缺点是难以细分复杂误差来源。
  • IRT 通过被试能力与项目参数的关系刻画测量精度，优点是精细且可实现自适应测验，但需要较大样本与复杂计算。
  • 概化理论将 CTT 与方差分析结合，把测量误差细分为被试、项目、评分者及其交互作用等多个来源，用概化系数取代单一信度系数。
    • 其核心思想是从具体情境推广到更广泛测量条件，因此真分数概念被扩展为“总体分数”或“全域分数”。
  • 因子分析与结构方程模型更多用于结构效度与复杂测量模型评估，多特质多方法矩阵则主要用于收敛效度与区分效度分析。
    • 它们并不直接替代传统信度指标，但构成理解测量质量更完整的理论版图。

项目分析中的实践问题

常模参照与标准参照测验

常模参照测验的目的是比较个体与群体中其他人的相对位置，因此更依赖区分度，强调项目能否最大限度地区分被试差异，并据此筛选理想项目。标准参照测验的目的是判断个体是否达到某一内容标准或教学目标，因此更强调项目内容与课程目标的相关性，而不是个体间差异的最大化。

Important

重点： 两类测验在项目分析上的核心差异，不在于是否计算难度和区分度，而在于项目优劣的判断标准不同。常模参照测验更依赖区分度；标准参照测验更依赖内容相关性与目标代表性。

Trap

常模参照和标准参照的差异主要在分数解释方式，而不是题型本身。同一道题既可以拿来排名，也可以拿来看达没达标，区别是你怎么解读分数。考试喜欢考这个边界，别被绕进去。 标准参照测验中，只要题目能够反映教育目标或被认为是重要内容，就可以纳入测验。因此它较少依据传统难度和区分度来删题。其难度常通过比较教学或训练前后的得分变化来理解，即看项目是否能反映掌握水平变化。由于这类测验中被试分数变异往往较小，相关法不一定适用，区分度分析可采用类似鉴别指数的方法。

flowchart LR
    A[测验解释取向] --> B[常模参照测验]
    A --> C[标准参照测验]
    B --> B1[关注人与人相对差异]
    B --> B2[强调区分度]
    B --> B3[用于选拔与排序]
    C --> C1[关注是否达到标准]
    C --> C2[强调内容相关性]
    C --> C3[用于达标判断]

这里必须明确边界：常模参照与标准参照的差异主要在分数解释方式，而不是题型本身。相同题目可以在不同解释框架下使用。前者不能直接推出个体是否掌握了某一标准，后者也不能据此精确判断个体在群体中的相对名次。

速度测验不宜使用常规项目分析

速度测验中的项目通常较容易，但题目数量多、作答时限短，导致大多数被试无法完成全部项目。这时项目位置会混入时间限制因素：前面题目通过率高，后面题目通过率低，未必是因为后面更难，而可能只是来不及作答；

同理，后面题目的“高区分度”也可能只是因为只有能力强、反应快或总分高者能做到后面。因此，传统难度和区分度分析不适用于速度测验。

这说明项目分析默认假定作答机会相对均等，而速度测验破坏了这一前提。若忽视此点，就会把时间位置效应误当成题目质量差异。

Trap

速度测验千万不能套用常规项目分析！ 后面的题通过率低，未必是因为难，很可能只是大家没时间做。如果硬算难度和区分度，你会把时间效应误判为题目质量。记住：项目分析的前提是”大家都有机会做完”，速度测验直接打破了这个前提。

选择题的选项分析

选择题分析不能只看正确率，还要分析各备选项是否发挥了应有功能。

Example

想象一下，你是出题人。如果你设计的干扰项没人选，说明它太假了；如果高分组在正确项和某个干扰项之间摇摆，说明这个干扰项伪装得太好，或者题目本身有歧义。选项分析就是当”侦探”，看考生的作答模式是不是符合预期。

常见异常情形及其含义包括：

  • 若所有人都选正确答案，项目可能过易，或正确选项带有明显暗示；
  • 若无人选择某错误选项，则该干扰项缺乏迷惑性。
  • 若选择率低于约 2% 可考虑修改或删除；若所有人都选错误答案，则可能是答案设置有误或教学本身出现问题；
  • 若高分组集中在两个答案之间，可能存在两个正确答案或表述含混；
  • 若高低分组正确率相同，则项目可能与目标能力无关，应删除或大改；
  • 若各选项作答人数几乎相同，则题目可能过难或题意模糊，导致随机作答。

这些现象表明，项目分析不仅是“这一题得分高不高”的问题，更是“作答模式是否符合理论预期”的问题。对于选择题而言，错误选项不是陪衬，而是影响项目性能的重要组成部分。

客观题中的猜测问题

客观题尤其是是非题和少选项选择题容易受到猜测影响。猜测会提高表面得分，使难度被高估，区分度和信度的解释也可能受扭曲。处理方式包括使用难度校正公式、设置“不知道”选项，或对答错题施加扣分。课上给出的校正规则可写为：

校正后分数=R−n−1W​

其中，R 为答对题数，W 为答错题数，n 为每题选项数。

Note

校正分数公式速记： 校正后分数=R−n−1W​ R=答对题数，W=答错题数，n=每题选项数。 这个公式核心思想是：把瞎猜猜错的期望损失扣掉。注意只适用于”纯猜测”情境，对有策略的排除法猜题也没辙，实际使用要谨慎。 该公式的思想是从总分中扣除随机猜错所带来的期望收益。它适用于客观题分数校正，但不能保证完全排除非随机猜测，也可能改变被试作答策略，因此在实践中需要结合测验目标谨慎使用。

有效性与可靠性的张力

项目筛选标准并不总是统一指向同一种“最佳项目”。

  • 若以项目与测验总分的相关作为主要依据，则倾向于保留与整体结构高度一致的项目，这有利于提高内部一致性信度和同质性，适用于检验理论构想的测验。
  • 若以项目与外部效标的相关为依据，则可能需要保留一些与总分相关不高、但能扩展内容范围和预测效度的项目，这在预测性测验中更重要。

因此，项目分析并不存在脱离用途的统一最优准则。提高内部一致性并不总能提高效度，扩大内容范围也可能降低内部一致性。可靠性高不等于效度高；效度提升有时还需要容忍一定程度的内部异质性。

因子分析

因子分析的功能

因子分析是一类用于揭示变量背后潜在结构的统计方法。其基本功能包括降维、提取共同因子、估计各变量在因子上的权重，并在必要时依据因子得分计算综合分数。它不是简单压缩变量数量，而是试图说明观测变量之间相关性的共同来源。

在心理测量中，因子分析的重要意义在于检验测验的结构效度。一个量表究竟测量单一维度还是多个维度，各题目是否按照理论预期聚集到相应维度上，都可以借助因子分析加以评估。

探索性因子分析与验证性因子分析

探索性因子分析（EFA）适用于理论结构尚不明确时，其目标是从数据中探索潜在维度。其特点是默认所有测量变量都可能与每一个潜变量有关，研究者需要根据结果解释因子的意义。EFA 回答的是“数据中可能存在哪些潜在因子”。

验证性因子分析（CFA）适用于理论结构较明确时，其目标是检验某一预设模型是否与数据相符合。研究者需要事先指定因子数量以及每个测量变量应与哪个潜变量相关。CFA 回答的是“给定理论模型是否与观测数据拟合良好”，并可用于比较不同理论模型。

flowchart LR
    A[因子分析] --> B[EFA 探索性因子分析]
    A --> C[CFA 验证性因子分析]
    B --> B1[理论结构未知]
    B --> B2[从数据中发现潜在维度]
    B --> B3[所有变量初始上可与各因子相关]
    C --> C1[理论结构已知]
    C --> C2[预设因子数量与路径]
    C --> C3[检验模型拟合与比较模型]

EFA 与 CFA 不宜在同一批数据上直接连续使用来“先探索再验证”，因为这样会把偶然样本特征再次当作被验证对象，造成验证偏乐观。

较合理的做法是使用不同样本，或将样本分为探索与验证两部分。

Note

也就是说，对于同一份量表，可以先做EFA，再做CFA，但是不能在同一群人身上。

Trap

“先用 EFA 探索结构，再用同一批数据做 CFA 验证”——听起来很完美，其实是大坑。同一批数据的偶然特征会被验证一遍，结果必然虚高。正确做法是换一批人，或者把样本拆成两半，一半探索一半验证。

因子分析的前提

因子分析成立的前提之一是变量之间存在较强的相关或共线性关系。若变量彼此几乎无关，则不存在可提取的共同因子，因子分析难以通过 KMO 检验和 Bartlett 球形检验。

这里要强调，因子分析不是“任何一组变量都能做”的通用降维工具，变量之间必须共享一定共同变异。（可以理解为由同一个隐藏的特质驱动的）

另一个重要前提是样本量。课上给出经验规则：至少应有 50 名被试，且样本量至少为变量数的 3 倍，理想情形下达到变量数的 5 倍以上；同时又指出在实践中一般建议样本量不少于 100，样本越大，因子分析结果越稳定可靠。样本量不足会使相关矩阵不稳定，导致因子数、载荷与旋转结果都容易波动。

Note

因子分析样本量经验法则（考试常考）：

  • 最低底线：n≥50 且 n≥3× 变量数
  • 理想状态：n≥5× 变量数
  • 实际操作：一般不少于 100 人

样本量不够，相关矩阵会很不稳定，因子载荷、旋转结果都会乱跳，得出的结构不可信。

正交旋转与斜交旋转

  • 若提取的各因子被假定彼此不相关，则采用正交旋转。
    • 正交旋转后的坐标轴互相垂直，因子相关为零，优点是解释较为简洁，常用于纯粹的维度压缩。
  • 若认为潜在因子之间可能相关，则采用斜交旋转。
    • 斜交旋转允许因子间相关，因而更贴近许多心理结构的现实情形，并且在斜交结果基础上还可能继续分析更高阶因子。

课上给出的实践原则是：开始时可先假设因子并非独立，优先尝试斜交旋转；若得到的因子间相关较低，例如 r<∣0.30∣，则可以转而使用正交旋转。

该原则体现出心理特质之间往往并非完全独立，过早假定正交可能会牺牲结构真实性。

Intuition

正交旋转就像把坐标轴掰成垂直的，假设”智力”和”创造力”完全没关系；斜交旋转则允许它们有一定夹角，更符合现实——心理特质之间往往是”有点相关但又不完全一样”。所以实际研究中，先尝试斜交，如果因子间相关很低（∣r∣<0.30），再退回正交旋转。

flowchart TD
    A[进行因子分析] --> B{变量间相关是否充分}
    B -- 否 --> C[不宜做因子分析]
    B -- 是 --> D{理论结构是否明确}
    D -- 否 --> E[EFA]
    D -- 是 --> F[CFA]
    E --> G{因子是否可能相关}
    F --> G
    G -- 是 --> H[斜交旋转]
    G -- 否 --> I[正交旋转]

因子分析结果的边界

因子分析只能发现数据中已有的共同结构。若测验材料本身没有包含某一潜在因子，因子分析不可能凭空“提取”出它。因子分析结果依赖于项目内容、样本特征与变量关系，因此它揭示的是在现有测量材料中可以识别的结构，而不是对心理构念本体的最终证明。

此外，因子分析所支持的是结构层面的证据，而不是全部效度证据。即使一个量表因子结构清晰，也不能因此直接推出它具有良好效标效度、预测效度或解释效度。

Trap

因子分析结果清晰 ≠ 构念真实存在！因子分析只是从数据相关结构里”挖”出潜在维度，它挖出来的东西取决于你放进去什么题、测了什么人。如果你没放”焦虑”相关的题目，因子分析不可能凭空给你挖出一个”焦虑”因子来。而且，因子结构好只是结构效度的证据，不能代替效标效度或预测效度。

本章核心辨析

难度高与题目难

在心理测量中，难度值 P 越高，表示答对率越高，题目越容易，而不是越难。日常语言中的“难度高”与测量学中的“难度值高”方向相反，不能混用。

常模参照与标准参照

常模参照关注个体相对位置，强调区分人与人之间的差异；标准参照关注是否达到既定目标，强调项目内容是否代表目标领域。前者不能直接判断是否达标，后者也不适合精细排序。

CTT 与 IRT 的信度观

CTT 倾向于给整个测验提供单一整体信度估计；IRT 则认为测量精度会随潜在特质水平变化，信息越高处精度越高。因此二者不是简单“新旧替代”关系，而是对测量误差结构的不同建模。

高区分度与高效度

高区分度项目通常有助于提升内部一致性和信度，但不必然提高效度。若测验目标是广泛预测外部表现，某些与总分相关较低但与外部效标相关较高的项目仍可能有价值。

因子分析与维度真实存在

因子分析结果反映的是变量相关结构，而不是构念存在的直接证明。只有当测验内容本身包含相应维度，并且样本与模型条件合适时，因子分析才可能揭示该维度。

本章知识主线总结

项目分析的核心在于利用定性与定量信息判断题目是否适合保留。难度说明项目在群体中的容易程度，区分度说明项目能否有效区分高低水平个体，两者必须联合解释。IRT 则进一步把项目分析推进到潜在特质量尺上，通过难度、区分度、猜测率和信息函数解释项目在不同能力区间中的作用。项目分析的具体标准又受到测验解释框架影响：常模参照测验更强调区分，标准参照测验更强调内容目标。因子分析则从结构层面回答测验是否测到了预定维度，并为量表结构效度提供统计依据。

说白了，这章就是在教你：怎么看一道题好不好，以及怎么看一张卷子结构对不对。把 CTT、IRT 和因子分析这三块的关系理清楚，考试就稳了。
