核心洞见：什么是人工神经网络（ANN）？

剥去各种花哨的名词，ANN 的本质就是一个用于函数逼近的框架。我们通过让它“看”大量的数据例子（Learning from examples），让它学会一套从输入（Input）到输出（Output）的映射规则。

这份笔记系统整理了训练一个神经网络所需要面对的所有核心元素：数据怎么喂、模型长什么样、怎么评价好坏、怎么优化更新，以及怎么防止它“死记硬背”（过拟合）。

模块一：神经网络的任务与学习范式

在搭积木之前，我们得先搞清楚我们要建什么。ANN 可以处理多种任务，并对应不同的学习方式。

1. 核心任务

  • 分类 (Classification)：输出离散标签。比如判断一张图片是猫还是狗。

  • 回归 (Regression)：输出连续数值。比如预测明天的气温是多少度。

  • 两者的组合/变体：复杂的现实任务往往是这二者的结合。

2. 学习范式

根据“老师”介入的程度，机器学习可以划分为：

  • 监督学习 (Supervised learning)：有明确的“老师”给出标准答案（Label）。模型预测，算误差，然后纠正。

  • 无监督学习 (Unsupervised learning)：没有老师。模型自己去数据里找规律、找聚类或降维。

  • 强化学习 (Reinforcement learning)：没有手把手的老师，只有一个“延迟的奖励信号”。模型通过不断试错来最大化长期收益。

模块二：数据与特征的现实考量

数据是 ANN 的燃料。但在把数据喂给网络之前，我们会面临维度陷阱，也需要做一系列“预消化”工作。

直觉：维度灾难 vs 数据的真实面貌

理论上的困境（维度灾难）：随着输入数据维度 D 的增加，要覆盖整个特征空间所需的训练样本量会呈指数级爆炸。

现实的希望（结构化数据）：虽然一张 100×100 像素的数字“3”的图片，在数学上位于 10000 维的极高维空间中，但它并没有那么自由！它的实际自由度可能只有 3 个（比如上下平移、左右平移、旋转）。数据的这种“高度结构化”特性，是我们能够用有限样本训练出有效神经网络的底气。

1. 数据预处理 (Pre-processing)

原始数据往往分布得乱七八糟，直接扔进网络会导致训练极其困难。我们需要对数据“整容”：

  1. 标准化 (Standardization)

    • 零均值化 (Zero-centered)：把数据的中心平移到原点。

    • 归一化 (Normalized)：缩放数据，让各个维度的波动幅度差不多。

    • 公式：y=σx−μ​ （其中 μ 是均值，σ 是标准差，运算都是逐元素进行的）。

  2. 白化 (Whitening)

    • 目的：去除不同特征之间的相关性（让数据分布变成一个圆球），加速收敛。

    • 过程：先算出数据的协方差矩阵 Σ，求出特征向量 Φ 和特征值 Λ。

    • 公式：y=Λ−1/2ΦTx （先通过 ΦT 去相关，再用特征值的平方根缩放）。

2. 数据增强 (Data Augmentation)

当数据不够或者为了让模型更“皮实”时，我们会人为制造数据。

  • 操作：翻转、旋转、去色、边缘增强、添加噪声等。

  • 目的：强迫模型学到“不变性”（Invariance，比如不管蝴蝶怎么翻转，它还是蝴蝶），提高抗噪鲁棒性，或者突出需要学习的关键特征。

模块三：模型容量、拟合与评价

我们希望网络能学到规律，而不是“死记硬背”。这就引出了模型容量（Capacity）和奥卡姆剃刀原则。

1. 模型复杂度与多项式拟合实验

假设我们要用 10 个样本点去拟合一条正弦曲线 y=0.5+0.4sin(2πx)，我们尝试用不同阶数的多项式 y=∑ai​xi 来逼近：

  • 0阶/1阶 (M=0, 1)：直线。模型太简单，连基本的波浪形都画不出，这叫 欠拟合 (Underfitting)。

  • 3阶 (M=3)：一条平滑的曲线，完美穿过并拟合了整体趋势。

  • 9阶 (M=9)：曲线发了疯似地上蹿下跳，完美经过了每一个训练数据点，但在未知的点上错得离谱。这叫 过拟合 (Overfitting)。此时看多项式的权重 w∗，会发现它们变成了极大或极小的异常数值（比如几十万）。

奥卡姆剃刀 (Occam's Razor)

在所有能同样好地解释已知观测数据的假设中，选择最简单的那一个！（简单有效原则）

2. 误差的三种形态

为了监控这种“死记硬背”，我们必须把数据切分，并在不同的数据集上计算误差：

  1. 训练误差 (Training error)：在训练集上算的误差。（过拟合时，它会降到 0）

  2. 泛化误差 (Generalization error)：在完全没见过的新数据上测试的误差。这才是我们真正关心的指标！

  3. 验证误差 (Validation error)：在训练过程中留出一小撮数据不参与反向传播，专门用来定期“摸底考试”，这就是验证集。

3. 泛化鸿沟与早停机制 (Early Stopping)

随着模型容量（或者训练步数）的增加，训练误差会一直往下降。但泛化误差会先降后升，两者之间裂开的那个巨大的口子，就叫 泛化鸿沟 (Generalization gap)。

graph LR
    A[开始训练] --> B(监控训练误差和验证误差)
    B --> C{验证误差是否连续上升?}
    C -- 否 --> D[继续训练一个 Epoch]
    D --> B
    C -- 是 --> E[早停 Early Stopping]
    E --> F[退回并保存验证误差最低时的权重]

停止准则 (Stopping criteria)：

  1. 训练误差不再下降：∣ΔE∣<δ （δ 是个极小值）。

  2. 早停 (Early stopping)：监控验证误差，一旦发现验证误差不再下降反而开始上升，说明模型开始“死记硬背”了，立刻停止训练。

模块四：网络构建的核心组件

1. 目标函数 / 误差函数 (Objective / Error Function)

这是模型优化的“指南针”。

  • 回归任务：通常用均方误差 (MSE)。
E=N1​i=1∑N​(y(xi​,W)−ti​)2
  • 分类任务：通常用交叉熵 (Cross-entropy)。对于二分类（目标 ti​∈{0,1}，输出 y 是概率）：
E=−N1​i=1∑N​ti​logy(xi​,W)

2. 激活函数 (Activation Function)

给网络加入非线性，不然哪怕堆叠一万层，本质上也还是个线性变换。

  • Sigmoid 函数：g(x)=1+e−x1​

    • 致命缺点：当输入很大或很小时，曲线变得极度平坦，这会导致梯度变为 0（Saturated，梯度消失/杀死梯度）。而且指数运算计算代价大。

  • ReLU 函数 (Rectified Linear Unit)：g(x)=max(0,x)

    • 优势：在正半轴永远不饱和，梯度恒为 1；计算极其高效（只要比 0 大就行）。在实际应用中，它比 Sigmoid/Tanh 收敛快得多。

3. 网络初始化 (Network Initialization)

初始权重的位置决定了你在误差地形图上的起点，太偏可能会掉进局部最优。

  • 策略：从均匀概率分布中随机选取，但要对权重进行归一化处理，使得每个神经元的加权输入能保持在“中等范围（midrange）”，防止一开始就落入激活函数的饱和区。

模块五：训练策略与优化算法

1. 训练协议 (Training Protocol)

数据是一次全喂进去，还是一个一个喂？

  • Epoch：把整个训练集的数据完整看过一遍，叫一个 epoch。每次 epoch 开始前要打乱数据顺序 (Randomize)，避免模型学到毫无意义的“先后顺序规律”。

  • 顺序模式 (Sequential / Stochastic / On-line)：每看一个样本，就算一次误差，更新一次权重。

    • 优点：省内存；这种充满随机性的更新轨迹像醉汉走路，反而更容易跳出局部极小值；好写代码。

  • 批量模式 (Batch / Mini-batch)：看一小批 (Batch) 样本，算出平均误差后，再更新一次权重。

    • 优点：收敛速度通常更快；理论更完备；非常适合 GPU 并行计算（现代深度学习的绝对主流）。

2. 优化算法 (Learning Algorithm)

我们需要找到让 E(W) 最小的权重 W。

方法一：梯度下降 (Gradient Descent - 一阶优化)

顺着梯度的反方向往下走。

  • 更新公式：
W=W−η∇W​E(W) 
（其中 $\eta$ 是学习率 learn rate）

方法二：牛顿法 (Newton Method - 二阶优化)

利用泰勒展开到二阶，不光看坡度（一阶导），还看坡的曲率（二阶导/海森矩阵）。

  • 公式推导：将 f(x) 泰勒展开求导置零后得到更新步长 δ=−f′′(xk​)f′(xk​)​。推广到矩阵：
W=W−(∇W2​E)−1∇W​E

  • 优点：收敛极快（直线抄近道走向谷底）。

  • 致命缺陷：计算海森矩阵 ∇W2​E 的逆矩阵在维度极高时（高维 W）的计算代价极其恐怖，所以在深度学习中很少直接使用纯牛顿法。

3. 学习率与动量 (Learning Rate & Momentum)

学习率与动量的直觉解释

学习率 η：就是步子迈多大。太大容易在谷底反复横跳（震荡），太小则走得比蜗牛还慢。

动量 Momentum α：想象一颗小球滚下山。如果它一直顺着同一个方向滚，速度应该越来越快；如果它遇到了坑坑洼洼频繁变向，速度就应该慢下来。

  • 带动量的随机梯度下降公式：
W(t+1)=W(t)−η⋅∇W​E(t)+α(W(t)−W(t−1))
  • 机制：如果连续几步权重的更新方向一致，动量项会放大更新步伐（加速）；如果方向相反（发生震荡），动量项会抵消部分更新（减速求稳）。实际操作中，动量通常设为 0.95 等较大的值。

模块六：正则化 (Regularization)

为了对付过拟合（比如之前提到的 M=9 多项式，权重爆炸），我们除了用早停法和增加数据量（比如把数据从 N=15 加到 N=100），最正统的做法是正则化。

1. 核心思想

不要只在乎拟合数据的误差，还要强行对复杂的模型结构进行惩罚。

E(w)=ED​(w)+λEW​(w)

  • ED​(w)：数据误差（Data Error），希望尽量拟合数据。

  • EW​(w)：权重惩罚项（Weight Penalty），希望权重尽量简单。

  • λ：权衡两者的调节系数。

2. 常见的正则化方法

  • L2 正则化 (Weight Decay / Ridge Regression)

    最常用的形式，假设 q=2：

EW​(w)=21​wTw 
梯度下降更新时会变成：
w=(1−ηλ)w−η∇W​ED​(w) 
也就是说，每次更新前，都先把原来的权重按比例缩水一点点（这就是“权重衰减 Weight Decay”的由来），强迫模型把不必要的极值权重压平。在多项式拟合中，加入 $\ln \lambda = -18$ 的正则项后，原本狂暴的曲线立刻变得平滑。

  • L1 正则化 (Sparse Representation)

    当惩罚项为 q=1（绝对值）时。

    • 几何直觉：L2 的等高线是一个圆（w12​+w22​），数据误差的椭圆等高线碰到圆时，交点通常在各个轴的中间。而 L1 的等高线是个菱形（∣w1​∣+∣w2​∣），误差椭圆去碰菱形时，极其容易碰到菱形的顶点（即某个维度 wi​=0）。

    • 结论：L1 正则化很容易产生稀疏矩阵（迫使很多不重要的特征权重直接清零），常用于特征选择。

  • 其他正则化策略

    • 平滑度约束 (Smoothness constraint)：直接惩罚输出函数的高阶导数 ∫∣∇2y∣dx。

    • 多任务学习 (Multi-task learning)：用同一个隐藏层网络同时去预测多个相关任务（共享表示 h(shared)）。要同时讨好多个任务，迫使网络去寻找更通用、更有普适性的特征，而不是死记硬背某一个任务的特例。

    • 数据增强 (Data augmentation)：前面提过，造点假数据、加点噪音，本身也就是在给模型增加难度，起到极好的正则化效果。
