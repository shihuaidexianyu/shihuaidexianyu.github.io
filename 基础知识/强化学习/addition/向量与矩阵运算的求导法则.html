向量与矩阵运算的求导法则

向量与矩阵运算的求导法则

Apr 22, 202624 min read

Summary

这是一份关于向量与矩阵求导的实战手册。我们会用“微分法”这把统一的钥匙，带你打开从最小二乘到行列式求导的一扇扇门。核心思想只有一句话：先把微分 df 写成标准形式，再直接读出梯度。

1. 记号约定

  • 标量：普通斜体小写字母 a,b,c,f∈R
  • 向量：带箭头的小写字母（默认列向量） x,y​,a,b∈Rn
  • 矩阵：粗体大写字母 A,B,C,X∈Rm×n

默认向量都视为列向量。

Note

约定好记号，就像比赛前统一规则一样重要。在矩阵求导里，不同教材的约定可能天差地别——我们这里采用“分子布局”结合微分法的思路，这样推导起来最自然。

2. 梯度与微分的定义

2.1 标量对向量求导

若 f(x):Rn→R，定义梯度 ∇x​f∈Rn 使得：

df=(∇x​f)Tdx

Intuition

这实际上就是：把 df 写成关于 dx 的线性形式，然后直接读出梯度。你可以把 dx 想象成给向量 x 施加了一个微小扰动，而 df 是这个扰动引起函数值的一阶变化。梯度就是那个“放大倍数向量”。

2.2 标量对矩阵求导

若 f(X):Rm×n→R，定义矩阵梯度 ∂X∂f​∈Rm×n 使得：

df=tr((∂X∂f​)TdX)

因此只要把 df 化成：

df=tr(GTdX)

就可直接读出：

∂X∂f​=G

Intuition

这是后面所有矩阵求导的核心。为什么要引入 trace？因为 trace 能把矩阵的“内积”形式写出来，而且具有循环不变性——这意味着我们可以像摆弄积木一样把矩阵因子换来换去，直到凑出标准形式。

你可以把 trace 想象成矩阵版的“点积”：tr(ATB) 就是矩阵 A 和 B 的 Frobenius 内积。

3. 基本微分法则

3.1 线性法则

常数的微分

d(c)=0

Intuition

常数不随变量变化，因此增量恒为零，所以微分也为零：dc=0。就像一块固定在墙上的石头，你推它，它不动，变化量为零。

和的微分

d(X+Y)=dX+dY

Intuition

按增量线性展开：(X+Y)+Δ(X+Y)=(X+ΔX)+(Y+ΔY)。因此一阶部分就是 d(X+Y)=dX+dY。这和我们熟悉的标量求导 (f+g)′=f′+g′ 一模一样。

数乘的微分

d(αX)=α⋅dX

其中 α 为常数。

Intuition

因为 α 不变，所以 Δ(αX)=α⋅ΔX，取一阶微分得 d(αX)=α⋅dX。常数因子可以直接提出来，和标量情形一样自然。

3.2 转置

d(XT)=(dX)T

Intuition

转置和“取增量”可交换：(X+ΔX)T=XT+(ΔX)T。所以其微分为 d(XT)=(dX)T。你可以理解为：先给矩阵每个元素加一个小扰动，再转置；和先转置再加扰动，结果一样。

3.3 乘积法则

d(XY)=(dX)Y+X(dY)

Intuition

对乘积作增量展开：d(XY)=d((X+ΔX)(Y+ΔY))。展开得： (X+ΔX)(Y+ΔY)=XY+ΔXY+XΔY+ΔXΔY 其中二阶小量 ΔXΔY 忽略，于是 d(XY)=(dX)Y+X(dY)。这和 (fg)′=f′g+fg′ 是同一个灵魂。

更一般地，若 A,B 为常矩阵，则：

d(AXB)=A(dX)B

Intuition

因为 A,B 不变，故 d(AXB)=A(dX)B。常数矩阵像两扇大门，中间的变量矩阵怎么变，两边的大门只是把这个变化“框住”而已。

3.4 逆矩阵

若 X 可逆，则：

d(X−1)=−X−1(dX)X−1

Intuition

这个公式长得有点像标量的 (1/x)′=−1/x2，只是矩阵世界里左右各有一个 X−1。推导也很优雅：由恒等式 XX−1=I 两边求微分，再用乘积法则解出 d(X−1)。

推导：由恒等式 XX−1=I 两边求微分：

d(XX−1)=dI=0

由乘积法则：

(dX)X−1+Xd(X−1)=0

左乘 X−1：

X−1(dX)X−1+d(X−1)=0

所以 d(X−1)=−X−1(dX)X−1。

Trap

注意这里的顺序不能乱！矩阵乘法不交换，所以 X−1 分别出现在 dX 的左边和右边，不是 −(X−1dX)2 那种形式。

3.5 行列式

若 X 可逆，则：

d(detX)=det(X)tr(X−1dX)

Intuition

行列式描述的是矩阵对空间的“体积缩放率”。当你给矩阵一个微小扰动 dX 时，体积的变化率正比于当前体积 det(X) 乘上 trace 项。trace 项可以看作是在所有方向上扰动的“平均相对变化”。

推导：先利用 det(X)=exp(logdetX)，对两边求微分：

d(detX)=exp(logdetX)d(logdetX)=det(X)d(logdetX)

而后面将证明 d(logdetX)=tr(X−1dX)，代入即得：

d(detX)=det(X)tr(X−1dX)

于是为了读梯度，把它写成标准 trace 形式：

d(detX)=det(X)tr(X−1dX)=det(X)tr((X−T)TdX)

因此：

∂X∂det(X)​=det(X)X−T

Example

这个公式在概率图模型和变分推断里很常见。比如在多元高斯分布中，协方差矩阵的行列式出现在归一化系数里，我们常常需要对它求导来优化参数。

3.6 对数行列式

若 X 可逆，则：

d(logdetX)=tr(X−1dX)

Intuition

取对数之后，行列式的复杂乘法结构变成了加法结构，求导结果也异常简洁——只和 X−1 有关。这是对数带给我们的礼物：把乘法变加法，把指数变乘法。

推导：取微扰 X+dX，则：

logdet(X+dX)=logdet(X(I+X−1dX))

利用 det(AB)=det(A)det(B)，有：

logdet(X+dX)=logdet(X)+logdet(I+X−1dX)

对小矩阵 E，有一阶展开：

logdet(I+E)=tr(E)+o(∥E∥)

令 E=X−1dX，则：

d(logdetX)=tr(X−1dX)

再写成标准形式：

tr(X−1dX)=tr((X−T)TdX)

所以：

∂X∂logdet(X)​=X−T

Example

在最大似然估计协方差矩阵时，对数行列式项经常出现。它美妙的求导结果 X−T 让很多推导变得异常清爽。

3.7 Trace 的基本性质

Trace 的微分

d(tr(X))=tr(dX)

Intuition

trace 本身是线性的 tr(X+ΔX)=tr(X)+tr(ΔX)，因此 d(tr(X))=tr(dX)。它就像求和符号 ∑ 一样，微分和求和可以交换顺序。

循环不变性

tr(ABC)=tr(BCA)=tr(CAB)

Intuition

这是矩阵求导里最常用、最省事的技巧。不管矩阵怎么围成一圈，trace 只关心“首尾相接”后的对角线和。所以我们可以像转方向盘一样把矩阵因子转来转去，直到凑出我们想要的 dX 在最右边的形式。

推导：由 trace 的定义，

tr(MN)=i∑​(MN)ii​=i,j∑​Mij​Nji​=j,i∑​Nji​Mij​=tr(NM)

所以：

tr(ABC)=tr((AB)C)=tr(CAB)

继续轮换即可得到全部结果。

4. 向量求导常用公式

设 x,y​,a,b 为向量，A,C 为常矩阵。

4.1 一次型

公式 1

∂x∂(aTx)​=a

Intuition

这告诉你：如果 f 是 x 各分量的线性组合，那么每个分量前面的系数就是对应的偏导数。非常直白，和一维里 (ax)′=a 一模一样。

推导：设 f(x)=aTx，则 df=aTdx。与定义 df=(∇x​f)Tdx 比较可得 ∇x​f=a，即 ∂x∂(aTx)​=a。

公式 2

∂x∂(xTa)​=a

Note

因为标量满足 xTa=aTx，所以直接由上式可得 ∂x∂(xTa)​=a。也可直接求微分：d(xTa)=(dx)Ta=aTdx，故梯度仍为 a。

4.2 双线性型

对 x 求导

∂x∂(xTAy​)​=Ay​

Intuition

把 y​ 看成常量，Ay​ 就变成了一个常向量。那么 xTAy​ 对 x 来说就是一个一次型，梯度自然就是 Ay​。

推导：设 f(x)=xTAy​。把 y​ 看成常量，则 df=(dx)TAy​。又因为这是标量，可写成：

Note

对标量用转置来完成特定形式的构造很常见。

df=(Ay​)Tdx

所以 ∇x​f=Ay​。

对 y​ 求导

∂y​∂(xTAy​)​=ATx

Intuition

这次把 x 看成常量。xTA 是一个行向量，而梯度需要是列向量，所以转置一下得到 ATx。

推导：这次把 x 看成常量：df=xTAdy​。又有 df=xTAdy​=(ATx)Tdy​，因此 ∇y​​f=ATx。

4.3 二次型

一般情形

∂x∂(xTAx)​=(A+AT)x

Intuition

二次型里 x 出现了两次，一次在左边、一次在右边。根据乘积法则，左边那个 x 对梯度贡献 Ax，右边那个贡献 ATx。所以总梯度是两者相加。

推导：设 f(x)=xTAx，求微分：

df=d(xTAx)=(dx)TAx+xTAdx

第二项改写为 xTAdx=(ATx)Tdx。第一项改写为 (dx)TAx=(Ax)Tdx。

因此：

df=((Ax)+(ATx))Tdx

所以 ∇x​f=(A+AT)x。

对称情形

若 A 对称，则：

∂x∂(xTAx)​=2Ax

Intuition

若 AT=A，则 (A+AT)x=2Ax。这就是你在机器学习里最常见到的形式，比如岭回归、SVM 里的二次项求导。

推导：若 AT=A，则 (A+AT)x=2Ax。

半二次型

∂x∂(21​xTAx)​=21​(A+AT)x

Note

直接从上一式乘上 21​ 得 ∇x​(21​xTAx)=21​(A+AT)x。若 A 对称，则 ∂x∂(21​xTAx)​=Ax。

4.4 欧氏范数

范数平方

∥x∥22​=xTx

于是：

∂x∂(xTx)​=2x

Intuition

这是二次型中特例，对应 A=I。xTx=xTIx，所以 ∂x∂(xTx)​=(I+IT)x=2x。这个结果在梯度下降里天天见——目标函数里的 L2​ 正则项求导就长这样。

半范数平方

∂x∂(21​∥x∥22​)​=x

Note

由上一式直接乘以 21​：∂x∂(21​xTx)​=21​⋅2x=x。

4.5 最小二乘

设 f(x)=∥Ax−b∥22​，则：

∂x∂f​=2AT(Ax−b)

Intuition

最小二乘是机器学习的基石。这个公式告诉你：残差 Ax−b 越大，梯度就越大；而 AT 则是把输出空间的残差“投影”回输入空间 x 的方向。

推导：记 r=Ax−b，则 f=rTr。先求微分：df=2rTdr。

而 dr=d(Ax−b)=Adx，所以 df=2rTAdx。改写为 df=2(ATr)Tdx。

故 ∇x​f=2ATr=2AT(Ax−b)。

若写成 f(x)=21​∥Ax−b∥22​，则：

∂x∂f​=AT(Ax−b)

Note

只是在上式前乘了 21​，故系数 2 抵消。这就是为什么优化问题里很多人喜欢写 21​ 在前面——求导后表达式更干净。

4.6 仿射变换后的二次型

设 f(x)=(Ax+b)TC(Ax+b)，则：

∂x∂f​=AT(C+CT)(Ax+b)

Intuition

这可以看作“先做一次线性变换，再算二次型，最后求导”。链式法则在这里自然而然地出现了：外层对中间变量的梯度乘上中间变量对 x 的 Jacobian。

推导：令 z=Ax+b，则 f=zTCz。由二次型公式，对 z 有 ∂z∂f​=(C+CT)z。又因为 dz=Adx，即 Jacobian 为 A，由链式法则：

∂x∂f​=AT∂z∂f​=AT(C+CT)z

代回 z 得：

∂x∂f​=AT(C+CT)(Ax+b)

若 C 对称，则化为：

∂x∂f​=2ATC(Ax+b)

5. 矩阵求导常用公式

设 A,B,C 为常矩阵，X 为矩阵变量。

5.1 Frobenius 内积

⟨A,X⟩=tr(ATX)

因此：

∂X∂​tr(ATX)=A

Intuition

矩阵的 Frobenius 内积就是 trace 形式。这个公式说明：如果 f 是 X 各元素的线性组合（系数由 A 给出），那么梯度就是系数矩阵 A 本身。

推导：设 f(X)=tr(ATX)，则 df=tr(ATdX)。这已经是标准形式：

df=tr((∂X∂f​)TdX)

比较可得 ∂X∂f​=A。

5.2 Trace 线性项

公式 1

∂X∂​tr(AX)=AT

Trap

这是最常见的出错点之一！结果不是 A，而是 AT。原因就在于矩阵梯度的定义里带了一个转置：df=tr((∂X∂f​)TdX)。所以你要把 tr(AdX) 改写成 tr((AT)TdX)，梯度才是 AT。

推导：设 f(X)=tr(AX)，则 df=tr(AdX)。把它写成标准形式：tr(AdX)=tr((AT)TdX)。所以 ∂X∂f​=AT。

公式 2

∂X∂​tr(XA)=AT

Note

由循环不变性，tr(XA)=tr(AX)，故直接由上一式可得结果。

公式 3

∂X∂​tr(AXB)=ATBT

Intuition

这个结果很对称：对 X 求导后，左右两个常数矩阵分别转置，然后按原来的左右顺序摆好。

推导：设 f(X)=tr(AXB)。求微分：df=tr(AdXB)。利用循环不变性，把 dX 放到最后：df=tr(BAdX)。再写成标准形式：

tr(BAdX)=tr((ATBT)TdX)

因此 ∂X∂f​=ATBT。

公式 4

∂X∂​tr(XTA)=A

Note

设 f(X)=tr(XTA)，则 df=tr(dXTA)。利用 tr(MT)=tr(M)，得 tr(dXTA)=tr(ATdX)，故 ∂X∂f​=A。

5.3 向量-矩阵-向量型

f(X)=aTXb

则：

∂X∂f​=abT

Intuition

这个公式非常漂亮：一个标量 f 对矩阵 X 的梯度，竟然是一个外积矩阵 abT。每个位置 (i,j) 的偏导数就是 ai​bj​，而这正好是 abT 的 (i,j) 元素。

推导：先把它写成 trace 形式：f=aTXb=tr(aTXb)。由循环不变性，f=tr(baTX)。于是 df=tr(baTdX)。写成标准形式：

tr(baTdX)=tr((abT)TdX)

因此 ∂X∂f​=abT。

5.4 二次型 Trace

公式 1

f(X)=tr(XTAX)

则：

∂X∂f​=AX+ATX

Intuition

和向量二次型一样，X 在这里也出现了两次。一次在左边贡献 AX，一次在右边贡献 ATX。两者相加就是总梯度。

推导：求微分：

df=tr(dXTAX)+tr(XTAdX)

第一项：tr(dXTAX)=tr((AX)TdX)。

第二项：tr(XTAdX)=tr((ATX)TdX)。

因此：

df=tr((AX+ATX)TdX)

若 A 对称，则化为 ∂X∂f​=2AX。

公式 2

f(X)=tr(XTAXB)

则：

∂X∂f​=AXB+ATXBT

Intuition

这个公式可以看作是上一个公式的“加权版”。右边的 B 像是给第二项加了一个权重矩阵。同样地，左右各贡献一项，但每一项都多了和 B 的乘法。

推导：求微分：

df=tr(dXTAXB)+tr(XTAdXB)

第一项：tr(dXTAXB)=tr((AXB)TdX)。

第二项先循环移动：tr(XTAdXB)=tr(BXTAdX)，再写成标准形式：tr(BXTAdX)=tr((ATXBT)TdX)。

因此：

df=tr((AXB+ATXBT)TdX)

5.5 Frobenius 范数

∥X∥F2​=tr(XTX)

因此：

∂X∂​∥X∥F2​=2X

Intuition

Frobenius 范数就是矩阵元素的平方和开根号。它的平方对 X 求导，结果是 2X——和向量里 ∥x∥22​ 求导得 2x 完全平行。

推导：这是上面公式的特例，对应 A=I：tr(XTX)=tr(XTIX)。故 ∂X∂​tr(XTX)=IX+ITX=2X。

∂X∂​(21​∥X∥F2​)=X

Note

前式乘 21​ 即得。这也是矩阵优化里（比如矩阵补全、低秩近似）最常见的正则项求导结果。

5.6 矩阵最小二乘

公式 1

f(X)=∥AX−B∥F2​

则：

∂X∂f​=2AT(AX−B)

Intuition

这和向量最小二乘 2AT(Ax−b) 长得几乎一样，只是把向量 b 换成了矩阵 B。本质上，矩阵最小二乘可以看作是很多个向量最小二乘并排站在一起。

推导：令 R=AX−B，则 f=tr(RTR)。由 Frobenius 范数求导公式，对 R 有 ∂R∂f​=2R。又因为 dR=AdX，所以由矩阵链式法则：

∂X∂f​=AT∂R∂f​=2ATR=2AT(AX−B)

也可以直接微分：

df=2tr(RTdR)=2tr(RTAdX)=2tr((ATR)TdX)

若写成 f(X)=21​∥AX−B∥F2​，则：

∂X∂f​=AT(AX−B)

公式 2

f(X)=∥XA−B∥F2​

则：

∂X∂f​=2(XA−B)AT

Intuition

注意这里 X 在左边乘 A，所以求导后 AT 出现在右边。这和公式 1 形成对称：谁在中间不动，谁的转置就跳到另一边。

推导：令 R=XA−B，则 dR=dXA。又 df=2tr(RTdR)=2tr(RTdXA)。循环移动：df=2tr(ARTdX)。写成标准形式：ART=(RAT)T。故 df=2tr((RAT)TdX)，所以 ∂X∂f​=2RAT=2(XA−B)AT。

公式 3

f(X)=∥AXB−C∥F2​

则：

∂X∂f​=2AT(AXB−C)BT

Intuition

这是公式 1 和公式 2 的“合体版”。X 左边有 A、右边有 B，求导后 AT 跳到左边，BT 跳到右边，而残差 AXB−C 留在中间。

推导：令 R=AXB−C，则 dR=AdXB。又 df=2tr(RTdR)=2tr(RTAdXB)。循环移动：df=2tr(BRTAdX)。写成标准形式：BRTA=(ATRBT)T。所以 df=2tr((ATRBT)TdX)。故 ∂X∂f​=2ATRBT=2AT(AXB−C)BT。

若写成半平方形式，则：

∂X∂​(21​∥AXB−C∥F2​)=AT(AXB−C)BT

5.7 逆矩阵相关

f(X)=tr(X−1A)

则：

∂X∂f​=−X−TATX−T

Intuition

这个结果看起来复杂，但结构很清晰：负号来自逆矩阵微分的负号，而两个 X−T 分别出现在 AT 的左右两侧。你可以把它和标量 (1/x)′=−1/x2 类比，只是矩阵版本更“对称”一些。

推导：先求微分：df=tr(d(X−1)A)。由逆矩阵微分公式 d(X−1)=−X−1(dX)X−1。代入得 df=−tr(X−1(dX)X−1A)。循环移动：df=−tr(X−1AX−1dX)。再写成标准形式：X−1AX−1=(X−TATX−T)T。所以 df=−tr((X−TATX−T)TdX)。于是得出结果。

5.8 行列式与对数行列式

这两条前面已经推过，这里直接汇总：

∂X∂det(X)​=det(X)X−T ∂X∂logdet(X)​=X−T

Example

这两个公式在协方差矩阵估计、图模型、度量学习等领域反复出现。logdet 的简洁结果尤其受欢迎，因为它让复杂的行列式优化变得像标量求导一样优雅。

6. 链式法则

6.1 向量形式

若 y​=y​(x),f=f(y​)

记 Jacobian 为 J=∂x∂y​​，则：

∇x​f=JT∇y​​f

Intuition

这是多元微积分里的链式法则。你可以理解为：函数 f 对 x 的敏感度，等于它对中间变量 y​ 的敏感度，再乘上 y​ 对 x 的敏感度。转置的出现是因为我们采用列向量梯度的约定。

推导：由微分链式法则，dy​=Jdx。又因为 df=(∇y​​f)Tdy​，代入得 df=(∇y​​f)TJdx。写成关于 dx 的标准形式：df=(JT∇y​​f)Tdx。所以 ∇x​f=JT∇y​​f。

特别地，若 y​=Ax+b，则：

∇x​f=AT∇y​​f

Note

因为此时 Jacobian 就是 J=A，直接代入上一式即可。神经网络里每一层的反向传播本质上都在反复用这个公式。

6.2 矩阵线性变换

若 Y=AXB 且 f=f(Y) 为标量函数，则：

∂X∂f​=AT∂Y∂f​BT

Intuition

这是向量链式法则在矩阵世界里的推广。如果 X 通过一个“三明治”变换 AXB 变成 Y，那么梯度也会以对称的“三明治”形式回传，只是左右面包片都转置了。

推导：先有 dY=AdXB。又 df=tr((∂Y∂f​)TdY)。代入得 df=tr((∂Y∂f​)TAdXB)。循环移动：df=tr(B(∂Y∂f​)TAdX)。改写成标准形式：

B(∂Y∂f​)TA=(AT∂Y∂f​BT)T

因此 ∂X∂f​=AT∂Y∂f​BT。

6.3 逐元素非线性

若 Y=ϕ(X)，其中 ϕ 逐元素作用，则：

dY=ϕ′(X)∘dX

Intuition

逐元素非线性（比如 ReLU、sigmoid）是最简单的变换：每个位置独立地通过一个标量函数。所以微分也是逐元素乘上局部导数，用 Hadamard 乘积 ∘ 表示。

推导：逐元素地看：Yij​=ϕ(Xij​)，故 dYij​=ϕ′(Xij​)dXij​。把所有元素拼回矩阵形式，就是 dY=ϕ′(X)∘dX，其中 ∘ 表示 Hadamard 乘积。

若 f=f(Y) 为标量函数，则：

∂X∂f​=∂Y∂f​∘ϕ′(X)

Intuition

这也很直观：X 每个位置的变化对 f 的影响，等于该位置通过 ϕ 后对 f 的影响，再乘上 ϕ 在该点的局部斜率。神经网络反向传播里的“梯度逐元素相乘”就是这个公式在行动。

推导：按元素写：df=∑i,j​∂Yij​∂f​dYij​。代入 dYij​=ϕ′(Xij​)dXij​ 得 df=∑i,j​∂Yij​∂f​ϕ′(Xij​)dXij​。因此对应元素上的梯度为 ∂Xij​∂f​=∂Yij​∂f​ϕ′(Xij​)。拼成矩阵即得上述结果。向量情形同理：

∇x​f=∇y​​f∘ϕ′(x)

7. 常见易错点（附解释）

7.1 二次型不一定直接等于 2Ax

一般有：

∂x∂(xTAx)​=(A+AT)x

只有当 A 对称时，才有：

∂x∂(xTAx)​=2Ax

Trap

左边那个 x 和右边那个 x 各会贡献一项，分别对应 Ax 与 ATx，所以一般是两项相加，不是直接乘 2。初学者最容易在这里省略转置，结果导致推导全错。

7.2 Trace 求导常常要转置

∂X∂​tr(AX)=AT

不是 A。

Trap

因为矩阵梯度的定义是 df=tr((∂X∂f​)TdX)，所以你要把结果凑成某个矩阵的转置乘 dX 的形式。而 tr(AdX)=tr((AT)TdX)，所以梯度是 AT。

7.3 半平方形式更整洁

优化里常写：

21​∥Ax−b∥22​或21​∥AX−B∥F2​

Note

原因：求导后正好把前面的 2 消掉，使表达式更简洁：

∂x∂​(21​∥Ax−b∥22​)=AT(Ax−b)

Example

在梯度下降实现里，写 21​ 不仅是为了数学上好看，更是让代码里的梯度表达式更干净，少一个系数 2 就少一次出错的机会。

7.4 对数行列式要求可逆

∂X∂logdet(X)​=X−T

默认要求 X 可逆，而且通常还要求 det(X)>0。

Trap

因为要定义 logdet(X)，至少需要 det(X) 非零；若在实数范围内讨论 log，通常还要求正值。在实际优化中，我们常常通过对 X 参数化（比如 Cholesky 分解）来保证这些条件自动满足。
