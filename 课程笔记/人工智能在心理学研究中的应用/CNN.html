CNN

CNN

Apr 26, 202615 min read

核心思想速览

为什么需要卷积神经网络（CNN）？ > 传统的全连接网络（MLP）在处理图像时极其笨重。一张普通的 512×512×3 彩色图片，如果展平成一维向量，长度会高达 786,432。这不仅会导致参数量爆炸，还会彻底破坏图像中极其重要的二维空间结构。

CNN 的破局点在于利用了自然图像的两大先验假设：局部性（Locality） 和 平移不变性（Translation Invariance）。它通过“局部感受野”和“参数共享”两大机制，不仅极大地减少了参数，还完美模拟了人类视觉系统“从局部边缘到整体轮廓”的认知过程。

1. 灵感来源：生物视觉系统与早期探索

CNN 并不是纯数学推导出来的，它有着极强的神经生物学背景。

1.1 视觉系统的层级结构 (Visual System in the Brain)

人类大脑处理视觉信息是一个层级递进的过程。信号从视网膜出发，经过外侧膝状体（LGN），逐步向后脑的视觉皮层传递，特征变得越来越复杂，感受野（神经元能看到的视野范围）也越来越大：

graph LR
    A[视网膜] --> B[LGN]
    B --> C("V1: 提取边缘和线条<br>Edges & Lines")
    C --> D("V2: 提取简单形状<br>Shapes")
    D --> E("V4: 提取复杂部件<br>Objects")
    E --> F("IT: 识别人脸和复杂物体<br>Faces")

1.2 简单细胞与复杂细胞 (Simple and Complex Cells)

Hubel 和 Wiesel 在 1962 年的经典实验（诺贝尔奖级工作）中发现了视觉皮层中的两类重要细胞，它们直接启发了 CNN 的核心机制：

  • 简单细胞 (Simple Cell)：它的感受野具有特定的方向性，比如只对“水平方向的边缘”有强烈反应。这启发了 CNN 中的卷积层（Convolution）。

  • 复杂细胞 (Complex Cell)：它连接着多个感受野相邻、偏好方向相同的简单细胞。不管刺激出现在局部视野的哪个特定位置，它都能产生响应。这提供了早期的“平移不变性”，直接启发了 CNN 中的池化层（Pooling）。

1.3 从 Neocognitron 到 AlexNet 的演进

  • Neocognitron (Kunihiko Fukushima, 1980)：第一次提出了多特征图（Multiple feature maps）、权重共享（Weights sharing）和类似 ReLU 的非线性激活，为现代 CNN 奠定了基础。

  • LeNet-5 (Yann LeCun, 1989)：确立了 Conv -> Pool -> Conv -> Pool -> FC 的经典架构，成功应用于手写数字（MNIST）识别。

  • AlexNet (Alex Krizhevsky, 2012)：在 ImageNet 挑战赛上以巨大优势击败传统方法（错误率从 26.2% 降到 16.4%），开启了深度学习的狂飙时代。它证明了更深的网络 + 更多的算力 + 海量数据可以产生奇迹。

2. 卷积层 (Convolution Layer)：CNN 的灵魂

卷积层是 CNN 提取空间特征的核心。与全连接网络中“每个神经元都要看全图”不同，卷积层基于两个极其重要的物理直觉：

  1. 局部性 (Locality)：图像中距离近的像素相关性很强（比如构成一片树叶），而距离远的像素一般没什么直接关系。

  2. 平移不变性 (Translation Invariance) 与 自相似性 (Composition)：同一个有意义的图案（比如一双眼睛、一条蕨类植物的边缘）可能出现在图像的任何位置。

基于这两点，卷积层引入了三大机制：

2.1 局部计算 (Local Computation)

在全连接网络中，输出 yi​ 依赖于所有输入 x：

yi​=j∈image∑​wj​xj​+bi​

而在卷积网络中，神经元只连接输入图像的一个小局部（例如 3×3 的窗口），这个窗口被称为感受野 (Receptive Field)：

yi​=j∈3×3∑​wj​xj​+b

直觉解释：就像是你拿着一个手电筒（Kernel / Filter）在巨大的地图上一点点扫过，每次只看手电筒照亮的一小块区域，而不是一眼看全图。

2.2 参数共享 (Parameter Sharing)

手电筒不仅光圈小，而且在滑动扫描全图时，使用的是完全相同的权重参数。

这就是为什么 CNN 能大幅降低参数量的原因，也是它为什么能做到“平移不变”的原因：无论猫出现在左上角还是右下角，同一个“找猫耳”的卷积核扫到那里时，都能输出高响应。

2.3 多特征图与层级抽象 (Multiple Maps & Hierarchical Layers)

  • 单层多通道：一个卷积核只能提取一种特征（比如横向边缘）。为了提取丰富的特征，我们会使用一组（一个 Bank）卷积核。每一个卷积核扫描全图后，会生成一个 2D 的切片，多个卷积核的切片叠在一起，就形成了输出特征图 (Output Feature Maps)。

  • 深层抽象：底层卷积提取简单的“边缘、纹理”；高层卷积通过组合底层特征，提取出“对象的部件”（如车轮、眼睛）；最终层级抽象出“整体对象”（如汽车、人脸）。

经典手工卷积核示例：

在深度学习之前，人们通常手动设计卷积核。例如水平边缘检测（Sobel Filter）：

​+1+2+1​000​−1−2−1​​

还有用于锐化的 Filter、用于高斯平滑的 Filter 等。现代 CNN 则是让网络自己去学习这些矩阵里的数值。

3. 非线性激活函数 (Nonlinear Activation)

如果只做卷积（本质是线性矩阵乘法叠加），不管叠多少层，整个网络最终也只等价于一个线性模型，无法拟合复杂的现实数据。因此，必须在每次卷积后加入非线性激活。

核心主角：ReLU (Rectified Linear Unit)

ReLU 几乎是现今所有 CNN 的默认选择。它的操作非常简单：把所有负数归零，保留所有正数。

f(x)=max(0,x)

直觉解释：特征图上的数值代表了“当前局部区域存在某种特征的强烈程度”。如果是负数，说明根本没有这个特征，直接掐断神经元信号（输出 0）；如果是正数，则特征越强信号越大。它不仅计算极快，还能有效缓解深层网络的梯度消失问题。

其他激活函数对比：

  • Sigmoid / Tanh：早期常用，但存在严重的饱和区域（输入太大或太小时梯度几乎为 0），会导致深层网络梯度消失。

  • Leaky ReLU：max(0.1x,x)，为了解决 ReLU 在负半轴直接“死掉”（梯度恒为 0）的问题，给负数分配一个极小的斜率。

  • Maxout / ELU：其他改良版本，各有优劣，但综合计算成本，普通 ReLU 依然是最能打的。

4. 池化 (Pooling) 与填充 (Padding)

4.1 池化层：宏观视角的“虚化”

池化操作（通常是 Max-pooling，取窗口内最大值）相当于对特征图进行降采样。例如用 2×2 的窗口、步长为 2 进行池化，长宽都会缩小一半。

为什么要池化？

  1. 获得微小平移不变性：只要特征在这个小窗口里出现过，最大值就能把它揪出来。哪怕特征因为形变稍微便宜了几个像素，输出也不受影响（完美契合复杂细胞的功能）。

  2. 减少参数量：降低分辨率，减轻后续计算压力。

  3. 扩大感受野：在缩小后的特征图上再做 3×3 卷积，实际上相当于在原始图片上看了更大的区域。

4.2 填充操作 (Padding)：保护边缘与控制尺寸

如果不加干预，图像经过卷积后，尺寸会越来越小，而且边缘像素由于处于角落，被卷积核扫到的次数远少于中心像素，导致边缘信息丢失。

因此，我们会在原图周围补上一圈 0（Zero-padding）。

计算输出尺寸的黄金公式：

设输入尺寸为 n，卷积核大小为 k，总填充量为 p（如果是单边填充则是 2×pad），步长为 s，则输出尺寸 O 为：

O=⌊sn−k+p​⌋+1

三种常见的 Padding 策略：

  1. Valid Padding (p=0)：完全不填充。输出尺寸必然缩小。

  2. Same Padding：让输出尺寸等于输入尺寸（假设步长 s=1）。此时总填充量要求 p=k−1。

  3. Full Padding (p=(k−1)×2)：尽量保留所有的边界效应。

5. CNN 经典工作流与工程黑魔法

5.1 典型的 CNN 架构

我们通常会将多组 [Conv -> ReLU -> Pool] 结构堆叠，最后将特征图展平（Flatten），接上全连接层（FC）完成最终的分类输出。

graph TD
    A[输入图片 32x32] --> B("Conv: 提取底层特征")
    B --> C("ReLU: 非线性激活")
    C --> D("Pool: 降维与平移不变")
    D --> E(...)
    E --> F(Conv + ReLU)
    F --> G(Pool)
    G --> H[展平 Flatten]
    H --> I("FC: 全连接层整合特征")
    I --> J("Softmax 输出类别概率: 车/船/马...")

5.2 权重初始化：Xavier Initialization

直觉痛点：深度网络如果随便乱给初始权重，比如权重太大，经过几十层乘下来，数值会爆炸；如果权重太小，信号传两层就衰减到 0 了（梯度消失）。

为了让信号在整个网络中稳定传播，我们需要保证输入和输出的方差一致。

Xavier 初始化机制：

从高斯分布或均匀分布中随机抽样，但其方差必须与该层输入输出的神经元数量（nj 和 nj+1）挂钩：

W∼Normal(0,nj+nj+12​)

或者更简化的形式：

W∼Normal(0,nj1​)

这样能有效避免深层网络在训练初期就崩盘。

5.3 批归一化 (Batch Normalization, BN)

这是深度学习中极为重要的“护城河”技术，用于缓解深层网络内部的数据分布偏移。

计算步骤（对一个 mini-batch 内的数据）：

  1. 算均值：μB​←m1​∑xi​

  2. 算方差：σB2​←m1​∑(xi​−μB​)2

  3. 标准化：让数据变成均值为 0，方差为 1 的分布：

x^i​←σB2​+ϵ​xi​−μB​​
  1. 缩放与平移 (Scale and Shift)：引入可学习参数 γ 和 β：
yi​←γx^i​+β 
> **为什么要加最后这一步？** 因为强制标准化可能会破坏前面网络好不容易学到的特征分布。加上可学习的 $\gamma$ 和 $\beta$，网络就有了退路：如果它觉得不归一化更好，它完全可以学出 $\gamma = \sigma_B, \beta = \mu_B$ 来抵消掉前面的操作。

BN 的巨大威力：

  1. 极大降低对初始化的敏感度：就算一开始权重给得不太好，BN 也能把信号强行拉回合理的区间。

  2. 加速收敛：可以使用更大的学习率。

  3. 自带正则化效果：因为计算均值和方差是基于 mini-batch 的，引入了轻微的随机噪声，这在一定程度上减少了对 Dropout 等传统正则化手段的依赖。
