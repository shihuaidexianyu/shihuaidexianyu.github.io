第二讲 物理层

Summary

这一讲咱们要搞懂比特是怎么变成电信号/光信号在物理世界里跑起来的。学完这章，你能说清楚带宽和噪声怎么限制传输速度，能区分各种编码和复用的优劣，还能在面试里把奈奎斯特和香农的边界聊得头头是道。

1. 物理层的职责与接口特性

物理层的任务是在物理介质上可靠地传输比特流。它关心的不是某一种具体传输介质本身，而是如何在各种传输媒体与通信手段存在差异的条件下，向上层提供一致的 ” 比特传输能力 “。因此，物理层的核心问题是：如何定义接口，如何刻画信号，如何使发送端产生可传输的物理信号，并使接收端能够正确恢复出比特序列。

从 OSI 的角度看，物理层需要规定接口的四类特性：

  1. 机械特性：接线器的形状、尺寸、引脚数、排列方式以及锁定装置。
  2. 电气特性：各条线上允许出现的电压范围与电信号约束。
  3. 功能特性：某条线上某一电平或信号的含义。
  4. 过程特性：不同功能事件出现的顺序与时序关系。

物理层向上层提供的不是 ” 报文 "" 帧 ” 或 ” 分组 “，而是发送与接收比特流的能力。连接的建立、维持与释放，也是在这一层的能力边界内讨论的。

flowchart LR
    A[上层数据] --> B[物理层接口约定]
    B --> C[信号形成]
    C --> D[传输介质]
    D --> E[信号接收与恢复]
    E --> F[比特流输出]

    B1[机械特性]
    B2[电气特性]
    B3[功能特性]
    B4[过程特性]

    B --- B1
    B --- B2
    B --- B3
    B --- B4

Intuition

你可以把物理层想象成快递的打包和运输环节。上层只管把东西（比特）交给你，至于你是用顺丰、中通还是无人机，箱子多大、贴什么标签、先贴面单还是先封箱——这些规矩就是物理层的四类特性。只要两边遵守同一套规矩，东西就能从 A 点搬到 B 点。

2. 数据通信的原理

2.1 数据通信系统的基本模型

一个数据通信系统可以抽象为三部分：源系统、传输系统和目的系统。源系统负责产生数据并将其转换为适合传输的信号；传输系统负责在信道中搬运信号；目的系统负责从接收信号中恢复数据。

在更抽象的表述中，通信系统可以写成：

信源→发送器→信道→接收器→信宿

其中噪声源会在信道中对传输信号产生影响。

flowchart LR
    S[信源 / Source] --> T[发送器 / Transmitter]
    T --> C[信道 / Channel]
    N[噪声源 / Noise] -.-> C
    C --> R[接收器 / Receiver]
    R --> D[信宿 / Destination]

Example

你打电话的时候，你的嘴是信源，手机麦克风+基带芯片是发送器，无线电波穿过的空气是信道，周围其他电子设备的干扰就是噪声源，对方手机是接收器，对方的耳朵是信宿。整个链条里任何一环掉链子，通话质量都会崩。

2.2 数据、信号与码元

数据是承载信息的实体，如比特序列。
信号是数据的电气或电磁表现，是信息在物理世界中的可传输形式。

按照连续性不同，信号可以分为两类：

  • 模拟信号：幅值连续，时间通常也连续。
  • 数字信号：幅值离散，时间上通常按离散间隔变化。

在时域中表示数字信号时，表示不同离散值的基本波形称为码元或符号。一个码元不一定只表示 1 个比特；若一个码元有 V 种不同状态，则它最多可以表示 log2​V 个比特。

Trap

模拟信号和数字信号的区别不在信息本身，而在表示方式。同一段语音既可以走模拟电路（幅值连续），也可以走数字电路（采样量化后离散）。考试常问”数字信号抗干扰更好”，本质上是因为数字信号只有有限几个判决电平，轻微失真不会导致误判，而模拟信号的每一点失真都会直接体现在信号里。

Note

码元是个容易轻视的概念，但它直接决定了后面奈奎斯特公式的理解。一个码元就像一盒棋子，盒子里有 V 种颜色，那这一盒最多能装 log2​V 比特的信息。QAM-64 的星座点有 64 种状态，所以一个码元装 6 比特。

2.3 信道与通信方式

信道可以理解为沿某一方向传送信息的媒体，其实现通常由传输设备 + 传输介质共同构成。

按通信方向划分，有三种基本方式：

  • 单工：只能单向传输。
  • 半双工：双方都能发送，但不能同时发送。
  • 全双工：双方可以同时发送与接收。
flowchart TB
    A[通信方式]
    A --> B[单工]
    A --> C[半双工]
    A --> D[全双工]

    B --> B1[仅 A→B]
    C --> C1[A↔B 交替]
    D --> D1[A↔B 同时]

Example

单工像广播，你只能听不能回话；半双工像对讲机，按住说话松手收听，咱俩不能同时吼；全双工像日常打电话，你骂你的，我骂我的，两边同时进行。很多面试题会拿”网线是不是全双工”来考，记住：现代以太网的双绞线通常是全双工，因为发送和接收各用一对线，物理上互不干扰。

2.4 物理层中的关键参数

物理层分析经常涉及以下参数：

  • 信号功率 S
  • 噪声功率 N
  • 信道带宽 B
  • 信噪比 S/N
  • 数据率：单位时间传输的比特数，单位为 bit/s
  • 误码率：传输错误概率

信噪比常用分贝表示：

SNRdB​=10log10​(NS​)

例如，10 dB 表示功率比约为 10，20 dB 表示约为 100，3 dB 表示约为 2。

Note

dB 的对数特性是个考试速算小技巧：功率翻倍 ≈ 3 dB，翻十倍 ≈ 10 dB。所以 30 dB 就是功率比 1000，不需要真的去按计算器。

2.5 信号、衰减与频谱

信号本质上是随时间变化的物理量。对于正弦载波，可写为：

s(t)=Asin(2πft+φ)

其中 A 为幅度，f 为频率，φ 为相位。信号经过信道时会发生衰减、失真与噪声叠加。若发送功率为 Ps​、接收功率为 Pr​，则衰减可用分贝表示为：

10log10​(Ps​Pr​​) dB

任意周期信号都可以分解为不同频率分量的叠加，因此既可以在时域中观察波形，也可以在频域中观察其频谱。频谱分析揭示了一个基本事实：信道能通过的频率范围决定了它能多忠实地传送原始波形。

Intuition

想象你在弹一首复杂的钢琴曲，琴键按得越快（高频），对音响的要求就越高。如果音箱只能放出低音，高音全被切掉了，你听到的曲子就会变钝、变糊。信道带宽就是那个音箱的频率范围，带宽越窄，高频越过不去，数字波形的边沿就越圆滑，越容易误判。

3. 通信理论

3.1 傅里叶分析与带宽限制

周期信号 g(t) 可以用傅里叶级数表示为若干谐波分量之和。其含义并不在于机械地记忆公式，而在于把 ” 复杂波形 ” 解释为 ” 多个频率分量的叠加 ”。

这一定义直接导出一个工程结论：若信道只能通过有限带宽，那么高频分量会被削弱或滤除，接收端看到的波形就会变钝、变形，严重时便无法正确判决码元。带宽越宽，能保留的谐波越多，波形越接近原始发送信号。

flowchart LR
    A[原始数字波形] --> B[频谱分解]
    B --> C[信道仅允许有限频率通过]
    C --> D[高频分量被削弱]
    D --> E[接收波形失真]
    E --> F[判决困难或产生误码]

Intuition

这就像是素描： bandwidth 越大，你能用的铅笔越细，线条越锐利；带宽小了，只能用粗蜡笔，边界全是圆的，0 和 1 的分界线就糊成一团了。

3.2 奈奎斯特定理：无噪声条件下的码元速率上限

在理想无噪声且带宽受限的信道中，为避免码间串扰，最大数据率满足：

Rmax​=2Blog2​V

其中：

  • B 为信道带宽；
  • V 为码元状态数；
  • 2B 对应最大符号率；
  • log2​V 对应每个符号能够携带的比特数。

这一公式说明，提高数据率有两条路径：

  1. 增加带宽 B；
  2. 增加每个码元可区分的状态数 V。

但第二条路径并非没有代价：状态数越多，接收判决越困难，对噪声和失真的容忍度越低。

Example

假设信道带宽 B=3 kHz，用二元调制（V=2），奈奎斯特极限是 2×3×1=6 kbit/s。如果改用 QAM-16（V=16），极限就变成了 2×3×4=24 kbit/s。但注意，这只是在”理想无噪声”下的纸上富贵，现实里噪声一上来，星座点一密，分分钟误判给你看。

Trap

奈奎斯特公式的前提条件是理想无噪声信道。很多同学习惯性看到带宽就套奈奎斯特，结果忽略了题目里可能已经给了信噪比——那道题真正要考的是香农公式。看清条件再下手！

3.3 香农公式：有噪声条件下的比特率上限

在带宽受限且存在高斯白噪声的条件下，信道极限容量为：

C=Blog2​(1+NS​)

这一定理说明，能否 ” 无差错 ” 传输最终受两类资源约束：

  • 频率资源：带宽 B
  • 能量资源：信噪比 S/N

香农公式中的 C 是理论极限，而不是任何系统都能轻易达到的实际速率。实际系统通常低于这一极限，但它给出了物理层设计的根本边界。

Example

还是刚才那条 3 kHz 的电话线，如果信噪比是 30 dB（即 S/N=1000），香农极限是 3000×log2​(1001)≈3000×9.97≈29.9 kbit/s。所以即便你把调制搞得天花乱坠，也不可能突破这个天花板。

Note

香农公式里那个”1+“别漏了。有些教材的近似推导会强调 S/N≫1 时近似为 Blog2​(S/N)，但标准写法永远是带那个”1+“。

3.4 奈奎斯特与香农的分工

奈奎斯特关注的是带宽限制导致的码间串扰问题，更接近 ” 符号变化能有多快 “；香农关注的是噪声存在时信息无差错传输的极限，更接近 ” 有效比特率能有多高 ”。

flowchart TB
    A[信道速率上限]
    A --> B[带宽限制]
    A --> C[噪声限制]

    B --> B1[奈奎斯特定理]
    B1 --> B2[最大符号率受限]
    B2 --> B3[避免码间串扰]

    C --> C1[香农公式]
    C1 --> C2[最大无差错比特率受限]
    C2 --> C3[受 S/N 与 B 共同影响]

Trap

面试高频题：“奈奎斯特和香农公式的区别是什么？“标准答案就是上图的分工——一个管”码元能变多快”（符号层面），一个管”比特能传多少”（信息层面）。如果题目同时给了带宽和信噪比，真正的上限是两者算出来的较小值。

3.5 同步：接收端为什么必须 ” 跟上发送端 ”

传输不仅要求 ” 信号到达 “，还要求接收端知道每一位的边界在哪里，这就是同步问题。

  • 异步传输：通常以字符为单位，字符之间间隔可变。每个字符带起始位、数据位、可选校验位和停止位。实现简单，但开销较大，适于低速传输。
  • 同步传输：发送与接收在位级保持同步，通常通过专门时钟线或从接收信号中提取时钟。帧更长，效率更高，更适于高速和长距离场景。
flowchart LR
    A[同步问题] --> B[异步传输]
    A --> C[同步传输]

    B --> B1[按字符发送]
    B --> B2[起始位/停止位]
    B --> B3[字符间隔可变]
    B --> B4[实现简单 适合低速]

    C --> C1[按位持续同步]
    C --> C2[从时钟线或信号中恢复时钟]
    C --> C3[长帧]
    C --> C4[效率高 适合高速]

Intuition

异步传输就像两个人跳 freestyle，每跳完一组动作喊一声”换”，中间休息多久无所谓；同步传输就像仪仗队阅兵，每个人的步子必须完全一致，通过喊口令或者听音乐来保持同步。异步开销大但容错高，同步效率高但对时钟要求严。

4. 编码调制

4.1 基带信号与调制的必要性

来自信源的原始数字信号称为基带信号。计算机输出的比特流本质上就是基带信号。基带信号往往包含低频甚至直流分量，而很多信道并不能直接传输这些成分；此外，在长距离场景中，基带信号容易衰减、畸变，且多个基带信号难以直接共享同一信道。

因此，需要对基带信号进行处理。按是否搬移频谱，可分为两类：

  • 基带调制：不搬移频谱，只改变基带波形以适应信道，又称编码。
  • 带通调制：利用载波把信号搬移到某一频段进行传输。
flowchart LR
    A[基带数据] --> B{是否搬移到载波频段}
    B -->|否| C[基带调制 / 编码]
    B -->|是| D[带通调制]

    C --> C1[输出仍是基带信号]
    D --> D1[输出为带通信号]

Intuition

基带信号就像一个人用方言在喊话，近处能听懂，远了就废了，而且不同人一起喊会互相干扰。调制就是给信号”换频道”——编码相当于把你的方言变成更清晰的发音方式（还在原地喊），带通调制则是把你搬到广播电台的频率上去发射，传得又远又不跟别人打架。

4.2 基带编码

4.2.1 NRZ

NRZ（Non-Return-to-Zero）用固定高低电平表示 0 和 1。它的实现简单，带宽利用率较高，但存在两个根本问题：

  1. 时钟恢复困难：若长时间连续发送相同比特，信号没有跳变，接收端难以提取位同步。
  2. 基线漂移：连续长串 0 或 1 会改变平均电平，使判决门限漂移。

对 NRZ 而言，若带宽为 B，则根据奈奎斯特思想，可认为数据率上限约为：

R≈2B

Example

假设你发送一长串 0（低电平），信号线在很长一段时间里就是一条平线。接收端盯着这条线，根本分不清你发了 10 个 0 还是 11 个 0——因为它不知道每个 0 占多长时间。这就是时钟恢复困难。

Trap

NRZ 的”带宽利用率较高”是相对于曼彻斯特编码而言的，不要把它当成绝对优点。在实际网络（如以太网）里，你几乎看不到裸的 NRZ，就是因为那两个问题太致命了。

4.2.2 NRZI

NRZI（Non-Return-to-Zero Inverted）是一种差分编码。它用 ” 有无跳变 ” 而非绝对电平表示比特，常见定义是 ” 跳变表示 1，不跳变表示 0”。这样可以缓解连续 1 带来的问题，但连续 0 时仍可能长期无跳变，因此同步问题并未根本消失。

Note

NRZI 的巧妙之处在于它把信息藏在了”变化”里，而不是”绝对值”里。这意味着即使整根线的正负极被接反了，你仍然能正确解码——因为”有没有跳变”不会因为极性反转而改变。USB 2.0 就用了类似的差分思想来抗干扰。

4.2.3 曼彻斯特编码与差分曼彻斯特编码

曼彻斯特编码在每个比特周期中间都发生一次跳变，因此天然携带时钟信息，具有良好的自同步能力。代价是频率成分更高、带宽开销更大，其数据率通常只有带宽的同数量级：

R≈B

差分曼彻斯特编码进一步引入差分思想：位中间始终有跳变，而位开始处是否跳变表示不同比特值。它兼顾了自同步能力与对极性反转的鲁棒性。

Example

曼彻斯特编码的设计真的很妙——它强制每个比特中间都要翻一次身，所以接收端只要盯着信号的边沿，就能源源不断地恢复出时钟。以太网 10Base-T 用的就是曼彻斯特编码。代价也明显：同样带宽下，数据率只有 NRZ 的一半左右，因为一半的”带宽预算”被花在了时钟同步上。

Trap

曼彻斯特编码和差分曼彻斯特编码的跳变规则不一样，容易搞混：

  • 曼彻斯特：中间跳变表示数据（比如高→低表示 1，低→高表示 0，具体看约定）；
  • 差分曼彻斯特：中间必跳变（用来同步），位开始处跳变表示 0，不跳变表示 1（也有反过来定义的版本，关键是理解”差分”在比较相邻位边界的含义）。 考试如果考具体定义，一定要看清楚教材用的是什么约定。

4.3 常见基带编码的比较

flowchart LR
    A[基带编码]
    A --> B[NRZ]
    A --> C[NRZI]
    A --> D[曼彻斯特]
    A --> E[差分曼彻斯特]

    B --> B1[简单]
    B --> B2[带宽效率较高]
    B --> B3[无自同步能力]

    C --> C1[差分表示]
    C --> C2[缓解连续 1 问题]
    C --> C3[连续 0 仍困难]

    D --> D1[位中间必跳变]
    D --> D2[可恢复时钟]
    D --> D3[带宽开销较大]

    E --> E1[保持自同步]
    E --> E2[对极性反转更稳健]

Trap

面试或者简答题里让你比较编码方式时，一定要从三个维度回答：

  1. 自同步能力（有没有保证跳变）；
  2. 带宽效率（R 和 B 的关系）；
  3. 抗干扰/抗极性反转能力。 只背名字和特点是远远不够的，得分点在于你能把这几个维度横向比较出来。

4.4 带通调制

带通调制的目标，是把基带信号映射到载波的某种参数变化上。若载波写为

s(t)=Asin(2πft+φ)

则可调制的典型参数为：

  • 幅度 A
  • 频率 f
  • 相位 φ

因此最基本的数字调制包括：

  • ASK / 调幅：用不同幅度表示不同符号；
  • FSK / 调频：用不同频率表示不同符号；
  • PSK / 调相：用不同相位表示不同符号。

4.4.1 ASK、FSK、PSK 的特点

  • 调幅实现简单，但抗干扰能力较弱；
  • 调频抗干扰优于调幅，但频带利用率不高；
  • 调相尤其是 BPSK，具有较好的抗噪声性能，是很多更复杂调制方式的基础。

Intuition

调幅就像你说话声音时大时小，大声表示 1，小声表示 0；调频就像你说话音调忽高忽低；调相就像你突然换了个发音的相位（比如正常念和倒着念）。调幅最容易受环境噪声影响——因为噪声本身往往就表现为幅度的随机波动。

4.4.2 QPSK 与 QAM

为提高每个符号承载的比特数，可以使用多元调制。

  • QPSK：使用四种相位，每个符号表示 2 比特；
  • QAM：联合使用振幅与相位。
    例如：
    • QAM-16：16 个星座点，每符号表示 4 比特；
    • QAM-64：64 个星座点，每符号表示 6 比特。

多元调制的本质，是在单位时间内让每个码元携带更多比特；但星座点越密，点间距离越近，对噪声、频偏、相位误差的容忍度越低，误码率越容易升高。

flowchart TB
    A[提高数据率] --> B[增大带宽]
    A --> C[提高每符号比特数]

    C --> D[QPSK<br>2 bit/符号]
    C --> E[QAM-16<br>4 bit/符号]
    C --> F[QAM-64<br>6 bit/符号]

    F --> G[判决更难]
    E --> G
    D --> G
    G --> H[对 SNR 的要求更高]

Trap

QAM 的星座图是理解它的关键。你可以把星座图想象成二维平面上的点阵，横轴是相位，纵轴是幅度（或者 I/Q 两路）。QAM-64 就是把 64 个点塞在同一个空间里，点与点之间的”安全距离”变小了。信噪比不够高的时候，噪声会把一个点吹到隔壁点那里去，这就产生了误码。所以 Wi-Fi 信号差的时候会自动降速（从高阶 QAM 退回到低阶），就是这个道理。

4.5 编码与调制的边界

在本课程语境中，基带调制通常直接称为 ” 编码 “，而带通调制才称为 ” 调制 “。两者都在物理层内工作，但解决的问题不同：

  • 编码解决的是 ” 如何把比特变成适合线路的数字波形 ”；
  • 调制解决的是 ” 如何把基带数据搬移到某个适合传输的频段 ”。

Note

不同教材对”编码”和”调制”的用词习惯可能略有不同。有的书把所有基带层面的波形设计都叫”编码”（比如曼彻斯特编码），把所有载波层面的频谱搬移都叫”调制”。面试时如果被问到区别，你就按上面这个逻辑回答，基本不会错。

5. 信道复用

5.1 复用的基本思想

当单个信源速率远低于信道总速率时，让每个用户独占一条物理线路会造成巨大浪费。复用的目标，就是让多路信号共享一条高带宽传输线路，在接收端再完成分离。

flowchart TD
    A1[用户A] --> M[复用器]
    B1[用户B] --> M
    C1[用户C] --> M
    M --> CH[共享信道]
    CH --> D[分用器]
    D --> A2[用户A数据]
    D --> B2[用户B数据]
    D --> C2[用户C数据]

Intuition

复用就像一条高速公路，如果只让一辆车跑，那是暴殄天物。于是大家想出了各种办法：有的按车道分（频分），有的按时间轮流走（时分），有的用不同颜色的车灯来区分（码分）。本质上都是想让这条昂贵的物理链路物尽其用。

5.2 频分复用 FDM

频分复用把总带宽分成若干互不重叠的子频带。每个用户长期占用其中一段频率范围，并在其上持续传输。

其关键特点是：

  • 多个用户同时发送；
  • 每个用户占用不同频带；
  • 相邻子信道之间常需设置保护带，以缓解滤波器非理想造成的串扰。

FDM 常见于广播电视系统、早期电话干线系统等。

flowchart TB
    A[总频带] --> B[子频带1]
    A --> C[子频带2]
    A --> D[子频带3]
    B --> B1[用户A]
    C --> C1[用户B]
    D --> D1[用户C]

Example

你开车时听收音机，不同的电台就是在不同频率上同时广播的，你的收音机通过调谐频率来”选择”你想听的那一路。这就是生活中最直观的 FDM。

5.3 OFDM：正交子载波思想

OFDM 并不是简单的 ” 分出很多互不相交的保护带 “，而是使用一组严格正交的子载波。这样，各子载波频谱虽然重叠，但在采样点上彼此正交，因此可以在不留传统保护带的情况下高效利用频谱。

OFDM 的重要特征包括：

  • 多个子载波可并行传输；
  • 每个子载波可使用不同的调制方式；
  • 便于根据各子载波信道质量自适应分配比特数；
  • 可以利用 IFFT/FFT 高效实现。

它在思想上兼具 ” 分而治之 ” 和 ” 频谱高效复用 ” 两种优点，因此在现代宽带无线与有线系统中都极其重要。

Intuition

OFDM 的设计真的很巧妙。传统的 FDM 就像几个歌手每人占一个房间，中间还得留隔音墙（保护带），防止串音。OFDM 则让所有人站在同一个舞台上，但要求他们的声音在数学上严格”正交”——你可以理解为，A 唱歌的时候 B 刚好在呼吸，B 唱歌的时候 A 刚好换气。只要精准配合，就不会互相干扰，而且舞台空间利用率极高。Wi-Fi 和 4G/5G 都重度依赖 OFDM。

Note

OFDM 里那个”正交”不是日常生活中说的”互相垂直”那么简单，而是指不同子载波在频域上满足特定的积分正交条件。工程上这个性质让 IFFT/FFT 能够高效实现调制和解调，是现代通信系统能用硬件低成本处理海量子载波的关键。

5.4 波分复用 WDM

在光纤中，不同波长的光可以类比为不同频率的子信道，因此 WDM 本质上是光域中的频分复用。它使得一根光纤上可以同时承载多个光载波，大幅提升总容量。

Note

WDM 是”波分”，FDM 是”频分”，本质上是一回事，只是因为光通信里习惯用波长 λ 而不是频率 f 来描述（毕竟 c=fλ，两者一一对应）。DWDM（密集波分复用）就是把波长间隔压得更小，在一根光纤里塞更多路光信号，是现代骨干网扩容的核心技术。

5.5 时分复用 TDM

时分复用把时间划分为周期性出现的帧，每个用户在每帧中占用固定时隙。所有用户共享同一频带，但在不同时间发送。

TDM 的关键前提是严格同步：用户必须知道自己的时隙边界，且时隙之间常需设置保护间隔以容忍同步偏差和远近效应。

flowchart TD
    F1["TDM帧1<br>A | B | C | D"] --> F2["TDM帧2<br>A | B | C | D"]
    F2 --> F3["TDM帧3<br>A | B | C | D"]

TDM 的问题在于：若某用户在某一轮没有数据，该用户时隙仍然空置，线路利用率下降。对于突发性很强的计算机数据，这种浪费尤为明显。

Example

TDM 就像四个人轮流打一桌麻将，不管你这把有没有牌要打，轮到你的 15 秒你必须坐那儿。如果你没牌出，这个时隙就浪费了。对语音这种稳定流还行，对上网这种突突停停的流量，TDM 就显得有点呆板。

Trap

TDM 的”帧”和链路层的”帧”是完全不同的概念，不要混淆。TDM 帧是物理层的时间结构，用来给各路信号分配时隙；数据链路层的帧是逻辑结构，用来封装上层数据。两者都叫 frame，但层次和含义完全不同。

5.6 统计时分复用 STDM

统计时分复用不再为每个用户永久保留固定时隙，而是按需动态分配。其核心思想是：谁有数据，谁占用时隙。这样更适合计算机网络中的突发数据流，能够显著提高链路利用率。

flowchart LR
    A[固定 TDM]
    A --> A1[每轮固定给每个用户分槽]
    A --> A2[空闲用户时隙会浪费]

    B[统计 TDM]
    B --> B1[按需分配时隙]
    B --> B2[适合突发数据]
    B --> B3[利用率更高]

Example

STDM 更像是滴滴拼车：不是每辆车固定等某个乘客，而是谁叫车谁上车，凑够人就出发。每个时隙前面通常要带上地址标签，否则接收端不知道这个时隙是属于谁的。以太网交换机里的缓存调度，本质上也是一种统计复用的思想。

5.7 码分复用 CDM / CDMA

码分复用不在频域或时域切分资源，而是让多个用户在同一时间、同一频带上传输，只是每个用户使用不同的码片序列。不同码序列在理想情况下互相正交，因此可以在接收端通过相关运算把目标用户的数据 ” 解扩 ” 出来。

5.7.1 码片序列与扩频

设原始数据率为 b bit/s，若每个比特扩展为 m 个码片，则发送码率变为 mb，带宽也相应扩展为原来的 m 倍。这就是扩频。

发送规则通常是：

  • 发送比特 1：发送自身码片序列；
  • 发送比特 0：发送该码片序列的反码。

5.7.2 正交性与解码

若用户 S 和用户 T 的码片向量正交，则有：

S⋅T=0

而用户自己的码片与自身的规格化内积为 1，与其反码的规格化内积为 −1。因此，接收端拿目标码序列去相关，就可以在混叠信号中恢复对应用户的比特。

flowchart TD
    A[用户A数据] --> EA[乘以码序列 CA]
    B[用户B数据] --> EB[乘以码序列 CB]
    EA --> SUM[同频同时间叠加]
    EB --> SUM
    SUM --> RA[与 CA 做相关]
    SUM --> RB[与 CB 做相关]
    RA --> OA[恢复用户A数据]
    RB --> OB[恢复用户B数据]

Intuition

CDMA 是个不太好直观理解的东西，咱们用个不太严谨但能帮助记忆的类比：想象在一个嘈杂的派对上，大家都在同时说话。但如果你和 A 约定好只用”普通话”，和 B 约定好只用”粤语”，理论上你在脑子里只”调谐”到普通话频道，就能从一片嘈杂中提取出 A 说的话。码片序列就是你的”语言约定”，正交性保证了你不会在听 A 的时候被 B 带跑。

Example

假设用户 A 的码片序列是 (+1,+1,+1,−1)。如果 A 要发比特 1，就发送 (+1,+1,+1,−1)；发比特 0 就发送 (−1,−1,−1,+1)。接收端收到混合信号后，用 A 的码片序列做内积：结果为正就是 1，为负就是 0，为 0 就说明这段信号里没有 A 的成分。

5.7.3 CDMA 的工程特征

CDMA 具有以下特点：

  • 抗干扰能力强；
  • 频谱展宽后不易被检测，具有一定隐蔽性；
  • 可用于蜂窝、卫星等无线广播型信道；
  • 需要较严格的功率控制，否则强用户会淹没弱用户。

Trap

CDMA 的”远近效应”是考试常考点。如果用户 A 离基站很近，信号很强，用户 B 离得很远，信号很弱，那么 A 的强信号会像一个”大嗓门”把 B”淹没”。因此 CDMA 系统必须有精密的功率控制，让所有用户到达基站的信号强度尽量一致。这也是为什么 CDMA 手机的功耗控制比 GSM 复杂得多。

6. 传输介质

6.1 传输介质的分类

传输介质是发送器与接收器之间的物理通路，可分为：

  • 有线传输介质：电磁波沿固体介质传播，如铜线、同轴电缆、光纤；
  • 无线传输介质：电磁波在自由空间传播。
flowchart LR
    A[传输介质]
    A --> B[有线]
    A --> C[无线]

    B --> B1[双绞线]
    B --> B2[同轴电缆]
    B --> B3[光纤]

    C --> C1[无线电]
    C --> C2[微波]
    C --> C3[卫星]
    C --> C4[红外/激光]

6.2 有线介质

6.2.1 双绞线

双绞线由两根彼此绝缘的导线绞合而成。其关键物理思想是：外部干扰在两条线上大致一致，形成共模噪声，而接收端使用差分方式提取信号，从而抑制干扰。

双绞线分为：

  • UTP：无屏蔽双绞线
  • STP：屏蔽双绞线

它既可用于模拟传输，也可用于数字传输，局域网接入场景极为常见。双绞线成本低、施工方便，但带宽与传输距离都有限。课程中给出的典型工程尺度是：以太网接入常见距离约为 100 m。

在类别上，常见关注点是：

  • Cat5：百兆级应用
  • Cat5e：可支持千兆级应用
  • Cat6 / Cat7：用于更高速率与更严格串扰约束场景

Intuition

双绞线的”绞合”是个非常巧妙的设计。两条线绞在一起，外部电磁干扰对两条线的影响几乎一样——就像两艘并排的小船被同一阵浪推得同样高。接收端只关心两条线之间的差值（差分信号），共模的干扰就被抵消掉了。绞得越密，抗干扰效果越好。

Note

Cat5 和 Cat5e 的区别不只是在纸面参数上——Cat5e 增加了对串扰和回波的测试要求，所以很多老 Cat5 线实际上也能跑千兆，但不符合规范。考试如果考到，记住数字：Cat5 → 100 Mbps，Cat5e → 1 Gbps，Cat6 → 10 Gbps（短距离）。

6.2.2 同轴电缆

同轴电缆由中心导体、绝缘层、外导体屏蔽层和保护外皮组成。其结构天然具有良好的抗干扰能力，因此适于较高数据率传输。

按用途可分为：

  • 50 Ω 同轴电缆：更常见于数字传输和局域网历史系统；
  • 75 Ω 同轴电缆：更常见于有线电视和模拟视频系统。

历史上，以太网曾使用基带同轴电缆，例如 10Base-5 与 10Base-2。对于同轴电缆，阻抗匹配很重要；终端不匹配会引起反射，破坏信号质量。

Note

同轴电缆的”50 Ω”和”75 Ω”不是直流电阻，而是特性阻抗，和电缆的几何结构、介质材料有关。面试时如果被问到为什么有两种阻抗，标准答案是：50 Ω 兼顾了功率容量和衰减（适合数字通信），75 Ω 衰减更小（适合长距离视频传输）。

6.2.3 光纤

光纤以光信号作为载体，核心原理是纤芯折射率高于包层，使光在满足临界角条件时发生全反射，从而沿纤芯传播。

flowchart TD
    A[光从纤芯射向包层界面]
    A --> B{入射角是否大于临界角}
    B -->|是| C[发生全反射]
    C --> D[沿纤芯继续传播]
    B -->|否| E[折射泄漏]

光纤按传播模式分为：

  • 多模光纤：多条不同路径同时传播，易产生模间色散；
  • 单模光纤：基本只有单一路径传播，适于更高速率与更长距离。

光纤系统通常由三部分组成：

  1. 光源：LED 或激光器，把电信号变为光脉冲；
  2. 光纤链路：负责长距离传输；
  3. 光检测器：光电二极管等，把光信号恢复为电信号。

光纤的主要优点是：

  • 通信容量极大；
  • 衰减小，中继距离长；
  • 抗电磁干扰与抗雷击能力强；
  • 无串音，保密性好；
  • 体积小、重量轻。

Intuition

光纤里走光的原理，和你在玻璃管里射一束激光差不多——只要入射角够大，光就会在纤芯和包层的界面上反复”弹”着前进。多模光纤就像一根粗管子，光可以沿各种角度弹，路径长短不一，所以到了对面有的光先到有的光后到，信号就”散”了（模间色散）。单模光纤把管子做得极细，只允许一种模式传播，大家的速度一样快，就能跑得更远、更快。

6.3 无线介质

6.3.1 无线电与频谱管理

无线传输使用自由空间作为介质。不同频段具有不同传播特性与应用边界。无线信号在空间中容易产生同频干扰，因此频谱需要由管理机构分配；少量频段，如部分 ISM 频段，可供低功率设备开放使用。

6.3.2 微波通信

微波具有较强方向性，常用于点到点视距通信。它的基本工程特征包括：

  • 频率较高；
  • 天线可以形成窄波束；
  • 以视距传播为主；
  • 远距离时需要中继站；
  • 易受多径、天气和地球曲率影响。

自由空间传播下，接收功率与距离平方成反比，可写为：

Pr​=Pt​(4πdλ​)2

该式揭示了一个基本事实：距离增大时，路径损耗迅速加剧。

Note

公式里的 λ 是波长，频率越高波长越短，路径损耗就越大。这就是为什么 5G 毫米波覆盖范围小、穿墙能力差——频率上去了，λ 变小，自由空间损耗急剧增加。系统必须通过更密的小基站来弥补这个物理规律。

6.3.3 卫星通信

卫星通信本质上是把卫星当作更高处的微波中继站。地球同步卫星位于高空固定轨道，覆盖面广，适于广播式业务，但传播时延较大。低轨卫星则轨道更低、时延更小，但需要多颗卫星组成星座，持续完成覆盖切换。

flowchart LR
    A[地面站A] -->|上行链路| S[卫星转发器]
    S -->|下行链路| B[地面站B]

Note

同步卫星的往返时延大约是 250 ms，这个数值在很多教材里会直接考。对于实时交互业务（比如视频会议、在线游戏），这个时延是很要命的。低轨卫星（如 Starlink）把轨道高度压到几百公里，时延能降到 20-40 ms 量级，代价是需要几千颗卫星组网才能保证无缝覆盖。

7. 通信系统

7.1 公用电话网与 ADSL

7.1.1 PSTN 的基本结构

公用电话交换网由以下部分构成：

  • 本地环路：连接用户与端局，传统上多为模拟双绞线；
  • 交换局：完成呼叫转接；
  • 中继线 / 干线：连接各交换局，通常为高速数字链路。
flowchart TD
    U1[用户] --> E1[端局]
    U2[用户] --> E1
    E1 --> T1[长途局]
    T1 --> C1[中心交换局]
    C1 --> T2[长途局]
    T2 --> E2[端局]
    E2 --> U3[用户]

从系统演化上看，电话网从早期模拟干线、FDM 复用，逐步发展为数字干线、PCM/TDM 复用。本地环路之所以长期保留铜线，是因为用户侧距离短、成本敏感，而干线侧则更适合高容量数字传输。

Intuition

电话网的演进逻辑特别有意思——它是”两头慢、中间快”。干线上早就换成了光纤和数字传输，但到你家门口那段铜线（本地环路）因为牵涉到挨家挨户重新布线，成本极高，所以一直赖着没换。ADSL、HFC、FTTx 本质上都是在解决”最后几公里”的问题。

7.1.2 PCM、T1 与 E1

PCM 的目标，是在电话局间中继线上传送多路话音。其历史问题在于不同地区形成了不同标准：

  • T1：24 路，1.544 Mbit/s
  • E1：30 路业务信道加控制与同步，2.048 Mbit/s

T1 的总速率可写为：

[(7+1)×24+1]×8kHz=1.544 Mbit/s

E1 的总速率可写为：

8×32×8kHz=2.048 Mbit/s

其中 T1/E1 都体现了 TDM 的思想：在固定时间帧中周期性为各路话音分配时隙。

Trap

T1 和 E1 的参数是死记硬背的考点。关键区分：T1 是美国/日本标准，24 路话音，每路 8 bit（7 bit 数据 + 1 bit 信令），加上 1 个帧同步位，总共 24×8+1=193 bit 每帧，采样率 8 kHz。E1 是欧洲/中国标准，32 个时隙（30 路话音 + 2 路控制），每时隙 8 bit，32×8=256 bit 每帧，采样率同样是 8 kHz。千万别把 24 和 30 搞反。

7.1.3 PDH 到 SONET/SDH

旧式 PCM 系统存在两个根本问题：

  1. 速率标准不统一；
  2. 高速场景下收发时钟并不严格同步。

PDH 属于准同步体系，各路时钟允许有小偏差，复接时需要插入额外脉冲，结构复杂。为解决这一问题，后续发展出 SONET/SDH，同步时钟来源统一，便于复接、分接和国际标准化。

SONET 的基本速率等级为 STS-1 / OC-1：

8×810×8kHz=51.84 Mbit/s

SDH 以 STM-1 为基本等级：

155.52 Mbit/s

可以把 SONET/SDH 理解为：在光纤高速传输时代，对 ” 速率等级、帧结构、复用方式、同步方式 ” 进行统一规范的体系。

flowchart LR
    A[PDH / 准同步] --> B[标准不统一]
    A --> C[复接复杂]
    A --> D[高速同步困难]

    E[SONET / SDH] --> F[统一速率等级]
    E --> G[全网同步]
    E --> H[易于复接与分接]

Note

SONET 是北美标准，SDH 是国际标准（欧洲和中国采用），但两者本质上兼容。STM-1 速率 155.52 Mbit/s 刚好是 STS-3 的速率（3×51.84）。现代光传输网的速率等级基本都是这个基数的整数倍：STM-4 → 622 Mbit/s，STM-16 → 2.5 Gbit/s，STM-64 → 10 Gbit/s。

7.1.4 宽带接入与 ADSL 的出发点

宽带接入的本质，是让用户侧最后一段接入链路能够承载更高数据率的互联网业务。早期拨号上网受限于电话用户线与模拟调制解调器，速率很低。ADSL 的思路不是推翻既有电话铜线，而是在原有用户线之上进行频谱再利用。

其关键观察是：

  • 电话业务只占用了低频窄带；
  • 用户线本身实际上可以通过更高频率成分；
  • 因而可把低频留给电话，把高频留给数据业务。

Intuition

ADSL 的设计思路特别值得学习——它不是推倒重来，而是”废物利用”。你家那根用了几十年的铜线，如果只用来传 4 kHz 的语音，简直是浪费。工程师们发现这根线其实能跑更高频率，于是把高频段”捡”来传数据，低频继续留给你打电话，两边互不干扰。这种在既有约束下找增量空间的思路，做系统设计的同学可以好好品一品。

7.1.5 ADSL 的机制

ADSL 是非对称数字用户线。它把用户线频谱切分为三部分：

  • 0∼4 kHz：传统电话
  • 较低高频段：上行
  • 更宽高频段：下行

非对称的原因在于居民接入普遍以下行为主，因此把更多频谱资源分配给下行。

flowchart LR
    A[一对铜线] --> B[低频: 电话]
    A --> C[中高频: 上行数据]
    A --> D[更高频: 下行数据]

ADSL 在用户端与端局端都需要调制解调器，并借助分离器把电话信号与 DSL 信号分开。端局侧通过 DSLAM 汇聚大量用户线数据，再接入上层网络。

flowchart LR
    U[用户设备] --> S1[分离器]
    P[电话机] --> S1
    S1 --> L[用户线]

    L --> S2[端局分离器]
    S2 --> T[本地电话网]
    S2 --> M[ADSL 调制解调器 / ATU-C]
    M --> D[DSLAM]
    D --> I[ISP / 区域宽带网]

Trap

ADSL 的”非对称”是面试常考点。记住：上行走低频窄带（因为高频衰减更严重，从你家到端局的上行信号比较弱），下行走高频宽带（端局设备功率大，能往你家里灌更多高频信号）。这个不对称不是随意设计的，而是受物理规律（高频衰减快）和业务模式（下载远大于上传）双重驱动的。

7.1.6 DMT：ADSL 的实现方式

ADSL 常采用 DMT（离散多音调）技术。其本质是在一对铜线上并行使用大量窄带子信道，每个子信道可看作一个小型调制解调器。不同子信道根据实际信道质量分配不同调制方式与比特数。

这一机制体现了两层思想：

  1. 在频域上使用多子信道并行传输；
  2. 在每个子信道上根据信道状况进行自适应分配。

因此，ADSL 实际数据率并不是固定常数，而与用户线长度、线径、串扰、噪声水平等因素密切相关。线路越长、条件越差，可用速率越低；线路质量极差时甚至无法开通。

Note

DMT 和 OFDM 在数学本质上是一回事，都是把信道切成很多窄带子载波来并行传输。ADSL 里的 DMT 可以看作 OFDM 在有线铜线场景下的工程实现。这也是为什么学了 OFDM 再回头看 DMT，会有一种”原来如此”的感觉——知识是相通的。

7.1.7 ADSL2 与 ADSL2+

ADSL2 与 ADSL2+ 在保留原理不变的前提下，通过扩大可用频谱、提升调制效率、引入无缝速率自适应等方式提高性能。它们的意义不在于 ” 重新发明接入方式 “，而在于对已有铜线条件进行更精细的频谱利用与链路管理。

7.2 有线电视网与 HFC

7.2.1 HFC 的出发点

传统 CATV 网是树形同轴电缆网络，主要面向单向电视广播。HFC（Hybrid Fiber Coax）是在此基础上发展起来的宽带接入体系，其目标是：保留既有覆盖广泛的电视电缆网络，同时引入更高容量和双向能力。

7.2.2 HFC 的结构

HFC 的主干部分改用光纤，从头端到光纤节点完成长距离高带宽传输；在光纤节点以下，仍使用同轴电缆分配到大量家庭用户。因此，它是一种 ” 光纤 + 同轴 ” 的混合系统。

flowchart TD
    H[头端 Headend] --> F[光纤干线]
    F --> N[光纤节点 / 光电转换]
    N --> C1[同轴分配网]
    C1 --> U1[用户1]
    C1 --> U2[用户2]
    C1 --> U3[用户3]

Intuition

HFC 和 ADSL 解决的是同一个问题（宽带接入），但出发点完全不同。ADSL 是从电话网的铜线出发做加法，HFC 是从有线电视网的同轴电缆出发做改造。一个的优势是每家独占一根线（虽然也是铜），另一个的优势是本来就是为视频广播设计的高带宽网络，改造潜力更大。

7.2.3 HFC 的频谱分配与业务特征

HFC 在同轴电缆上采用频分方式划分上行、下行与广播业务频段。其总体特征是：

  • 以下行为主；
  • 上下行带宽非对称；
  • 电视与数据在同一物理基础设施上共存；
  • 同一段同轴电缆上的用户共享传输资源。

因此，HFC 与 ADSL 的关键差别在于：ADSL 的本地铜线通常是用户独占的，而 HFC 中同轴段往往是多个用户共享的。

7.2.4 Cable Modem

电缆调制解调器是 HFC 的用户侧接入设备。下行通常可以提供较高带宽；而上行由于多用户共享，需要通过竞争或调度机制申请发送机会。这一结构决定了 HFC 的性能不仅取决于线路质量，也取决于同时在线用户数量与上行调度机制。

7.2.5 ADSL 与 HFC 的对比

flowchart LR
    A[ADSL]
    A --> A1[用户本地环路独占]
    A --> A2[受用户到端局距离影响大]
    A --> A3[铜线频谱切分 电话+数据]

    B[HFC]
    B --> B1[同轴段多用户共享]
    B --> B2[下行为主]
    B --> B3[性能受共享负载影响]

Trap

ADSL 和 HFC 的对比是简答题的高频素材。核心差异就两个：一是介质（铜线 vs 光纤+同轴），二是共享模式（用户独占 vs 同轴段共享）。很多同学习惯性说”ADSL 慢、HFC 快”，这是不准确的——ADSL 的速度取决于你离端局多远，HFC 的速度取决于同一段电缆上有多少人在抢带宽。高峰期 HFC 可能比 ADSL 还慢。

7.3 FTTx 光接入

FTTx 表示 ” 光纤到某处 ” 的一类接入方案，其共同思路是：尽可能把光纤向用户侧推进，从而提升接入带宽并降低铜线段长度。

常见形式包括：

  • FTTH：光纤到户
  • FTTB：光纤到楼
  • FTTC：光纤到路边

它们的差别并不在于核心原理，而在于光纤终止点的位置不同，以及从该点到最终用户的介质不同。

flowchart LR
    H[局端] --> O[光纤干线]
    O --> S[光分路器]
    S --> H1[FTTH: 直接到家庭]
    S --> B1[FTTB: 到楼内]
    S --> C1[FTTC: 到路边机柜]

FTTx 代表了接入网长期演进方向：让高容量光纤越来越靠近用户，逐步减少最后一段铜线的限制。

Note

现在国内新建小区基本都是 FTTH 了，但老小区里你可能还能看到 FTTB（光纤到楼道，楼里再用网线或同轴分）。PON（无源光网络）是 FTTH 的主流实现方式，通过光分路器让一根光纤分给几十户人家，大幅降低了局端设备成本。GPON 下行 2.5 Gbit/s，XGS-PON 已经到 10 Gbit/s 对称了。

7.4 移动通信网

7.4.1 蜂窝思想与频率复用

移动通信系统的核心，不是单一一条大覆盖链路，而是把地理区域划分为多个小区，在不同小区重复使用同一频谱资源。这样可以在有限频谱下支持更多用户。

flowchart TB
    A[蜂窝小区1]
    B[蜂窝小区2]
    C[蜂窝小区3]
    D[蜂窝小区4]

    A --- B
    B --- C
    C --- D
    A --- D

    A1[相同频点可在足够远处重用]

小区越小，频率复用越密，系统容量越高；但同时基站数量、切换复杂度与网络建设成本也会上升。

Intuition

蜂窝思想是移动通信里最美妙的点子之一。想象你有一块大披萨（频谱），直接全给一个基站用，所有人抢，根本不够分。蜂窝网络把披萨切成很多小块，每个小区拿一块，但相隔足够远的小区可以拿同样的口味（频率），因为他们不会互相干扰。把小区做得越小，同样的频谱就能喂饱更多的人——这也是为什么演唱会、体育场馆里会临时加装”微蜂窝”或者”皮蜂窝”。

7.4.2 GSM 的复用结构

GSM 在物理层上体现了 ” 频分 + 时分 ” 的组合思想：

  1. 先在频域上划分出多个 200 kHz 子信道；
  2. 再在每个子信道上使用 TDM，把时间切成多个时隙分配给不同用户。

因此，GSM 不是只靠一种复用方式工作，而是把 FDM 与 TDM 组合起来，用于移动接入这一共享无线信道场景。

flowchart LR
    A[总频带] --> B[200kHz 子信道1]
    A --> C[200kHz 子信道2]
    B --> B1[时隙1]
    B --> B2[时隙2]
    B --> B3[时隙3]
    C --> C1[时隙1]
    C --> C2[时隙2]

Note

GSM 每个载波 200 kHz，分成 8 个时隙，所以一个载波最多同时服务 8 个全速率用户（不考虑话音激活、半速率等优化）。这就是为什么 2G 时代经常出现”基站看着不少，人一多就网络繁忙”的情况——它的资源粒度比后来的 3G/4G 粗得多。

7.4.3 从 1G 到 5G 的演进方向

移动通信代际演进，可以粗略理解为：

  • 1G：模拟话音；
  • 2G：数字话音；
  • 3G：数字话音与数据；
  • 4G：高速数据；
  • 5G：更高带宽、更低时延、更密集组网。

5G 的重要物理层特征包括：

  • 使用更高频段，获得更宽带宽；
  • 小区进一步缩小，基站更密；
  • 引入大规模 MIMO；
  • 通过波束赋形提高定向传输能力；
  • 支持 D2D 等更灵活的链路形式。

频率越高，虽然可用带宽越大，但传播越接近直线、覆盖越小、衰减越明显，因此系统必须通过微基站部署与天线阵列技术来补偿传播劣势。

flowchart TD
    A[更高频率] --> B[更宽带宽]
    A --> C[更大传播损耗]
    C --> D[更小小区]
    D --> E[更密基站]
    E --> F[波束赋形 / 大规模 MIMO]

Note

5G 用毫米波（24 GHz 以上）就是这个矛盾的极致体现：带宽大到惊人，但穿墙能力弱到离谱，一片树叶、一场雨都能显著影响信号。所以 5G 毫米波基站必须非常密集，而且大量依赖波束赋形来把能量像手电筒一样精准照向用户，而不是像传统基站那样”广播”。这也是为什么 5G 建设成本高，而且室内覆盖尤其麻烦。

8. 本讲知识主线总结

物理层研究的不是 ” 信息内容 “，而是比特如何变成信号，信号如何在介质中传播，以及传播能力受什么物理边界约束。这一层的知识主线可以概括为以下链条：

  1. 数据必须变成可传输信号；
  2. 信号经过信道会受到带宽、噪声、衰减和失真的约束；
  3. 奈奎斯特定理和香农公式给出了速率上界；
  4. 同步机制决定接收端能否正确判定位边界；
  5. 编码与调制决定比特如何映射到可传输波形；
  6. 复用决定多个用户如何共享物理资源；
  7. 不同传输介质决定可用频段、距离、成本与抗干扰能力；
  8. 电话网、ADSL、HFC、FTTx 与移动通信网，都是在这些共同物理原理上形成的不同系统实现。
flowchart TD
    A[数据] --> B[编码/调制]
    B --> C[信号]
    C --> D[信道传播]
    D --> E[带宽/噪声/衰减限制]
    E --> F[同步与判决]
    F --> G[恢复比特]
    G --> H[系统实现: 电话网/ADSL/HFC/FTTx/移动网]
