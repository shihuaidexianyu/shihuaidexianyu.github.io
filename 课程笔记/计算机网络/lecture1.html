第一讲 计算机网络概述

一、计算机网络的作用和定义

1. 计算机网络的地位

计算机网络是在计算设备之间传输数据、共享资源并支撑应用运行的系统。它使分散的计算机、移动终端、传感器和服务器能够在统一的通信规则下协同工作，从而把孤立的计算能力转化为可互联、可共享、可扩展的信息基础设施。

从社会层面看，网络不仅承载文件传输、电子邮件、即时通信、信息检索、导航定位、电子商务和网络教育等应用，也深刻改变了知识获取、生产组织、商业活动与社会交往方式。网络的价值随着互连规模扩大而显著增加。若将网络视为由 N 个结点组成的系统，则其潜在互连价值常用梅特卡夫定律近似表达为

V∝N2

这一定律强调的并不是严格的物理规律，而是互连规模扩大后连接机会、协作机会和资源共享机会的快速增长。

2. 计算机网络的基本构成

计算机网络由三个基本要素构成。

第一，通信设备。通信设备负责接收、处理、转发或接入数据，典型设备包括网络接口卡、调制解调器、中继器、交换机、路由器和无线接入点等。

第二，通信线路。通信线路是结点之间的连接媒介，可以是双绞线、同轴电缆、光纤等有线介质，也可以是无线电链路等无线介质。

第三，通信协议。协议规定了数据交换时双方必须遵守的规则、格式、动作与时序，是网络能够正常工作的前提。互联网中的 TCP/IP、局域网中的 IEEE 802.3 以太网协议以及无线局域网中的 IEEE 802.11 都属于典型协议。

3. 计算机网络的定义

计算机网络可以定义为：由若干通用、可编程的硬件通过通信线路互连而成，并在协议控制下传送多种类型数据、支持广泛应用的系统。

这个定义包含两层重要含义。

其一，网络连接的对象不局限于传统意义上的计算机，而包括智能手机、智能传感器、摄像头、服务器等各种具备处理和通信能力的端系统。

其二，网络并不是为单一数据类型服务的专用系统，而是能够承载数据、语音、视频以及后续不断出现的新型应用。

4. 计算机网络的基本特征

计算机网络最重要的两个特征是连通性与共享性。

连通性是指分散的用户和主机在逻辑上彼此可达，能够交换信息。这里的可达性主要是逻辑意义上的：用户通常并不知道对方的具体物理位置，也不必直接感知中间路径和设备。

共享性是指网络中的信息、软件、硬件和服务能力可以被多个用户共同使用。网络将远端资源表现为可被本地调用的服务，这是现代分布式计算、云服务和网络应用的基础。

5. 用图描述网络

从抽象角度看，网络可以表示为由结点和链路构成的图。结点表示主机、交换机、路由器等设备，链路表示它们之间的物理或逻辑连接。为了简化表示，复杂网络也常被抽象成 ” 云 “，以突出端点之间的逻辑连通关系而隐藏中间细节。

二、互联网的定义和组成

1. 互联网与互连网

以小写字母开头的 internet 是通用名词，泛指若干计算机网络互连而成的网络。只要多个网络能够互连并相互通信，就可以构成一个互连网，而不要求统一采用某一种协议。

以大写字母开头的 Internet 则是专用名词，特指当前全球最大的开放式互联网。它以 TCP/IP 协议族作为共同的通信规则，由数量巨大的网络和设备构成，目标是在全球范围内提供连通性与资源共享。

因此，Internet 是 internet 的一种特定实现，但并不是所有互连网都等同于 Internet。

2. 互联网的发展脉络

互联网的发展可以概括为三个阶段。

第一阶段是从 ARPANET 向互联网演化。1983 年 TCP/IP 成为 ARPANET 的标准协议，这使得使用 TCP/IP 的不同网络能够互连，因此 1983 年通常被视为互联网诞生的重要时间点。1990 年 ARPANET 正式关闭。

第二阶段是三级结构互联网。此时网络一般按主干网、地区网和校园网或企业网分层组织，呈现较为清晰的骨干—区域—接入结构。

第三阶段是多层次 ISP 结构。互联网服务提供者成为互连与接入的核心组织者，网络逐步演化为主干 ISP、地区 ISP、本地 ISP 以及各种内容提供者、企业网络和互联网交换点共同构成的多层互联系统。

    A[ARPANET 阶段] --> B[三级结构互联网]

    B --> C[多层次 ISP 互连]

    C --> D[全球开放互联网]

万维网的出现极大推动了互联网普及。WWW 将超文本、浏览器和统一资源定位机制结合起来，使非专业用户也能方便地使用互联网，从而推动互联网在 20 世纪 90 年代进入快速增长阶段。

3. 互联网的总体组成

从工作方式看，互联网可划分为边缘部分与核心部分。

边缘部分由所有连接在网络上的主机组成，也称端系统。这部分是用户直接使用的部分，负责运行应用、产生和接收数据。

核心部分由大量网络和将这些网络互连起来的路由器组成，负责为边缘部分提供连通性与交换功能。

接入网位于用户端系统与互联网第一个路由器之间，是用户进入互联网的桥梁。它既不等同于互联网核心，也不等同于端系统本身，而是连接用户与互联网的过渡层。

4. 端系统与进程通信

主机之间通信的准确含义并不是 ” 机器和机器 ” 之间直接对话，而是 ” 运行在主机上的进程与运行在另一台主机上的进程 ” 之间进行通信。网络最终服务的是应用进程之间的交互。

端系统之间的通信方式主要有两类。

（1）客户/服务器方式

客户/服务器方式描述的是进程之间的服务关系。客户是服务请求方，服务器是服务提供方。客户在需要通信时主动发起请求，因此必须事先知道服务器的地址；服务器通常持续运行，被动等待来自客户的请求，并可同时处理多个会话。


sequenceDiagram

    participant Server as 服务器进程

    Client->>Net: 发起连接/请求

    Net->>Server: 转交请求

（2）对等方式

对等方式不预先固定服务请求方和服务提供方的角色。只要双方都运行对等软件，任一主机都可以同时作为资源请求方和资源提供方。P2P 系统适合大规模分布式资源共享，但其每个参与节点本质上仍然在不同会话中分别扮演客户和服务器角色。

5. 端系统通信的编程抽象：Socket

在实际编程中，端系统通信通常通过 Socket API 完成。Socket 提供了应用进程与传输层之间的编程接口，使应用能够通过统一原语完成连接建立、数据发送与接收。

Socket 支持两类典型服务。

一类是字节流服务，强调可靠、按序地传输连续字节，典型对应 TCP。

另一类是数据报服务，强调以报文为单位发送，不保证可靠到达，典型对应 UDP。

不同应用进程通过不同端口号区分，端口号可理解为主机内部的逻辑服务入口。

6. 接入网

接入网是从用户端系统到互联网中第一个路由器之间的网络，其作用是把用户接入互联网。它解决的问题不是全网转发，而是 ” 用户如何进入网络 ”。

常见接入技术如下。

（1）DSL

DSL 利用现有电话线提供数据接入。语音与数据通过不同频率复用，因此可以在同一物理介质上同时传输。用户侧通过 DSL modem 接入，运营商侧通过 DSLAM 汇聚。DSL 的典型特点是上下行不对称，下行速率通常高于上行速率。

（2）电缆接入

电缆接入基于有线电视网络，常见于 HFC 架构。不同家庭共享到 headend 的接入链路，由 CMTS 完成终结和汇聚。与 DSL 相比，电缆接入的共享性更强，用户之间更明显地共享接入带宽。

（3）家庭网络

家庭网络通常由调制解调器、家用路由器、防火墙、NAT、以太网接口和无线接入点组成。其职责是在家庭内部组织局域连接，并向外通过单一接入链路进入互联网。

（4）企业接入网

企业和校园网络常使用以太网。主机通过交换机接入，内部可能部署 Web、邮件等服务器，并通过企业路由器连接 ISP。

（5）无线接入网

无线局域网解决建筑物、教室、家庭等较小范围内的无线接入问题；广域蜂窝无线网络则由运营商提供更大范围的移动接入能力。

7. 互联网核心：分组交换

互联网核心采用分组交换。它要解决的问题是：在没有为每对通信方预留专用电路的前提下，如何让海量主机共享网络资源并实现端到端通信。

（1）为什么不采用电路交换作为主流机制

电路交换在通信前需要先建立一条端到端的专用物理通路，通信期间资源被独占，结束后再释放。它适合持续、稳定的数据流，例如传统电话语音；但计算机数据通常具有突发性，大量时间处于空闲状态。若为每个会话长期保留专用资源，链路利用率会很低。


stateDiagram-v2

    [*] --> 建立连接

    建立连接 --> 数据传输

    数据传输 --> 释放连接

    释放连接 --> [*]

（2）分组交换的基本机制

发送端先将较长报文切分为较短的数据段，再为每个数据段加上首部，形成分组。首部中包含目的地址、源地址等控制信息。路由器收到分组后，先暂存在缓存中，再查找转发表，最后将其转发到合适的输出端口。这一过程称为存储转发。


flowchart LR

    M[报文] --> S[分段]

    S --> P1[分组1 = 首部 + 数据]

    S --> P2[分组2 = 首部 + 数据]

    S --> P3[分组3 = 首部 + 数据]

    P1 --> R1[路由器缓存]

    P2 --> R1

    P3 --> R1

    R1 --> F[查表并转发]

    F --> D[目的主机重组]

在互联网中，每个分组可以独立选择路径，因此不同分组未必沿完全相同的路径到达目的地。分组交换的核心设备是路由器，核心操作是转发。

（3）路由与转发

转发是局部动作，指路由器根据分组首部信息，在输入端口收到分组后，决定将其送往哪个输出端口。

路由是全局过程，指依据某种路由算法确定从源主机到目的主机的整体路径或下一跳选择策略。路由产生转发表，转发使用转发表。


flowchart TD

    A[分组到达输入端口] --> B[缓存]

    B --> C[检查首部中的目的地址]

    C --> D[查找转发表]

    D --> E[确定输出端口]

    E --> F[进入输出队列]

    F --> G[发送到下一跳]

（4）分组交换的优点与代价

分组交换的优点在于，链路资源按需动态分配，能够适应突发流量，具有较高利用率；不需要先建立专用电路，启动较快；多个分组可以独立选路，灵活性和生存性较好。

其代价在于，每个分组都要携带首部，产生协议开销；路由器中存在缓存和排队，因此会引入处理时延与排队时延；路径变化也会使时延和到达顺序更加复杂。

（5）统计复用

分组交换的重要思想是统计复用。用户并不在整个生命周期内独占链路，而是在真正有数据要发送时才占用带宽。多个用户若活跃时段错开，就可以共享同一链路。

若有 N 个独立用户，每个用户在任一时刻活跃的概率为 p，则同时活跃用户数 X 服从二项分布：

X∼Binomial(N,p)

若链路一次最多支持 K 个用户全速发送，则超载概率为

P(X>K)=i=K+1∑N​(iN​)pi(1−p)N−i

统计复用的收益来自 ” 平均情况下大量用户并不会同时满负荷发送 “，但它也意味着在不利情况下用户可能经历性能下降。

8. 路径探测

Traceroute 是从端系统角度观察网络路径的典型工具。它并不改变核心网的路由机制，而是通过探测不同跳数的中间节点，帮助用户理解从源到目的地之间经过了哪些路由器。这说明网络路径既是协议控制的结果，也可以被端系统部分观测。

三、计算机网络体系结构

1. 为什么要分层

计算机网络是高度复杂的系统。若把地址、路由、差错恢复、连接管理、应用语义、物理传输等问题全部放在单一协议中统一处理，设计、实现、维护和互操作都会非常困难。分层的目的就是将复杂问题拆解为相对独立的子问题，每层完成明确功能，并通过层间接口协作。

分层的好处在于结构清晰、便于实现、便于维护、便于标准化，也允许某一层在不破坏整体结构的前提下独立演进。其代价在于层间封装会增加开销，不同层之间也可能出现功能重复。

2. OSI、TCP/IP 与五层结构

OSI 七层模型在概念上完整清晰，但复杂且缺乏市场成功。TCP/IP 体系结构则在实践中获得广泛采用，通常划分为应用层、传输层、网络层和网络接口层。为了兼顾教学上的清晰性与工程上的现实性，常采用五层结构：应用层、传输层、网络层、数据链路层和物理层。


graph TD

    A[应用层] --> B[传输层]

    B --> C[网络层]

    C --> D[数据链路层]

    D --> E[物理层]

五层结构中，各层解决的问题如下。

应用层面向具体网络应用，定义应用进程之间交换的信息格式与交互语义。

传输层在主机之间提供进程到进程的通信能力，负责复用与分用、可靠性、流量控制或无连接传输等。

网络层负责把分组从源主机送达目的主机，关键问题是逻辑寻址与路由转发。

数据链路层负责相邻结点之间的帧传输，处理成帧、链路访问控制、差错检测等问题。

物理层负责在物理媒体上发送和接收比特流。

3. 协议、服务与层间关系

协议是对等层实体之间通信时必须遵守的规则集合，其核心包括三部分：语法、语义和同步。

语法规定数据与控制信息的格式；语义规定在何种条件下发送何种控制信息以及收到后应作何响应；同步规定事件发生的顺序。

服务则是某层通过层间接口向上一层提供的能力。协议是水平概念，服务是垂直概念。上层看到的是下层提供的服务，而不直接接触其内部协议细节。

服务访问点 SAP 是相邻两层交互的逻辑接口。服务数据单元 SDU 是层间交换的数据单位；协议数据单元 PDU 是某层协议处理后的数据单位。一个 SDU 可以被封装成一个或多个 PDU，也可以由多个 SDU 合并构成一个 PDU。

4. 封装与解封装

分层体系最典型的工作过程是封装与解封装。发送端中，上层数据逐层向下传递，每层在前面添加本层首部；到达数据链路层时再添加链路层首部和尾部；物理层则把整个帧转化为比特流发往媒体。接收端则执行反向操作，逐层剥去首部和尾部，还原出应用数据。

    A[应用数据] --> B[应用层 PDU]

    J --> K[接收端应用层获得应用数据]

5. 各层的典型功能

分层体系中的常见功能包括：

差错控制，用于提高通信可靠性；

流量控制，用于抑制发送端速率，防止接收端处理不过来；

分段与重装，用于将较大数据块拆分为更小单元并在接收端恢复；

复用与分用，用于让多个高层会话共享一个较低层的传输通道；

连接建立与释放，用于在交换数据前后管理逻辑连接状态。

6. 分层与现实互联网的关系

OSI 是法律意义上的国际标准，但 TCP/IP 成为事实上的国际标准。互联网中的大量应用运行在 TCP 或 UDP 之上，由 IP 提供统一的网络层服务，而 IP 又能够运行在多种底层网络之上。因此 TCP/IP 协议族形成了典型的 ” 沙漏型结构 “：下层网络技术多样，上层应用多样，中间由 IP 作为统一窄腰连接起来。


graph TD

    A1[HTTP]

    A2[DNS]

    A3[SMTP]

    A4[RTP]

    A1 --> IP[IP]

    A2 --> IP

    A3 --> IP

    A4 --> IP

    IP --> N1[以太网]

    IP --> N2[WiFi]

    IP --> N3[蜂窝网络]

    IP --> N4[其他链路技术]

在路由器中，处理通常止于网络层。路由器需要理解分组首部、执行查表和转发，但不参与端系统上的应用语义和端到端传输语义。

7. 协议设计的边界：蓝军问题

协议设计必须考虑异常情况，而不能只在理想条件下成立。蓝军问题说明：在消息和确认都可能丢失的环境中，无法通过有限轮确认机制让双方在逻辑上达到绝对确定的共同知识。因此，现实网络协议通常追求的是 ” 在概率意义上足够可靠 “，而不是在不可靠信道上获得绝对的、无条件的确定性。

四、计算机网络的性能指标

1. 速率、带宽与吞吐量

速率是单位时间内传送的比特数，也称数据率或比特率，单位通常为 bit/s、Mbit/s 或 Gbit/s。它表示发送能力，常见于链路额定速率或协议标称速率。

带宽有两种常见含义。传统通信中，带宽表示信号频带宽度，单位是 Hz；在计算机网络中，带宽常表示信道单位时间内可通过的最高数据率，单位是 bit/s。二者的联系在于：频域带宽越宽，通常可支持的最高数据率越高。

吞吐量则是实际测得的单位时间内成功通过某个网络、链路或接口的数据量。它是运行结果，而不是理论上限，因此通常不超过链路带宽。

2. 时延及其组成

时延是数据从网络一端到另一端所需的时间。端到端时延通常由发送时延、传播时延、处理时延和排队时延组成。

发送时延是把一个分组的所有比特推入链路所需的时间。若分组长度为 L bit，发送速率为 R bit/s，则

dtrans​=RL​

传播时延是信号沿介质传播的时间。若链路长度为 m，信号传播速率为 v，则

dprop​=vm​

处理时延是主机或路由器检查首部、执行差错检测、查找转发表等操作所需的时间。

排队时延是分组在输入队列或输出队列中等待服务的时间，受当前负载影响最大。

因此，总时延可表示为

dtotal​=dtrans​+dprop​+dproc​+dqueue​ 

flowchart LR

    A[发送主机] -->|发送时延| B[链路]

    B -->|传播时延| C[路由器]

    C -->|处理时延| D[队列]

    D -->|排队时延| E[下一跳]

发送时延与传播时延必须严格区分。提高链路带宽会减小发送时延，但不会改变电磁波在介质中的传播速度，因此不会直接减小传播时延。

3. 存储转发与多跳时延

在分组交换网络中，路由器通常需要完整接收分组后才能转发到下一跳。若忽略传播时延且链路速率相同，则一个长度为 L 的分组经过两跳链路的最小发送时延近似为

2RL​

更一般地，若经过 n 跳且每跳发送速率均为 R，则发送部分的累计时延为

nRL​

这说明多跳路径下，存储转发会使时延随着跳数增长。

4. 时延带宽积

时延带宽积表示一条链路在传播过程中 ” 容纳 ” 了多少比特，其定义为

BDP=dprop​×R

它反映了链路中正在传播但尚未到达接收端的比特总量，也可理解为 ” 以比特为单位的链路长度 “。时延带宽积越大，链路越像一条能容纳更多比特的长管道。

5. 往返时间 RTT

往返时间 RTT 是从发送方开始发送数据到收到接收方确认所经历的总时间。它不仅包含双向传播时延，还包含中间设备的发送、处理与排队时延。对于交互式协议和拥塞控制而言，RTT 是非常关键的指标。

6. 利用率

信道利用率是某条信道被有效使用的时间比例；网络利用率则是全网利用率的加权结果。利用率并非越高越好，因为当利用率逼近 1 时，排队时延会急剧上升。

在理想化排队模型下，若网络空闲时的时延为 D0​，当前利用率为 U，则当前时延常近似表示为

D=1−UD0​​

该公式表明，当 U 接近 1 时，时延会快速发散。

7. 其他非性能特征

除上述性能指标外，网络还需要关注费用、质量、标准化程度、可靠性、可扩展性、可升级性以及可管理性和可维护性。这些特征决定了网络能否从实验系统成长为可长期运行的工程系统。

8. 典型计算题的分析框架

分析网络题目时，应先分清使用的是电路交换还是分组交换，再分别识别瓶颈速率、传播时延、是否存在建立连接开销、是否采用存储转发以及分组长度是否改变了流水化效果。对分组交换而言，首个分组的到达时间和后续分组的稳态到达节奏往往不同，二者必须分开处理。

五、计算机网络的分类

1. 按作用范围分类

按覆盖范围可分为个人区域网 PAN、局域网 LAN、城域网 MAN 和广域网 WAN。

PAN 的范围通常在数米到十米量级，常见于蓝牙等近距离互联。

LAN 的范围通常局限于房间、建筑或校园，典型技术包括以太网和 WiFi。

MAN 通常覆盖城市范围，常见于城域接入和运营商网络。

WAN 覆盖范围更大，可跨地区、跨国家甚至跨洲际，互联网本身就是最大的广域互连系统。

2. 按使用者分类

按使用者可以分为公用网和专用网。公用网面向公众提供服务，只要满足接入条件即可使用；专用网则为特定组织、特定业务或特定安全要求而构建。两者都可以传送计算机数据，只是服务对象和管理边界不同。

3. 按功能位置分类

从功能位置看，还可以单独把接入网作为一类特殊网络。它专门承担用户接入互联网的职责，其设计目标与核心网和纯终端网络不同。

六、计算机网络的历史

互联网的技术起点通常追溯到 20 世纪 60 年代的分组交换思想。Kleinrock 的排队理论、Baran 的分布式通信构想以及 ARPANET 的实践共同奠定了现代互联网的基础。

1969 年 ARPANET 以少量结点启动，随后规模逐步扩大。1983 年 TCP/IP 成为其标准协议，异构网络互连获得统一基础。20 世纪 90 年代，ARPANET 和 NSFNET 相继退出历史舞台，互联网走向商业化；WWW 的普及则进一步推动了全球互联网爆发式增长。

中国的互联网发展起步于 20 世纪 80 年代的联网实验。1994 年我国正式连入国际互联网，随后 CHINANET、CERNET、CSTNET 等全国性网络相继发展，形成了具有教育科研、运营商和公众服务等多种类型的互联网基础设施。

七、计算机网络的标准

1. 为什么标准化重要

网络是典型的互联系统。若不同厂商、不同地区、不同层次的设备和协议不能互操作，就无法形成真正的大规模网络。因此，标准化的核心目标不是抽象上的统一，而是实际互操作性。

2. 主要标准组织

在国际电信领域，ITU 负责电信相关标准与协调工作。

ISO 是重要的国际标准化组织，曾主导 OSI 参考模型。

IEEE 在局域网和无线网络技术中具有重要地位，例如 IEEE 802.3 和 IEEE 802.11。

互联网标准化则主要围绕 ISOC、IAB、IRTF 和 IETF 展开，其中 IETF 通过工作组推进具体协议设计与演进。


graph TD

    ISOC[ISOC] --> IAB[IAB]

    ISOC --> IETF[IETF]

    ISOC --> IRTF[IRTF]

    IETF --> WG[工作组 WG]

    IRTF --> RG[研究组 RG]

3. RFC 体系

互联网标准文档以 RFC 形式发布。RFC 不全都是正式标准，但它们构成了互联网技术文献与标准化过程的基本载体。典型标准化轨迹包括从互联网草案到建议标准，再到互联网标准。与此同时，还存在实验性、历史性和信息性 RFC。

RFC 体系体现了互联网标准的开放性：协议往往在实际部署和持续修订中逐步成熟，而不是在封闭环境中一次性定型。

4. 新标准的演进示例：Multipath TCP

Multipath TCP 试图解决多接口终端和多路径网络环境中的带宽利用与可靠性问题。若终端同时具备 WiFi 与蜂窝网络接口，传统 TCP 往往只能在单一路径上工作；MPTCP 则允许一个逻辑连接在多条路径上并行传输，从而提高吞吐能力、增强容错性，并根据路径拥塞情况动态调度数据。

这表明网络标准不是静止不变的，它会随着接入环境、应用需求与系统架构的变化持续演化。

八、计算机网络的安全

1. 安全问题的根源

互联网早期建立在开放互联和用户相互信任的背景之上，因此对安全问题考虑不足。随着网络规模扩大、参与者多样化和攻击动机增强，安全已经成为各层协议设计中不可回避的问题。

网络安全关注三个基本问题：攻击者如何破坏网络；系统如何防御攻击；体系结构如何演进以提升安全性。

2. 典型攻击形式

（1）嗅探

在共享介质或无线网络中，攻击者可通过混杂模式捕获经过本机的数据帧，从而窃取口令、会话信息或其他敏感数据。应对方式的核心是加密，使被截获的数据无法直接被理解。


graph LR

    A[主机 A] -->|发送分组| B[接入介质]

    B --> C[主机 B]

    B -. 被动复制 .-> E[嗅探者]

（2）地址伪造

攻击者可伪造源地址，冒充受信任主机发送分组。这类攻击利用了网络层只按目的地址进行转发而不天然验证源身份的特点。应对的关键是认证与源地址验证。

（3）恶意软件与僵尸网络

病毒、蠕虫等恶意程序能够感染主机、窃取信息、远程控制设备，并将受害主机构造成僵尸网络。大量受控主机可进一步被用于发起分布式拒绝服务攻击。

（4）DoS 与 DDoS

拒绝服务攻击通过制造大量虚假流量占用服务器、链路或中间设备资源，使正常用户无法获得服务。分布式拒绝服务攻击利用多个攻击源同时发起攻击，具有流量规模大、溯源困难和防御成本高等特点。

    Z2[受控主机2] --> T

    Z3[受控主机3] --> T

    Z4[受控主机4] --> T

    C2[攻击控制者] --> Z1

    C2 --> Z2

    C2 --> Z3

    C2 --> Z4

3. 安全设计的基本思路

安全不能依赖单点补丁，而应在多个层次协同实现。典型思路包括：通过加密保护机密性，通过认证保证身份可信，通过完整性校验防止篡改，通过访问控制限制权限边界，通过监测与审计发现异常，通过协议和架构演化减少可利用面。

从工程角度看，安全机制常常需要与性能、兼容性、可部署性之间进行平衡，因此网络安全始终是体系结构设计的重要组成部分，而不是可后加的附属功能。

九、本讲知识主线总结

第一，计算机网络的本质是以协议为纽带、以可编程硬件为载体、以资源共享和连通性为目标的互联系统。

第二，互联网是当前最成功的计算机网络实现形式，其结构可概括为边缘部分、接入网和核心部分三者协同。

第三，端系统通信真正服务的是进程之间的交互；核心网的关键机制则是基于路由器的分组交换与存储转发。

第四，分层体系结构是管理网络复杂性的根本方法。现实互联网以 TCP/IP 为事实标准，并常用五层结构进行理解。

第五，网络性能分析必须区分速率、带宽、吞吐量和多种时延，并理解统计复用、RTT、时延带宽积和利用率之间的关系。

第六，网络不是纯技术堆叠，而是历史演进、标准化协作和安全对抗共同塑造的开放工程系统。
