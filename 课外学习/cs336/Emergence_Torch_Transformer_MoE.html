Lecture1

随着模型参数规模的急剧增大（从 7.6 亿 到 1750 亿），前馈神经网络（FFN） 逐渐取代注意力机制（Attention）成为绝对的算力消耗主力（计算占比从 44% 飙升至约 77%），而 Attention 的计算占比反而显著下降（跌至 17%）；这说明在超大模型训练中，单纯优化 Attention 带来的整体收益会被巨大的 FFN 开销稀释，FFN 才是大模型真正的计算瓶颈。

涌现

  1. 性能的 ” 顿悟式 ” 飞跃（The “Hockey Stick” Curve）

这是涌现最直观的视觉表现。

  • 潜伏期： 在模型规模（参数量或计算量）达到某个临界点之前，模型在特定任务上的表现极度平坦，几乎徘徊在随机猜测（Random Guessing）的水平，没有任何进步迹象。
  • 突变期： 一旦跨过这个特定的临界值，模型的能力曲线会突然近乎垂直地拉升。它不是从 10 分 涨到 20 分，而是直接从 0 分 跳变到 60 分 甚至更高。
  1. 小模型的 ” 不可外推性 “（Unpredictability）

你无法通过观察小模型的表现来预测大模型是否会拥有某项能力。

  • 如果我们观察一个 10 亿 参数的模型（表现为 0）和一个 50 亿 参数的模型（表现仍为 0），线性外推的结论是 ” 这个任务太难，模型学不会 ”。
  • 但现实是，到了 100 亿 参数时，能力可能突然出现。这种表现打破了传统的 scaling law（在简单任务上性能随规模平滑增长的规律）。
  1. 训练损失与任务能力的 ” 割裂 “（Loss vs. Capability Divergence）

这是涌现现象中一个非常反直觉的观测结果：

  • 微观层面： 模型的训练损失（Training Loss） 是随着训练过程平滑、连续下降的（一直在变好）。
  • 宏观层面： 模型在下游复杂任务（如算术、逻辑推理）上的准确率 却是阶跃式变化的（长时间不变，然后突然变好）。
  • 表现： 这意味着模型在 ” 预测下一个词更准 ” 这一指标上的平滑进步，在很长一段时间内完全无法转化为解决具体问题的能力，直到某一刻突然转化成功。
  1. 复杂任务的 ” 全有或全无 “（All-or-Nothing）

涌现通常只出现在多步骤、高复杂度的任务中（如思维链推理、代码生成、三位数乘法）。

  • 在这些任务中，模型要么完全不会（输出胡言乱语），要么就能做对。
  • 很少观察到 ” 做对了一半 ” 或者 ” 逐渐接近正确答案 ” 的中间状态。表现形式上呈现出一种极端的二元对立。

Tokenizer（分词器） 是所有现代自然语言处理（NLP）和大型语言模型（LLM，如 GPT、Llama）的第一道门槛。

简单来说，它的作用是将人类能读懂的 ” 文本 “，转换成机器能读懂的 ” 数字列表 “。

1. 核心概念：为什么需要 Tokenizer？

计算机本质上是一个数学计算器，它无法直接处理 “Apple” 这个单词，它只能处理数字（向量/矩阵）。

Tokenizer 的工作就是建立一个映射关系（Vocabulary/词表）：

  1. 输入："I love AI"
  2. 切分：["I", "love", "AI"]
  3. 映射：[23, 456, 1209] （这些数字就是 Input IDs）

之后，模型（Model）会拿着这些数字去查表（Embedding Table），将其转化为高维向量进行计算。

关键点：Tokenizer 决定了模型 ” 看 ” 世界的颗粒度。

2. 三种主流的分词粒度

Tokenizer 如何把一句话切开？这是分词器设计的核心难题。主要经历了三个阶段的演变：

A. 词粒度 (Word-based)

最直观的方法，按空格或标点切分。

  • 例子："unhappily" ["unhappily"]
  • 优点：保留了完整的语义。
  • 缺点：
    • 词表爆炸：英语有数十万单词，如果都要存，词表会巨大。
    • OOV 问题 (Out of Vocabulary)：如果训练时见过 “cat”，没见过 “cats”，模型就会把 “cats” 标记为 <UNK> (Unknown)，导致信息丢失。
B. 字符粒度 (Character-based)

把每个字母或汉字当做一个 Token。

  • 例子："unhappily" ["u", "n", "h", "a", "p", "p", "i", "l", "y"]
  • 优点：词表很小（英语只要 26 字母 + 符号），几乎不会有 OOV。
  • 缺点：
    • 序列过长：一句话变得非常长，大大增加了模型的计算负担（注意：LLM 的上下文窗口是有限的）。
    • 语义缺失：单个字母 "h" 通常没有独立含义，模型需要花费更多层数去学习组合意义。
C. 子词粒度 (Subword-based) —— 当前主流

这是目前 GPT、BERT 等模型都在使用的方案。它在 ” 词 ” 和 ” 字 ” 之间寻找平衡：常用词保持完整，生僻词拆分为词根。

  • 例子："unhappily" ["un", "happi", "ly"]
  • 原理：
    • "un" (前缀，表示否定)
    • "happi" (词根)
    • "ly" (后缀，表示副词)
  • 优点：既控制了词表大小（通常 3 万~10 万），又解决了生僻词问题（通过组合认识的词根来理解新词），且序列长度适中。

3. 主流的 Subword 算法

你可能会经常听到以下几个术语，它们都是实现 ” 子词切分 ” 的具体算法：

算法名称简介代表模型
BPE (Byte Pair Encoding)统计频次，不断合并出现频率最高的相邻字符对。GPT-2, GPT-3, GPT-4, LLaMA
WordPiece类似 BPE，但在合并时考虑了语言模型的概率提升，而非仅仅是频次。BERT, DistilBERT
SentencePieceGoogle 推出的一个库，把句子视为原始字节流（Raw Stream），无需预先按空格分词，对中文/日文等不使用空格的语言非常友好。T5, LLaMA (实现层面)

GPT-4 的 BPE 特性：
现在的 LLM 通常使用 Byte-level BPE。它们不是处理 Unicode 字符，而是处理 UTF-8 字节。这意味着它们可以处理任何语言，甚至是 Emoji 表情符号或代码中的特殊符号，而不会出现乱码。

4. 实际工作流程：从文本到 ID

当你在 Python 中调用 tokenizer 时，内部发生了这四步：

  1. Normalization (标准化)：

    • 清理文本，例如去除多余空格，统一转小写（如果是 uncased 模型），处理 Unicode 规范化（NFC/NFD）。

  2. Pre-tokenization (预切分)：

    • 根据空格或标点符号，把句子粗略切分成 ” 单词块 ”。
    • 中文特殊性：中文没有空格，通常需要专门的切分逻辑或直接使用 SentencePiece。

  3. Model Encoding (模型切分)：

    • 应用 BPE 或 WordPiece 算法，把单词块切分成最终的 Token（子词）。
    • 例："Transforming" ["Transform", "ing"]

  4. Mapping (映射)：

    • 查阅词表（Vocabulary），将 Token 转换为 ID。
    • 例：["Transform", "ing"] [10938, 283]

5. 常见误区与特别注意

  1. Tokenizer != Embedding

    • Tokenizer：把文本变成整数 ID（比如 50256）。
    • Embedding：模型的第一层，把整数 ID 变成一个 1024 维的浮点数向量。

  2. Token 数量 != 单词数量

    • 在英语中，1000 个单词大约等于 1300 个 Token（经验值：1 Token 0.75 单词）。
    • 在中文中，现在的模型（如 Qwen, Yi）优化较好，通常 1 个汉字对应 1~1.5 个 Token，但在老模型（如 GPT-3）中，一个汉字可能被拆成 2-3 个 Token，非常费钱且占用上下文窗口。

  3. 特殊 Token

    • 词表中通常包含一些功能性 Token，例如：
    • [CLS] / <s>：句子开始。
    • [SEP] / </s>：句子结束。
    • [PAD]：填充位（因为矩阵运算要求长度一致）。
    • [MASK]：用于掩码语言模型训练。

个人的问题

命题一：分词粒度与语义表征的非对齐性 (Tokenization-Semantic Misalignment)
1. 表述

假设: 设人类认知空间中的语义概念集合为 C，自然语言的文本空间为 S。定义分词函数 T:S→Zn，其中 Zn 为离散的 Token ID 序列空间。若 T 的切分边界与 C 中概念的形态学边界（Morphological Boundaries）存在显著的非同态性（Non-homomorphism），即 ∃s∈S，使得 s 对应的单一概念 c∈C 被切分为无独立语义的子序列 ⟨t1​,t2​,…,tk​⟩。

推论: 这种切分会导致模型 M 在潜在空间（Latent Space）构建概念表征时引入额外的计算开销（Computational Overhead），并导致对长程依赖（Long-term Dependency）和特定领域（如算术、逻辑）的理解产生偏差（Bias）。

2. 解答

这一问题揭示了当前 LLM 架构中**” 统计压缩 ” 与 ” 语义完整性 ” 之间的根本矛盾**。

  • 表征碎片化 (Representation Fragmentation)：

    在机械可解释性研究中，我们通常寻找对应特定概念的 ” 特征方向 “（Feature Direction）。如果分词器将一个概念（如 “unbelievable”）切分为 [“un”, “believ”, “able”]，模型的第一层 Embedding 无法直接获得 ” 难以置信 ” 这个完整语义。

    • 后果：Transformer 的前几层（Layer 0-2）必须充当 ” 软性分词器 “，利用注意力机制（Attention Heads）将分散的 Token 聚合（Aggregate）成一个完整的语义单元。这浪费了模型的深度和参数容量。

  • 归纳偏置错配 (Inductive Bias Mismatch)：

    人类阅读是基于 ” 语素 “（Morpheme）的，而 BPE 算法是基于 ” 频率 “（Frequency）的。

    • 算术灾难：这是最典型的例子。对于十进制数 1000，BPE 可能切分为 [10, 00]，而 1001 切分为 [100, 1]。这种切分破坏了数字的位值系统（Place-value System）。模型难以通过简单的线性变换学习 1000+1=1001，而必须记忆大量的非线性映射。这也是为什么 LLM 在做大数加减法时表现不佳的根本原因之一——它们在 ” 看 ” 数字的方式上是 ” 失读症 ” 患者。
命题二：分词映射的双射性与可逆性 (Bijectivity And Reversibility of Tokenization)
1. 表述

假设: 定义分词过程为函数 Encode E:S→V∗（其中 S 为所有可能的字符串集合，V∗ 为词表中 Token ID 的所有有限序列集合）。定义解码过程为函数 Decode D:V∗→S。

性质： 分词系统应当构成一个双射（Bijection），即满足严格的可逆性：

∀s∈S,D(E(s))≡s

这意味着文本空间 S 与 序列空间 V∗ 是同构的（Isomorphic）。

2. 解答

在数学理想状态下，这个定义是完美的。但在工程实践中，这个 ” 双射 ” 通常是有损的（Lossy）或有条件的（Conditional）（Conditional）（Conditional）（Conditional）或。

  • 规范化破坏了单射性 (Normalization breaks Injectivity)：

    实际上，Tokenizer 通常包含一个预处理步骤 P（Normalization），例如去除多余空格、Unicode 规范化（NFC/NFD）或转小写。

E(s) = \text{Model}(P(s))

若 $P(s_1) = P(s_2)$ 且 $s_1 \neq s_2$（例如 $s_1=$"Latte"，$s_2=$"latte" 在某些 uncased 模型中），则 $E(s_1) = E(s_2)$。此时，$D(E(s_1)) = s_2 \neq s_1$。 **结论：** 它是满射（Surjective）但非单射（Injective）。这意味着你无法总是无损还原原始文本的格式信息。 - Byte-level BPE 的修正 (Restoring Bijectivity)：为了逼近你提到的“完美双射”，GPT-4 和 LLaMA 等现代模型采用了 Byte-level BPE。它们将 $\mathcal{S}$ 定义为 UTF-8 字节流 而非 Unicode 字符流。 - 在此定义下，只要输入是合法的 UTF-8 字节序列，现在的 Tokenizer 确实极其接近双射函数。即使是未见过的字符（OOV），也会回退到字节编码，从而保证信息不丢失。 - 从解释性角度看“双射”的意义：如果 $E$ 是双射，我们可以安全地假设 Token 序列 $\mathbf{z}$ 包含了输入 $s$ 的全部信息（Total Information）。这对于研究 LLM 内部机制至关重要：如果我们发现模型表现出某种偏见，我们可以确定这种偏见不是因为信息在分词阶段丢失了（例如把生僻字变成了 <UNK>），而是模型在处理这些 Token 时产生的内部动态。 ## Byte-level Bpe ### 核心产物文件 BPE 训练完成后，会生成两个关键文件，它们共同构成了分词器（Tokenizer）的“大脑”。 #### tokenizer_vocab.json`（词汇表） - 找到最佳 Pair 并合并。 - **复杂度**：$O(\text{Dataset\_Size} + \text{Iterations} \times \text{Unique\_Words})$。 - **适用场景**：绝大多数 NLP 任务（GPT-2, LLaMA 等）。 ------ #### 详细对比分析 | **维度** | **我的思路 (分布式迭代)** | **ai的思路 (预分词 + 单线程迭代)** | | --------------- | ------------------------------------------------------------ | ------------------------------------------------------------ | | **数据冗余** | **极高**。单词 "the" 在语料中出现 100 万次，你的 Worker 就要扫描它 100 万次。 | **无**。单词 "the" 被压缩为字典中的 1 条记录，权重为 1,000,000。只扫描 1 次。 | | **Python 开销** | **致命**。每轮迭代涉及进程间通信 (Pickling/IPC)，Python 的多进程启动和数据传输开销巨大。 | **低**。多进程仅在第一步启动一次。后续迭代完全在内存中进行，无 IPC 开销。 | | **I/O 操作** | **高**。如果在内存中维护大文件副本，内存压力大；如果读磁盘，I/O 瓶颈严重。 | **极低**。只读取一次文件，后续全在内存操作小字典。 | | **实现难度** | **高**。需要处理复杂的进程同步、状态广播。 | **中**。利用 `collections.Counter` 即可轻松实现。 | | **作业合规性** | **不推荐**。作业明确指出“directly merging bytes across the corpus is computationally expensive” 。 | **推荐**。符合作业流程：Pre-tokenization $\rightarrow$ Compute Merges 。 | ------ #### 最终代码策略 基于上述分析，我们采用**策略二**。以下是实现该策略的详细步骤，涵盖了我们之前讨论的所有优化点。 ##### 阶段一：并行预分词 (Parallel Pre-tokenization) **目标**：将巨大的文本文件转换为一个紧凑的 `Dict[tuple[int], int]`。 1. **文件分块 (Chunking)**： - 使用 `find_chunk_boundaries` 确定每个进程读取的 `start` 和 `end` 字节位置。 - **关键点**：确保边界对齐到特殊 Token 或换行符，防止切断单词。 2. **Worker 处理 (Map)**： - **读取**：`file.seek(start)` -> `file.read(size)`。 - **正则切分**：使用 GPT-2 正则 `r"""'(?:[sdmt]|ll|ve|re)| ?\p{L}+|..."""` 。 - *解释*：这能将 "dog." 切分为 "dog" 和 "."，防止 "dog." 被视为一个新词。 - **字节化**：将切分出的 `str` 编码为 `bytes`，再转为 `tuple[int]`（作为字典 Key）。 - **统计**：使用 `collections.Counter` 统计局部词频。 3. **汇总 (Reduce)**： - 主进程收集所有 Worker 的 Counter，使用 `update()` 方法合并为全局 `vocab_counts`。 ##### 阶段二：BPE 迭代训练 **目标**：在压缩后的字典上生成合并规则。 1. **初始化**： - `merges = []` - `vocab_map = {0..255: b'\x00'..b'\xff'}` (用于记录 ID 对应的真实字节) 2. **循环 (Target Merges 次)**： - **统计 Pair**：遍历 `vocab_counts`，计算所有相邻 Pair 的加权频率。 - *公式*：`pair_count[p] += word_count` - **寻找最佳**：`max(pairs, key=...)`。 - *Tie-breaking*：频率相同时，按字节序（Lexicographical）决定 。 - **记录**：`merges.append((byte_a, byte_b))`。 - **更新字典**：使用 `merge_vocab` 函数更新 `vocab_counts`。 - *优化*：使用 `if p0 not in ids: continue` 进行快速跳过（Fast Fail）。 ##### 阶段三：构建产物 **目标**：生成作业要求的最终文件。 1. **构建词汇表 (`tokenizer_vocab.json`)**： - **Optimized Approach (零拷贝做法)**：在 Rust 函数中使用引用（Reference）或视图（View）。 - `fn process(text: &str)` 或 `fn process(text: &[u8])` - **后果**：Rust 直接读取 Python 内存中的字节，**不进行复制**。 ------ #### 如何使用 PyO3 ##### 环境准备 Tensor (张量) PyTorch 的核心数据结构，本质是一个多维数组。Tensor 不是矩阵，而是指向内存的“带步长（Stride）”的指针。 - **Storage (存储)**: 实际保存数据的连续内存块（一维数组）。 - **View (视图)**: 只是对 Storage 的一种“解释”或“窗口”（包含形状 `shape`、步长 `stride` 等元数据）。 - **关键点**: 改变 View 的形状（如 `view()` 或 `reshape()`）通常**不复制**数据，这非常高效；但如果你不小心复制了数据（如 `.contiguous()`），显存可能会翻倍。 - *解释*: 想象 `Storage` 是一长条面团，`View` 是模具。你可以用不同模具看面团，但面团本身没变。 #### 存储机制 (The Pointer Model) - **Storage (本体)**：一维的、连续的内存块（Raw Memory）。 - **Metadata (元数据)**：描述如何从 Storage 中“解释”出多维形状。核心是 `stride`。 - `stride(0)`：行索引 +1 时，在内存中跳过多少个元素。 - `stride(1)`：列索引 +1 时，在内存中跳过多少个元素。 - 寻址公式： 对于 Tensor x，逻辑坐标 (r, c) 对应的物理内存索引是：$\text{offset} = r \times \text{stride}(0) + c \times \text{stride}(1)$ #### View Vs Copy (切片与视图) - **Views are Free**：`x[0]`, `x[:, 1]`, `x.view()`, `x.transpose()` 等操作**不复制数据**，只是创建了一个新的 Metadata 对象指向同一块 Storage。==相当于指针== - **后果**：修改 `y` 会同时也修改 `x`（这是副作用，也是特性）。 - **断言**：`same_storage(x, y)` 为 True。 #### Contiguous (连续性陷阱) 这是新手最容易遇到的 RuntimeError。 - **现象**：当你对矩阵做转置 `y = x.t()` 后，虽然逻辑上它是 (N, M)，但物理内存中它是跳跃访问的。此时 `y.is_contiguous()` 为 False。 - **报错**：对非连续 Tensor 调用 `.view()` 会报错，因为 `view` 假设物理内存是连续的。 - **修复**：`y.contiguous()`。这会强制发生 **Deep Copy**，重新排列物理内存，使其连续。代价是显存翻倍和计算时间。 ### 计算操作 #### Element-wise (逐元素) - **特性**：输入输出形状相同，可并行化程度极高。 - **示例**：`x + x`, `x * 2`, `x.sqrt()`。 - **实用算子**：`triu()` (Triangular Upper)。用于生成 Transformer 中的 **Causal Mask**（因果掩码），将矩阵左下角置 2. 初始化危机

为什么不能随便用 torch.randn 初始化？

  • 方差爆炸：如果输入 x 维度很大（例如 16384），简单的矩阵乘法会将数值的方差放大 D​ 倍。
  • 后果：层数越深，数值越大，导致梯度爆炸（NaN）或消失（0）。

3. 解决方案：Xavier / Kaiming Init

  • 原理：为了保持每一层输出的方差不变（Unit Variance），权重的初始方差应为 1/D。
  • 操作：w = randn(D, K) / sqrt(D)。
  • 截断正态分布 (Truncated Normal)：代码中使用了 trunc_normal_，截断范围 [-3, 3]。这是为了防止极端离群值破坏训练初期的稳定性。

内存优化

在 GPU 训练中，计算速度往往非常快，导致 数据传输 (CPU → GPU) 容易成为瓶颈。

1.1 机制解析
  • 默认行为 (Paged Memory)：普通的 CPU Tensors 存储在分页内存（Paged Memory）中。操作系统可能会根据内存压力将这些数据交换（Swap）到硬盘上。
  • 锁页内存 (Pinned Memory)：这是一种特殊的内存分配方式。操作系统保证这块内存永远不会被换出到硬盘，并且物理地址是连续的。
1.2 为什么快？

GPU 无法直接从分页内存读取数据（因为它不知道数据物理地址在哪里，或者数据可能被换出了）。

  1. 普通传输：CPU Paged → CPU Pinned (临时缓冲) → GPU。需要两次拷贝。
  2. 锁页传输：CPU Pinned → GPU。利用 DMA (Direct Memory Access) 技术，直接通过 PCIe 总线传输，无需 CPU 参与，且只需一次拷贝。
1.3 异步流水线 (Asynchronous Pipeline)

通过 non_blocking=True，我们可以实现计算与传输的重叠（Overlap）。

  • 同步 (默认)：CPU 搬运数据 → GPU 等待 → GPU 计算 → CPU 等待 → CPU 搬运…
  • 异步 (优化)：
    • 当 GPU 正在计算 Batch N 时。
    • CPU 同时在搬运 Batch N+1 到显存（DMA 操作，不占 CPU 算力）。
    • CPU 同时在读取 Batch N+2 从硬盘到内存。

随机性与可复现性

深度学习中的 ” 蝴蝶效应 ” 非常显著。参数初始化的微小差异、数据洗牌顺序的不同、Dropout 的随机丢弃，都会导致最终模型性能天差地别。 为了 Debug 或对比实验结果，必须控制随机种子。

三层随机源

你需要同时锁定三个层面的随机性，缺一不可：

  1. PyTorch 层面：控制权重初始化 (nn.init)、Dropout、数据加载器的 shuffle 等。
  2. NumPy 层面：如果你在数据预处理中用了 np.random（如数据增强）。

\text{Time (s)} = \frac{\text{Work (FLOPs)}}{\text{Speed (FLOP/s)}}

#### 计算量 -FLOPs - **NVIDIA A100**: 任何一次加法或乘法都算一个 FLOP(浮点运算)。 - **NVIDIA H100**: - 假设矩阵 $A$ 大小为 $(M, K)$，矩阵 $B$ 大小为 $(K, N)$。 - 计算 $C = A \times B$ (结果大小 $M, N$)。 - **FLOPs 定义**: $\text{FLOPs} \approx 2 \times M \times N \times K$ - *解释*: 结果矩阵有 $M \times N$ 个元素，每个元素是通过 $K$ 次乘法和 $K$ 次加法（共 $2K$ 次运算）得到的。 - *重要性*: Transformer 中全是矩阵乘法，记住 $2MNK$ 这个公式就能估算 90% 的计算量。 #### 计算速度 -FLOP/s - 定义: $\text{FLOPs} / \text{Memory Access (Bytes)}$。 - **矩阵乘法 (MatMul) 的 FLOPs**: 算术强度高（如大矩阵乘法），GPU 拼命算，显存带宽够用。 - **公式**: 算术强度低（如 Element-wise 操作，ReLU, LayerNorm），GPU 算得快，但在这等数据从显存传过来。 - *应用*: FlashAttention 的核心思想就是减少显存访问，把 Memory Bound 变成 Compute Bound。 ### 算力核算 (Compute Accounting) **Compute Bound (计算受限)** #### 矩阵乘法 (GEMM) 所有深度学习计算的原子单位都是矩阵乘法：$C = A \times B$。 - **Memory Bound (带宽受限)**：设 $A: [M, K]$，$B: [K, N]$，则 $C: [M, N]$。 - FLOPs (Floating Point Operations)：

\text{FLOPs} \approx 2 \times M \times N \times K

- **Why $2$?** - 计算输出矩阵的一个元素，需要 $K$ 次乘法 + $K$ 次加法（MAC, Multiply-Accumulate）。 ### 宏观估算：Transformer 训练总算力 这是本课最重要的公式之一，用于估算整个训练周期的计算量。 - 公式：

C_{\text{train}} \approx 6 \times P \times D

- $P$ (Parameters): 模型参数量。 - $D$ (Dataset size): 训练数据的 Token 总数。 - 推导逻辑（Why 6?）： 对于每个 Token，我们需要经历三个阶段： 1. **Forward Pass**: 约为 $2P$ FLOPs（一次矩阵乘法 $2 \times$ 权重矩阵大小）。 2. **Backward (Activation Gradient)**: 计算对输入 $X$ 的梯度，约为 $2P$ FLOPs。 3. **Backward (Weight Gradient)**: 计算对权重 $W$ 的梯度，约为 $2P$ FLOPs。 - Total = $2P + 2P + 2P = 6P$。 ### 时间估算公式 有了总算力需求，结合硬件性能，即可算出时间。 - 公式：

\text{Time (sec)} = \frac{6 \times P \times D}{N_{\text{GPU}} \times \text{Peak_FLOPs} \times \text{MFU}}

- **参数详解**： - $N_{\text{GPU}}$: 卡数。 - **Peak_FLOPs**: 硬件理论峰值（查 H100 白皮书，注意精度是 FP16/BF16 Tensor Core）。 - **MFU (Model FLOPs Utilization)**: 模型算力利用率。这是衡量你代码写得好坏的指标。 - **经验值**：不做优化 $\approx 30\%$；一般优化 $\approx 40-50\%$；极致优化（FlashAttention等）$> 50\%$。 ------ ### 显存核算 (Memory Accounting) **核心问题：我的模型会不会 OOM (Out Of Memory)？** 显存不是一个黑盒子，它被精确地切分为四块蛋糕： #### 1. 模型参数 (Model Weights) - **定义**：模型静态权重。 - **估算**：$P \times \text{dtype\_size}$。 - FP32 = 4 bytes; FP16/BF16 = 2 bytes。 #### 2. 优化器状态 (Optimizer States) —— **最大的隐形杀手** - **定义**：AdamW 等优化器为了更新参数而维护的历史统计量。 - **AdamW 的显存消耗（每参数）**： - 参数本体 (Master Copy, usually FP32): 4 bytes - 一阶动量 Momentum ($m$, FP32): 4 bytes - 二阶动量 Variance ($v$, FP32): 4 bytes - (加上梯度与BF16参数) - **口算模板**：**AdamW 默认约占用 16 Bytes / Parameter**（如果是混合精度训练）。 - *这意味着：存 1GB 的模型权重，需要额外 3-4GB 的显存给优化器！* #### 3. 梯度 (Gradients) - **定义**：$\nabla W$，与参数形状相同。 - **估算**：通常与参数同精度，或 FP32。约 4 Bytes / Param。 #### 4. 激活值 (Activations) —— **最动态的瓶颈** - **定义**：Forward 过程中产生的中间结果（Layer outputs, Attention scores）。**必须保存**，否则 Backward 时没法用链式法则求导。 - **特征**： - **不随参数量线性增长**。 - **随 Batch Size ($B$) 和 Sequence Length ($L$) 线性（或平方）增长**。 - **公式**：$\text{Activations} \propto B \times L \times H \times N_{\text{layers}}$。 - **常见坑**：Transformer 的 Attention 矩阵大小是 $L^2$，长序列训练时显存会爆炸。 - **对策**：**Activation Checkpointing (Rematerialization)**。不存中间结果，反向传播时重算。用算力（多算 33%）换显存（节省 5x-10x）。 ------ ## 显存分析 训练时的显存消耗主要由以下**四部分**组成：

\text{Total Memory} \approx \text{Params} + \text{Activations} + \text{Gradients} + \text{Optimizer States}

假设我们使用 `float32` (每个数值占 4 bytes)。 1. **模型参数 (Parameters)** - **定义**：模型的权重 ($W$) 和偏置 ($b$)。 - **估算**：对于你的线性模型 `Cruncher`，参数量约为 $D^2 \times \text{layers}$。 - **公式**：$M_{param} = N_{param} \times 4 \text{ bytes}$ 2. **激活值 (Activations)** - **定义**：前向传播过程中每一层的输出。 - **作用**：为了计算梯度（反向传播），必须把中间层的输出存下来。 - **特点**：**这是显存爆炸的元凶**。它与 Batch Size ($B$) 和 Sequence Length ($L$) 成正比。 - **公式**：$M_{act} = B \times L \times D \times \text{layers} \times 4 \text{ bytes}$ 3. **梯度 (Gradients)** - **定义**：损失函数相对于每个参数的导数。 - **特点**：形状与参数完全一致。 - **公式**：$M_{grad} = N_{param} \times 4 \text{ bytes}$ 4. **优化器状态 (Optimizer States)** - **定义**：优化器为了更新参数而维护的历史信息（如动量 Momentum、方差）。 - **特点**：最吃显存的部分之一。 - **SGD**：0 或 1 倍参数量（视是否用 momentum）。 - **Adam**：**2 倍参数量**（保存了一阶矩 $m$ 和二阶矩 $v$）。 - **公式**：$M_{opt} \approx N_{param} \times 8 \text{ bytes}$ (以 Adam 为例) ------ ## 算力估算 除了显存，我们需要知道训练需要算多久。这通常用 **FLOPs** (Floating Point Operations，浮点运算次数) 来衡量。 ### 2.1 估算公式 对于基于 Transformer 或大型全连接层的模型，训练一步（One Step）的计算量有一个经典的经验公式：

\text{FLOPs per Step} \approx 6 \times B \times N_{param}

为什么是 6？ 这是一个近似值，来源于矩阵乘法的性质： - **前向传播 (Forward)**：约 2 FLOPs/param（乘法+加法）。 - **反向传播 (Backward)**：约 4 FLOPs/param。 - 计算对输入的梯度：2 FLOPs。 - 计算对权重的梯度：2 FLOPs。 - **合计**：$2 + 4 = 6$。 > **注意**：这个公式忽略了 Softmax、LayerNorm 等操作，因为在大模型中，矩阵乘法（Linear Layers）占据了 99% 的计算量。 ## 硬件基准测试 代码片段中包含了非常实用的硬件检测与性能测试工具。 ### 6.1 为什么需要 `torch.cuda.synchronize()`？ 在 `time_matmul` 函数中，这行代码至关重要： ```Python def time_matmul(a, b): torch.cuda.synchronize() # 1. 等待之前的任务结束 # Start Timer a @ b # 2. 发射计算核心 torch.cuda.synchronize() # 3. 等待计算真正完成 # Stop Timer ``` **原因**：CUDA API 是**异步**的。当你 Python 执行 `C = A @ B` 时，CPU 只是把任务“发射”到了 GPU 队列里就立刻返回了（耗时几微秒）。如果不加 `synchronize()`，你测出来的只是“发射任务的时间”，而不是“执行任务的时间”。 # Lecture3 ## 核心架构演变 ### 标准 Transformer 回顾 最初的 Transformer 架构（Vanilla Transformer）旨在解决序列建模问题，其设计包含以下核心组件： - 层级结构（Layer Structure）： 由多个 Transformer Block 堆叠而成。每个 Block 包含两个主要子层：多头自注意力机制（Multi-Head Self-Attention）和逐位前馈网络（Position-wise Feed-Forward Network）。 - 位置编码（Position Embeddings）： 使用 正弦/余弦（Sinusoidal）函数 生成固定的位置向量。这些向量直接相加到输入的 Token Embedding 中，赋予模型绝对位置信息。公式如下：

PE_{(pos, 2i)} = \sin(pos / 10000^{2i/d_{model}})

PE_{(pos, 2i+1)} = \cos(pos / 10000^{2i/d_{model}})

- 归一化（Normalization）： 采用 LayerNorm，且放置在残差连接之后，称为 Post-Norm。 公式结构为：$x_{l+1} = LayerNorm(x_l + SubLayer(x_l))$。 特点：LayerNorm 会对输入的均值和方差进行标准化，并且包含可学习的偏置项（Bias）。 - 激活函数（Activation）： FFN 层使用 ReLU 激活函数。 计算公式：$FFN(x) = \max(0, xW_1 + b_1)W_2 + b_2$。 ### 现代 LLM 的普遍共识 随着模型规模的扩大（从 GPT-3 到 Llama、PaLM 等），研究人员发现原始架构在训练稳定性及效率上存在局限。现代主流 LLM 形成了一套新的“标准配置”： - **归一化位置：Pre-Norm（前置归一化）** - **变化**：将归一化层移动到多头注意力或 FFN 的输入之前，而不是残差连接之后。 - **公式结构**：$x_{l+1} = x_l + SubLayer(LayerNorm(x_l))$。 - **原因与优势**：Post-Norm 在深层网络中容易导致梯度爆炸或消失，使得训练极不稳定，通常需要极其小心地进行学习率预热（Warm-up）。Pre-Norm 将归一化置于残差分支内，保护了主干（Main Residual Path）的信号传播，显著提升了训练稳定性，允许模型使用更大的学习率并训练得更深。 > [!note] > > > ## 残差 > > 残差就是指神经网络层只负责学习**“理想输出”与“原始输入”之间的差值（即增量修改部分）**，而不是从零去拟合完整的输出结果。 > > 在现代大模型（Pre-Norm 架构）的每一层中，数据的流动： > > 1. 主干保留（Identity Path）： > > 上一层的输出 $x$ 进入这一层后，直接走“主干道”。在这条路上，$x$ 不做任何修改，保持原样直接传送到层的末尾。 > > 2. 分支计算（Residual Branch）： > > 与此同时，系统会从主干上**“复制”**一份 $x$ 出来进入“加工车间”（即分支）。 > > - 在现代 LLM 中，这份复制的数据会先经过 **Norm**（归一化），再进入 **Attention** 或 **FFN** 进行复杂的计算。 > - 计算出的结果，我们称为“增量”或“残差”，记为 $F(x)$。 > > 3. 加回主干（Add）： > > 计算结束后，分支产生的 $F(x)$ 会汇入主干道，与原本保留的 $x$ 进行逐元素相加。 > > - **结果**：$New\_x = x + F(x)$ - **归一化类型：RMSNorm（均方根归一化）** - **变化**：使用 RMSNorm 替代 LayerNorm。 - **原理**：LayerNorm 同时减去均值并除以方差，而 RMSNorm 省略了均值中心化（re-centering）的操作，仅根据输入的均方根进行缩放。 - **公式**：$y = \frac{x}{\sqrt{||x||_2^2 + \epsilon}} \cdot \gamma$。 - **原因与优势**： 1. **计算效率**：减少了计算均值的操作，在 GPU 上计算速度更快。 2. **参数更少**：移除偏置项，仅保留缩放参数 $\gamma$。 3. **效果相当**：实验表明其在语言建模任务上的表现与 LayerNorm 持平甚至更好。 - **激活函数：SwiGLU（Swish Gated Linear Unit）** - **变化**：使用 SwiGLU 替代 ReLU 或 GeLU。 - **原理**：这是一种门控线性单元。它包含两个线性投影层，其中一个通过 Swish 激活函数后作为“门”来控制另一个线性投影的信息流。 - **公式**：$FFN_{SwiGLU}(x) = (Swish(xW) \otimes (xV))W_2$。 - **注意**：由于引入了额外的权重矩阵 $V$，为了保持参数量与标准 Transformer 一致，通常会将中间隐藏层维度（$d_{ff}$）从 $4d_{model}$ 降低到约 $\frac{8}{3}d_{model}$。 - **原因与优势**：相比 ReLU 的“硬”截断，SwiGLU 提供了更平滑的非线性表达能力，已被证明能稳定提升模型的下游任务性能。 - **位置编码：RoPE（Rotary Position Embeddings，旋转位置编码）** - **变化**：使用 RoPE 替代绝对位置编码（正弦或学习到的 Embedding）。 - **原理**：RoPE 不再将位置信息相加到 Embedding 上，而是将 Query 和 Key 向量在向量空间中进行**旋转**。旋转的角度由位置索引决定。 - 核心特性：

\langle f(q, m), f(k, n) \rangle = g(q, k, m-n)

这意味着两个 Token 的注意力分数仅取决于它们的相对距离（$m-n$），而非绝对位置。 - **公式**：它结合了绝对位置编码的实现便利性和相对位置编码的长距离外推能力，非常适合处理长文本序列。 - **原因与优势** - **偏置项（Bias）：移除偏置（No Bias）**：在全连接层（Linear Layers）和归一化层（LayerNorm/RMSNorm）中移除加性偏置项（Bias term）。 - **变化**： 1. **原因与优势**：在大规模训练中，偏置项有时会导致数值不稳定。 2. **稳定性**：虽然偏置项占用的参数量很少，但移除它们可以减少内存读写（Memory Access）开销，微幅提升训练吞吐量。 ------ ## 架构变体详解 ### 归一化层 (Normalization)：位置与方式的变革 归一化层的作用类似于对数据进行 " 整理 "，确保每一层的输入分布保持在一个合理的范围内，防止数值过大（爆炸）或过小（消失）。 #### 1. 位置之争：Pre-Norm Vs. Post-Norm 这实际上是关于 " 归一化应该放在哪里 " 的选择，它直接决定了梯度的流动是否顺畅。 - **内存与计算** - **Post-Norm (原始设计的痛点)**：数据先经过子层（如 Attention）处理，与残差相加后，**结构**进行归一化。 - 公式：$x_{l+1} = LayerNorm(x_l + Sublayer(x_l))$ - **再**：这种结构把归一化放在了 " 主干道 "（残差连接）上。随着网络层数加深，主干上的梯度会被反复归一化，导致反向传播时梯度很难传回底层。这使得模型在训练初期极其不稳定，必须小心翼翼地使用 " 学习率预热 "（Warm-up），即刚开始用极小的学习率慢慢 " 热身 "，否则模型很容易崩溃。 - **问题** - **Pre-Norm (现代主流的解法)**：数据在进入子层**结构**先进行归一化，计算完后直接加回主干，主干上不再有归一化层。 - 公式：$x_{l+1} = x_l + Sublayer(LayerNorm(x_l))$ - **之前**：这种设计让主干（残差路径）保持 " 干净 "。梯度可以沿着这条没有障碍的高速公路直接流向底层。结果是训练非常稳定，甚至可以省去复杂的 Warm-up 阶段，直接使用较大的学习率训练，显著提升了收敛速度。 - **优势**：虽然 Pre-Norm 是主流，但为了追求极致的信号稳定性，Gemma 2 采用了 " 双重归一化 "，即在 Pre-Norm 的基础上，在残差合并之后又加了一层 Post-Norm，以确保信号幅值在极深网络中依然受控。 #### 2. 算法之争：LayerNorm Vs. RMSNorm 这是关于 " 如何进行归一化 " 的选择，核心在于效率与效果的权衡。 - **特例（Gemma 2）** - 它做两件事： 1. **LayerNorm (传统做法)**：减去均值，把数据中心移到 0。 2. **去中心化 (Centering)**：除以标准差，把数据幅度统一。 - 它还包含一个可学习的偏置项 $\beta$。 - **缩放 (Scaling)**：这种做法统计学上最严谨，但计算步骤较多。 - **观点** - **RMSNorm (现代首选)**：研究发现，对于神经网络的收敛来说，" 把数据中心移到 0" 并不重要，重要的是 " 把数据幅度统一 "。 - **核心洞察**：它直接去掉了减均值的步骤，只保留缩放（除以均方根）。同时，它也去掉了偏置项 $b$。 - **做法**： 1. **优势**：少算一个均值，少做一次减法。 2. **计算更少**：不需要存储偏置项参数。 3. **参数更少**：虽然从纯数学运算量（FLOPs）看，归一化占比很小，但在 GPU 训练中，数据搬运（Memory IO）是瓶颈。RMSNorm 因为计算简单、参数少，减少了内存读取和同步的开销，从而在实际运行时间（Wallclock time）上带来了明显的提升。 ------ ### 2.2 激活函数 (Activations)：从开关到门控 激活函数赋予了神经网络非线性能力，现代 LLM 的演进方向是 " 更平滑 " 和 " 可控性更强 "。 - **速度更快** - $max(0, x)$。最简单粗暴，小于 0 直接变为 0，大于 0 保持不变。它的问题是 " 硬截断 "，缺乏平滑性。 - **ReLU (过去式)** - GPT 系列使用。它给 ReLU 加了一个平滑的曲线过渡，允许少量的负数通过，符合概率分布的直觉，比 ReLU 表现更好。 - **GeLU (过渡态)** - **SwiGLU (现代标准)**。这是最重要的概念。SwiGLU 不再仅仅是对一个值进行变换，而是将输入一分为二（或者说通过两个独立的线性层）： 1. **核心机制：门控 (Gating)**：$xW$ 2. **信号部分**：$xV$ - 它用 " 门控部分 " 经过 Swish 激活后，去控制 " 信号部分 " 的通过量。 - **门控部分**：$FFN_{SwiGLU}(x) = (Swish(xW) \otimes xV)W_2$ - 其中 $Swish(x)=x*\sigma(x)$,$\sigma(x)$ 是 sigmoid 函数. - **公式**：$\otimes$ 是逐元素乘法。这就像一个可学习的过滤器，网络自己决定让信号的哪一部分通过，哪一部分被抑制。 - **直观理解**：因为以前只需要一个 $W$ 矩阵，现在需要 $W$ 和 $V$ 两个矩阵，参数量激增。为了公平对比（保持总参数量不变），使用 SwiGLU 时通常会将隐藏层维度（$d_{ff}$）从原来的 4 倍模型维度降低到约 2/3（即 $\frac{8}{3} d_{model}$）。 > ### 激活函数的演进逻辑 —— 从 ReLU 到 SwiGLU > > #### 1. 传统的基准：ReLU (Rectified Linear Unit) > > - **参数平衡**：**核心逻辑** 。 > - **硬门控 (Hard Gating)**：$FFN(x) = \max(0, xW_1 + b_1)W_2 + b_2$ 。 > - **公式**： > - 这是一个 " 非黑即白 " 的单向阀门。 > - **机制**：只要数值小于 0，直接归零（截断）；只要大于 0，原样通过。 > - **规则**： > - **局限性**：处于负值区间的微弱信号可能包含语义信息，但被无情丢弃。 > - **信息丢失**：它只是单纯的线性通过或阻断，无法模拟特征之间复杂的调制关系。 > > #### 2. 现代的标准：SwiGLU (Swish Gated Linear Unit) > > - **缺乏交互**：**核心逻辑**。 > > - 公式：$FFN_{SwiGLU}(x, W, V, W_2) = (Swish(xW) \otimes xV)W_2$ > > - 机制拆解： > > SwiGLU 将同一个输入 $x$ 投影到了两个不同的空间，类似于将原来的一条路拆成了 " 控制电路 " 和 " 数据电路 "： > > 1. **数据驱动的门控 (Data-Driven Gating)**：负责**信号路径 ($xV$)**。代表特征的具体数值信息（即 " 是什么 "）。 > 2. **提供内容**：负责**门控路径 ($xW$)**。经过 Swish 激活后，生成一个 $(0, 1)$ 之间的实数值，充当 " 阀门开度 "（即 " 该不该过 "）。 > 3. **判断重要性**：两者进行逐元素乘法。门控信号动态地 " 调制 " 内容信号的幅度 。 > > #### 3. 深度解析：为什么 " 自己管自己 " 是合理的？ > > 对于 " 输入自己决定自己是否通过 " 这一机制（即所谓的 " 君子协定 "），其本质是**乘性交互 ($\otimes$)**。 > > - **上下文感知**： > > - **从 " 死规则 " 到 " 自适应 "** 像是一个死板的安检员，只看数值正负（硬规则）。 > - **ReLU** 像是一个智能调音台。输入内容 $x$ 包含了上下文信息，网络通过学习矩阵 $W$，学会了识别哪些上下文是重要的。 > - **SwiGLU**：在阅读时，你的大脑（门控）根据读到的句子内容（输入），自动决定要不要对这句话高亮记忆（放大信号）还是略读跳过（抑制信号）。**比喻** > > - 乘性交互的优势： > > SwiGLU 引入了乘法（Multiplication），这使得神经元之间可以形成 " 如果 A 发生，则 B 增强 " 的复杂逻辑关系，比 ReLU 的单纯加法/截断具有更强的表达能力。 > > #### 4. 工程实现的代价与平衡 > > - 参数成本： > > 由于 SwiGLU 需要两组权重矩阵（$W$ 和 $V$），而 ReLU 只需要一组，这会导致参数量增加。 > > - 解决方案： > > 为了保持模型总参数量与标准 Transformer 一致，通常会将中间隐藏层的维度（$d_{ff}$）从原来的 $4d_{model}$ 下调至约 $\frac{8}{3}d_{model}$（即缩小 2/3）。 > > - 结论： > > 即便减少了维度，SwiGLU 带来的性能提升（通常在困惑度 perplexity 和下游任务上更好）依然超过了维度的损失，因此成为了 LLaMA、PaLM 等现代大模型的标准配置 。 ------ ### 2.3 层级排列：串行 Vs. 并行 这是关于 Transformer Block 内部 " 子层执行顺序 " 的工程优化。 - **内容本身决定了它是否值得被注意。** - **串行 Block (Serial - 标准做法)**：`Input -> Attention -> Add -> Norm -> MLP -> Add` - **流程**：MLP 的输入依赖于 Attention 的输出。就像接力赛，MLP 必须等 Attention 跑完才能接棒。这在逻辑上更符合直觉（先混合信息，再处理信息）。 - **逻辑** - **并行 Block (Parallel - 高效做法)**：`Input -> Attention` 和 `Input -> MLP` 同时进行。 - **流程**：Attention 和 MLP 都直接读取同一个输入 $x$，各自计算完后，把结果直接加在一起。 - 公式：$y = x + MLP(LN(x)) + Attn(LN(x))$ - **逻辑**： 1. **优势**：由于两者输入相同，在 GPU 上可以将 Attention 的投影矩阵（Q,K,V）和 MLP 的投影矩阵（Up）合并成一个巨大的矩阵乘法一次性做完。 2. **算子融合 (Fusion)**：减少了 GPU 内核启动的开销和内存读取次数，通常能带来约 15% 的训练提速。 - **速度提升**：对于参数量较小的模型，这种 " 互不通气 " 的并行计算可能会导致微小的性能下降；但在百亿、千亿参数的大模型上，这种影响几乎可以忽略不计，因此为了训练效率，PaLM 等大模型选择了这种方案。 ------ ## Position Embeddings ### 核心问题：为什么 Transformer 需要位置编码？ Transformer 的核心组件——自注意力机制（Self-Attention）——本质上是**置换不变（Permutation Invariant）**的。 这意味着，如果你打乱输入句子的单词顺序（例如将 " 我爱你 " 改为 " 你爱我 "），Attention 模块计算出的 " 词与词之间的关联度 " 在数值上是完全一样的（如果不考虑因果掩码）。对于 Transformer 来说，如果没有额外的位置信息，它看到的只是一个 " 词袋 "（Bag of Words），无法理解语序和时序。 因此，我们必须显式地注入位置信息。 ------ ### 3.1 早期方案演进 在 RoPE 出现之前，业界主要经历了三种主流方案，它们各有优劣： #### 1. 正弦/余弦编码 (Sinusoidal Position Embeddings) 这是 2017 年《Attention Is All You Need》原始论文中提出的方案。 - **代价**：使用不同频率的三角函数生成固定的位置向量。 - 对于偶数维度：使用 $sin$ 函数。 - 对于奇数维度：使用 $cos$ 函数。 - 频率从 $2\pi$ 到 $10000 \cdot 2\pi$ 几何级数递减。 - **原理**：**操作方式**。将生成的位置向量 $PE$ 直接**加法 (Additive)**到 Token 的 Embedding 上：$X_{final} = X_{embed} + PE$。 - **加**：虽然设计初衷是希望利用 $\sin(\alpha+\beta)$ 的线性关系来捕捉相对位置，但在自注意力机制的 $Q \cdot K^T$ 运算中，展开后会出现无法消除的交叉项（Cross-terms）。这意味着它虽然包含相对位置信息，但并非完美的 " 相对位置平移不变性 "。 #### 2. 绝对位置编码 (Learned Absolute Position Embeddings) 这是 GPT-2 和 GPT-3 采用的方案，非常简单粗暴。 - **局限性**：不使用函数生成，而是直接初始化一个可学习的矩阵 $P \in \mathbb{R}^{L_{max} \times d}$。 - 第 1 个 token 加向量 $p_1$，第 2 个 token 加向量 $p_2$，以此类推。 - **原理**：**操作方式**。 - **加法 (Additive)**：**局限性**。 - 如果训练时设定最大长度 $L_{max}=2048$，模型就只学到了前 2048 个位置向量。推理时如果遇到第 2049 个 token，模型完全不知道该用什么向量（因为没学过），这直接限制了模型处理超出训练长度文本的能力。 #### 3. 相对位置编码 (Relative Position Embeddings) 这是 T5 和 ALiBi 等模型采用的思路。 - **外推性（Extrapolation）极差**：不再强求给输入向量加东西，而是直接修改 Attention 的计算公式。 - 它认为：词义本身不需要位置，只有在**计算关注度（Attention Score）**时，距离才重要。 - **原理**：**操作方式**。 - 标准 Attention：$Score = Q \cdot K^T$ - 相对 Attention：$Score = Q \cdot K^T + B_{(i-j)}$ - 其中 $B$ 是一个根据距离 $i-j$ 查表得到的偏置标量。 - **偏置 (Bias)**：虽然效果不错，但它改变了 Attention 的核心计算逻辑，增加了推理时的复杂度和 KV Cache 管理的难度。 ------ ### 3.2 RoPE (Rotary Position Embeddings) —— 旋转位置编码 RoPE 是目前的 " 版本答案 "，它巧妙地结合了**局限性**和**绝对位置编码的实现便利性**。 #### 1. 核心直觉：不要 " 加 " 位置，要 " 转 " 位置 之前的方案大多是 " 纹身 " 式的（把位置信息加在 Embedding 上），而 RoPE 是 " 时钟 " 式的。 想象一个二维向量 $(x, y)$。 - **相对位置编码的数学性质**：我们将这个向量在平面上逆时针**绝对位置 m** $m \cdot \theta$ 角度。 - 第 1 个词的向量转 $\theta$ 度。 - 第 2 个词的向量转 $2\theta$ 度。 - **旋转**： - 当我们计算第 $m$ 个词（Query）和第 $n$ 个词（Key）的点积（相似度）时，点积的结果只与它们之间的**相对位置 m-n**有关。 - 夹角差 = $(m \cdot \theta) - (n \cdot \theta) = (m-n) \cdot \theta$。 - **夹角差**：只要相对距离 $m-n$ 固定，无论它们在序列的绝对位置哪里（比如第 1 和 2，还是第 100 和 101），它们的旋转角度差都是一样的，点积结果也一样。 #### 2. 数学实现：复数域的旋转 为了将这个直觉应用到 $d$ 维向量上，RoPE 使用了**结论**策略： - **分组旋转**：将 $d$ 维的 Query 和 Key 向量切分为 $d/2$ 个子对。例如 $d=128$，就切成 64 对双元素向量。 - **两两分组**：将每一对 $(q_{2i}, q_{2i+1})$ 看作一个复数，乘以旋转因子 $e^{im\theta_i}$。 - 在实数矩阵形式下，这等价于乘以一个分块对角矩阵（Rotation Matrix $R_m$）：

R_m = \begin{pmatrix} \cos m\theta_1 & -\sin m\theta_1 & 0 & 0 \ \sin m\theta_1 & \cos m\theta_1 & 0 & 0 \ 0 & 0 & \cos m\theta_2 & -\sin m\theta_2 \ 0 & 0 & \sin m\theta_2 & \cos m\theta_2 \ \end{pmatrix}

−乘性注入：RoPE是目前主流编码中少有的乘性(Multiplicative)编码，而非加性。

f(x, m) = R_m \cdot x

#### 3. RoPE 的决定性优势 - 绝对形式，相对效果：代码实现时，我们只需要对 Q 和 K 分别进行旋转（绝对位置操作），但在 Attention 点积计算后，自然就得到了仅依赖相对距离的结果。

\langle R_m q, R_n k \rangle = q^T R_m^T R_n k = q^T R_{n-m} k

- **免费的外推能力**：由于位置是通过旋转角度定义的，理论上在推理时，我们可以旋转出比训练时更大的角度（即更远的位置），模型依然能保持一定的数学连贯性（虽然太远会存在精度衰减，但比直接查表法好得多）。 - **无需额外参数**：旋转矩阵是预设的（正弦/余弦函数），不需要像 GPT-3 那样学习巨大的位置矩阵。 > [!note] > > > 每移动一个token,对应旋转角度为$\theta$,这是一个超参数,可以称作为基准频率. > > 在学术界和工程实现中，它通常被称为 **RoPE Base**（记作 $b$ 或 $\theta_{base}$）。 > > ### 1. 这个超参数控制什么？ > > 回顾 RoPE 的角度公式： > >

\theta_i = b^{-2i/d}

其中 $b$ 就是你说的基准频率，默认值通常是 10,000。 - **物理含义**：它决定了旋转位置编码的**“波长”**或**“周期”**。 - **直观理解**： - 想象一把刻度尺。 - $b$ 决定了这把尺子**“最小刻度”和“最大量程”之间的跨度**。 - $b$ 越大，尺子的量程（能表示的最大距离）就越长，但刻度可能会变得稀疏。 ### 2. 为什么要改这个超参数？（长文本扩展） 当你想要把一个模型的上下文窗口从 4k 扩展到 32k、100k 甚至 1M 时，**直接修改这个超参数**是最主流的做法。 这被称为 **RoPE Scaling（RoPE 缩放）**。 - 如果不改 $b$ (比如维持 10,000)： 当序列长度超过训练时的长度（比如 4096）时，旋转角度会超出模型见过的范围，模型会“晕头转向”，Perplexity（困惑度）会瞬间爆炸。 - 改大 $b$ (比如改为 500,000)： 通过增大底数，你实际上是降低了旋转的速度。 - **效果**：原本 4096 个位置转完一圈，现在可能需要 10 万个位置才转完一圈。 - **意义**：你把更长的物理距离，“压缩”进了模型熟悉的旋转角度范围内。这就好比把地图的**比例尺缩小**了（Zoom Out），虽然路变长了，但在模型眼里，角度的变化率依然在它熟悉的范围内。 ### 3. 工业界的实战案例 现在的顶级模型几乎都在调整这个超参数： - **Llama 1 / 2**： - Context: 2k / 4k - **RoPE Base: 10,000** (标准值) - **Code Llama** (专门处理长代码)： - Context: 100k - **RoPE Base: 1,000,000** (为了支持超长上下文，暴力拉大) - **Llama 3**： - Context: 8k / 128k - **RoPE Base: 500,000** (显著调大，以支持原生 8k 并易于外推) ### 4. 调节它的代价（Trade-off） 你可能会问：*“既然调大就能无限长，为什么不直接设成一个亿？”* 因为这符合**“分辨率与量程的权衡”**原理： 1. 分辨率下降（Resolution Loss）： 当你把 $b$ 调得极大，旋转变得极慢。相邻两个 Token（比如位置 500 和 501）之间的旋转角度差 $\Delta \theta$ 会变得非常非常小。 如果 $\Delta \theta$ 小到被浮点数精度误差（Float16/BFloat16）淹没，模型就分不清这两个词的顺序了。这会导致短距离的理解能力下降。 2. 低频灾难： 过大的 Base 会导致高频分量减少，模型可能在捕捉局部细节（Local Structure）时变弱。 ### 总结 - **它是超参数吗？** 是的，关键超参数。 - **怎么调？** - 想处理**长文本** $\rightarrow$ **调大** $base$（比如 50w, 100w）。 - 想追求**短文本**极致精度 $\rightarrow$ **维持** $base$（10000）。 - **需要重训吗？** - 如果只改 $base$，通常需要进行少量的**微调（Fine-tuning）**来让模型适应新的旋转速度。这比从头训练（Pre-training）要快得多。

Hyperparameters

尽管模型参数量从几亿（100M）膨胀到万亿（1T），研究人员发现，为了保证训练效率和算力利用率，某些比例关系应保持惊人的恒定。

4.1 前馈网络维度比率 (dff​/dmodel​)

这是决定 Transformer 块中 FFN（前馈神经网络）” 容量 ” 的关键参数。

  • 核心问题：训练这个模型需要多少算力？需要多久？
    • 标准设定：4 倍扩展 (4dmodel​)：如果主干网络（Residual Stream）的维度 dmodel​ 是 4096，那么 FFN 中间层的维度通常设为 4096×4=16384。
    • 概念：这是经典的 ” 瓶颈（Bottleneck）” 结构设计。
      • 主干网络负责信息的 ” 压缩传输 ” 和 ” 路由 ”。
      • FFN 层负责将特征投影到更高维的空间进行 ” 解缠（Disentangle）” 和非线性变换。就像把揉成一团的纸（低维）在更大的桌子（高维）上展开铺平，更容易看清上面的字。
  • 原理

    • GLU 调整：38​ 倍扩展 (≈2.67dmodel​)：当你将激活函数从 ReLU 升级为 SwiGLU 时，网络结构发生了变化。

    • 背景：

      • 数学账本（参数预算守恒）：包含 2 个矩阵（上投影 Up、下投影 Down）。参数量 ≈2×dmodel​×dff​。
      • 标准 FFN：包含 3 个矩阵（门控 Gate、信号 Value、输出 Output）。参数量 ≈3×dmodel​×dff′​。

    • 计算：为了让两种架构的总参数量保持一致（即 ” 加量不加价 ”），我们需要缩小中间维度 dff′​。

2 \times (4d) \approx 3 \times d_{ff}’ \implies d_{ff}’ \approx \frac{8}{3}d \approx 2.67d

- **复数乘法**：这是为了在引入更强的门控机制（SwiGLU）时，不增加显存和计算负担而做出的精确数学折中。 - 特例（T5 的教训）： Google 的 T5 模型曾尝试将这个比率设为极端的 64 倍。但这会导致参数过度集中在 FFN 层，不仅计算昂贵，而且并没有带来成比例的性能提升（边际效应递减）。后续模型迅速回归到了 4 倍或 2.67 倍的理性区间。 > [!note] > > > FP8 训练 (全链路低精度训练) 这是 DeepSeek-V3 能够以极低成本训练出来的关键工程突破。 - **默认模式：** 传统大模型通常使用 BF16（16 位浮点数）进行训练。 - **推测采样 (Speculative Decoding)：** FP8（8 位浮点数）的数据大小只有 BF16 的一半，计算速度理论上是 BF16 的两倍（在 H100/H800 GPU 上）。 - **背景：** 8 位表示的数值范围极小，非常容易出现**FP8 的优势：** 或 **挑战：**，导致训练发散（Loss 飞掉）。 - **溢出 (Overflow)** - **下溢 (Underflow)** 不是对整个矩阵做量化，而是把大矩阵切成极小的块（Tile），对每一小块单独计算量化参数。这样即使某一块数值很大，也不会影响其他块的精度。 - **DeepSeek 的解决方案：** 虽然前向传播和反向传播的激活值（Activation）和梯度（Gradient）使用 FP8，但为了累积微小的更新，核心的主权重副本可能依然保持较高精度，或者采用特殊的更新策略。 - **细粒度量化：** 实现了真正的全流程 FP8 训练，不仅显存占用减半（可以塞进更大的 Batch Size），训练速度也得到了巨大的提升，大幅降低了训练万亿参数模型的算力门槛。
