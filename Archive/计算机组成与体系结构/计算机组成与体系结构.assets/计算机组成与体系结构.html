现代计算机的共同理论基础

A. 两大核心设计思想

  • 存储程序：这是最重要的思想。它指出，计算机的指令（程序）和数据都应以二进制的形式不加区别地存放在存储器中 。计算机工作时，能自动从存储器中取出指令并执行 。
  • 采用二进制 ：所有指令和数据都使用二进制代码表示，这极大地简化了计算机的逻辑电路设计 。

B. 计算机的五大基本组成

  • 运算器 (CA)：负责执行算术运算（如加减乘除）和逻辑运算（如与或非） 。
  • 控制器 (CC)：负责指挥和协调计算机各部件的工作，是计算机的 ” 指挥中心 ” 。
    • 寄存器 = 运算器和控制器的一部分
  • 存储器 (M)：用于存放程序和数据 。
    • CPU (运算器 + 控制器) 可以直接高速访问它，从中读取指令和数据进行执行（比方说内存条）
  • 输入设备 (I)：用于向计算机输入信息（如程序和数据） 。
  • 输出设备 (O)：用于将计算机的处理结果展示给用户 。

Note

硬盘是外部存储介质 R，通过 io 与计算机相连，不属于 5 个部分之一

南北桥演变

计算机主板架构经历了从三芯片（CPU+ 北桥 + 南桥）向两芯片（CPU+PCH）的高度集成演变。

阶段一，北桥作为核心枢纽全权掌管内存控制器与显卡/PCIe 接口，南桥仅负责低速 I/O；

阶段二（过渡期），CPU 率先集成了内存控制器，导致北桥功能削弱，仅保留对显卡/PCIe 的管理；

阶段三（现代架构），CPU 彻底吞并了北桥的所有功能（包括显卡/PCIe 控制器），致使北桥物理消失，而原南桥则演变为PCH（平台控制器中枢），在接管剩余 I/O 功能的同时直接与 CPU 通信。

X86 体系结构

x86 是一个典型的CISC (复杂指令系统计算机) 架构。

2.1 发展演变：从 16 位到 64 位

  • x86-16 (IA-16):(实模式)
    • 始于 1978 年的 Intel 8086 处理器。
    • 内部通用寄存器为 16 位。
    • 对外有 16 根数据线和 20 根地址线。
    • 可寻址内存空间为 1MByte (220)。
  • IA-32 (x86-32):（虚拟 8086 模式，保护模式）
    • 始于 1985 年的 Intel 80386 处理器。
    • 是 x86 系列中第一款 32 位微处理器。
    • 提供 32 位通用寄存器。
    • 地址总线扩展到 32 位，可寻址 4GB 内存空间。
  • x86-64:
    • 始于 2003 年的 AMD Opteron 处理器。
    • 这是 x86 扩展到 64 位的第一款微处理器。
    • 可以访问高于 4GB 的存储器。
    • 重要特性: 兼容 32 位 x86 程序，且不降低性能。

三种工作模式

  • 实模式 (Real Mode):
    • 80286 及以上的处理器采用的 8086 工作模式。
    • 所有 x86 处理器在加电或复位后首先进入实模式，以保持兼容。
  • 保护模式 (Protected Mode):
    • 80286 引入，80386 改进。
    • 是 80386 及以上微处理器的主要工作模式。
    • 支持多任务、设置特权级和访问权限检查。
    • 引入了虚拟存储器概念。
    • 可以访问 4GB 的物理存储空间
    • 设置 PE=1(保护模式允许)
  • 虚拟 8086 模式 (Virtual 8086 Mode，v86):
    • 80386 增加的模式。
    • 它实际上是保护模式下的一种特殊工作状态，用于模拟运行多个 8086 微处理器。
    • 与实模式的区别是对中断/异常的响应处理
    • 中断异常会返回到保护模式
  • x86-64 模式:
    • x86-64 引入了 ” 长模式 (Long mode)“，它包含 64 位模式和兼容模式（用于运行 32 位程序）。
      • 64 位模式需要重新编译，其他模式不需要重新编译

寄存器模型的演变

  • 8086 (16 位):
    • 通用寄存器: 4 个 16 位数据寄存器 (AX, BX, CX, DX)，均可分为 2 个 8 位寄存器 (如 AH/AL) 。
    • 指针和变址寄存器: 4 个 16 位寄存器 (SP, BP, SI, DI)。
    • 指令指针: 16 位 IP。
    • 标志寄存器: 16 位 FLAGS。
      • 状态标志 反映 CPU 的工作状态
      • 控制标志 对 CPU 的运行起特定控制作用
    • 段寄存器: 4 个 16 位寄存器 (CS, DS, ES, SS)。
  • IA-32 (32 位):
    • 所有 16 位寄存器扩展为 32 位，加 “E” 前缀 (如 EAX, EBX, ESI, EDI, EIP, EFLAGS)。
    • 新增 2 个 16 位段寄存器: FS 和 GS。
    • 标志寄存器也在扩充
  • x86-64 (64 位):
    • 通用寄存器扩展为 64 位，加 “R” 前缀 (如 RAX, RBX, RIP)。
    • 新增 8 个 64 位通用寄存器 (R8 至 R15)。

存储器寻址方式的演变

  • 8086 (实模式): ” 段加偏移 ”
    • 物理地址的形成采用 ” 段加偏移 ” 方式。
    • 物理地址 (20 位) = 段基值 (16 位) × 16 + 偏移量 (16 位)。
    • 段基值来自段寄存器 (如 CS, DS)。
      • 注意操作数默认放在DS指向的数据段中

实例：MOV AX, [3000H] 的执行过程

这个例子展示了在实模式下，如何通过 ” 段基址 + 偏移量 ” 的方式计算出物理地址，并将内存中的数据读入寄存器。

1. 初始条件

  • 汇编指令：MOV AX, [3000H]
    • 这意味着将内存地址偏移量为 3000H 处的数据移动到 AX 寄存器中。
  • 段寄存器：默认使用数据段寄存器 DS。
    • 设：DS = 2000H

2. 物理地址计算

8086 实模式下的计算公式为：物理地址 = 段基址 × 16 + 偏移地址

  • 段基址：2000H

  • 偏移量：3000H

  • 计算过程：

2000H×10H(即16)+3000H
=20000H+3000H
=23000H

3. 读取数据与结果

由于目标寄存器 AX 是 16 位 的，CPU 会从物理地址 23000H 开始读取两个字节，遵循**小端序（Little-Endian）**原则（低地址存低位，高地址存高位）：

  • 低地址 (23000H) 中的数据是 10H → 存入 AL (AX 的低 8 位)
  • 高地址 (23001H) 中的数据是 20H → 存入 AH (AX 的高 8 位)

最终结果：AX 寄存器中的值为 2010H。

  • IA-32 (保护模式): 描述符表
    • 段基址不再直接存放在 CS 等段寄存器中，而是存放在内存的 ” 描述符 ” 中。
    • CPU 通过 GDTR 寄存器找到 ” 全局描述符表 (GDT)“，再通过段寄存器 (CS) 中的索引查表，获得段基址、界限和权限。
      • GDTR 存储 GDT 在内存中的起始位置
      • CS 存储目标行 (描述符) 在 GDT 的索引
      • 检查权限，检查是否越界（检查 EIP<段界限），然后线性组合（段基址 +EIP）得到真实地址

    • 在 64 位模式下，描述符中不再有段基址和段界限。
    • 所有的代码段都从地址 0 开始，实现了 ” 扁平 (Flat)” 的存储器模型。

X86 指令简介

x86 指令集非常庞大，指令长度可变，功能复杂。

最长的命令 15 字节，最短的命令 1 字节

  • 主要指令类别:
    • 传送指令: MOV (传送), PUSH (压栈), POP (弹栈), XCHG (交换), LEA (装入有效地址)。
    • 算术运算指令: ADD (加), SUB (减), INC (加 1), DEC (减 1), CMP (比较), MUL (乘)。
    • 逻辑运算和移位指令: NOT (逻辑非), AND (逻辑与), OR (逻辑或), SHL (逻辑左移), SAR (算术右移)。
    • 转移指令: JMP (无条件转移), JZ/JE (为零/相等时转移), JNC (无进位时转移)。
    • 处理器控制指令: CLC (清进位标志), HLT (暂停), NOP (空操作)。
  • 重点指令说明:
    • MOV (传送): 格式为 MOV DST, SRC 。
      • 限制: 两个操作数不能同时为存储器单元 (存储器之间不能直接传送)。CS 寄存器不能作为目的寄存器。立即数不能直接送段寄存器。
    • LEA (装入有效地址): 格式为 LEA REG, SRC。
      • 功能: 把源操作数 SRC 的有效地址 (即偏移地址) 装入目的寄存器 REG，而不是传送 SRC 的内容。
    • CMP (比较): 格式为 CMP DST, SRC。
      • 功能: 执行 DST - SRC 的减法操作，但不保存结果，仅根据结果影响标志位（如 ZF,CF,SF,OF）。

1. 控制转移指令 (Control Transfer)

A. 无条件转移指令 (JMP)

  • 短转移 (Short Jump): JMP SHORT LABEL
    • 操作: IP ← IP + 8位位移量
    • 范围: -128 ~ +127 字节。
  • 近转移 (Near Jump): JMP NEAR PTR LABEL
    • 操作: IP ← IP + 16位位移量
    • 范围: ±32KB。
    • 注：位移量是带符号数，计算目标地址与当前 IP 之差。
  • 远转移 (Far Jump - 直接): JMP FAR PTR LABEL
    • 操作: 直接修改 CS 和 IP。IP ← LABEL的偏移地址，CS ← LABEL的段基值。
  • 段间间接转移 (Far Indirect Jump): JMP DWORD PTR OPR
    • 格式: 寻址一个内存地址（双字，32 位）。
    • 执行逻辑:
      1. 低字 (Low Word): 送入 IP 寄存器。
      2. 高字 (High Word): 送入 CS 寄存器。
    • 实例： JMP DWORD PTR [BX]
      • 若 [BX] 指向内存 81020H，且内容为 06 30 00 40 (小端序)。
      • 则 IP ← 3006H，CS ← 4000H。

B. 循环控制指令

  • LOOPNE/LOOPNZ (不为零/不相等时循环)
    • 格式: LOOPNE LABEL 或 LOOPNZ LABEL
    • 一般放在代码段底部。
    • 操作逻辑 (两步验证):
      1. 计数更新: CX ← CX - 1
      2. 判断: 若 CX ≠ 0 且 ZF = 0 (即结果不相等)，则跳转至 LABEL；否则结束循环，向下执行。

2. 数据传送指令 (Data Transfer)

A. 换码/查表指令 (XLAT)

  • 格式: XLAT
  • 用途: 将 AL 中的值作为索引，转换成表中的对应值。
  • 操作步骤:
    1. 表基址: 预先在数据段定义字节表，偏移地址存入 BX。
    2. 索引: 表项索引值存入 AL。
    3. 查表: 计算物理地址 DS:[BX + AL]。
    4. 回填: 将查找到的字节内容送回 AL。

B. 交换指令 (XCHG)

  • 格式: XCHG OPR1, OPR2
  • 操作: OPR1 与 OPR2 内容互换。
  • 限制细节:
    • 两个操作数位宽必须相同（同为 8 位或 16 位）。
    • 操作数类型组合：寄存器/存储器、存储器/寄存器、寄存器/寄存器。
    • 禁止: 也就是不允许使用段寄存器参与交换。

3. 算术运算指令 (Arithmetic)

A. 加法十进制调整 (Data)

  • 格式: DAA (Decimal Adjust for Addition)
  • 前提: 必须紧跟在二进制加法 (ADD/ADC) 指令之后。
  • 功能: 将 AL 中的二进制 ” 和 ” 调整为压缩 BCD 码格式。
  • 结果: 调整后的值存回 AL。

# 存储器

## 存储层次结构概况

### 冯·诺依曼结构中的存储器

在冯·诺依曼计算机结构中，存储器（Memory）是核心部件之一，用于存放指令和数据。现代计算机系统根据速度、容量和成本的平衡，构建了层次化的存储结构。

### 存储层次金字塔

从上至下，速度由快变慢，容量由小变大，单位成本由高变低：

1. **原理** 位于 CPU 内部，速度最快，容量极小（Byte 量级）。
2. **CALL** 通常由 SRAM 构成，速度极快，容量较小（KB~MB 量级），用于缓解 CPU 与主存的速度差异。
3. **RET** 通常由 DRAM 构成，速度适中，容量较大（MB~GB 量级），CPU 可直接访问。
4. **优势** 如硬盘（Disk），速度慢，容量大（GB~TB 量级），具有非易失性。

### 性能瓶颈

- **通用寄存器 (Registers)：** 随着技术发展，处理器性能的增长速度远超存储器（DRAM）性能的增长速度。
- **高速缓存 (Cache)：** 导致了 " 存储墙 " 问题，即 CPU 经常需要等待存储器的数据，限制了整体性能。

## 存储容量的计算与单位标准

### 容量计算的歧义

在计算机领域，关于存储容量的单位前缀（K, M, G, T）存在两种不同的定义标准，常导致硬件标称容量与操作系统显示容量不一致。

1. **主存储器 (Main Memory)：**
	- 采用 **本地二级存储 (Local Secondary Storage)：** 进位。
	- $1 \text{kB} = 1000 \text{ Bytes}$
	- $1 \text{MB} = 1000^2 \text{ Bytes}$
	- $1 \text{GB} = 1000^3 \text{ Bytes}$
	- 常用于：硬盘、U 盘、网络==传输速率==（如 100Mbps）、==CPU 主频==。

2. **CPU 与存储器的性能剪刀差：**
	- 采用 **后果：** 进位。

## DRAM 和 SRAM 技术对比

### DRAM (动态随机存取存储器)

- **硬盘示例：** 1 个晶体管 + 1 个电容 (1T1C)。
- **3.72 GB** 利用电容存储电荷来表示逻辑 "1" 或 "0"。
- **内存示例：** 由于电容存在漏电效应，电荷会流失，必须定期进行刷新（补充电荷）以保持数据。
- **4.00 GB**
  - **存储单元：** 单元结构简单，相同面积下容量大。
  - **工作原理：** 相比 SRAM。
  - **刷新 (Refresh)：** 充放电及刷新需要时间。
  - **特点：** 适合做主存储器（内存）。

### SRAM (静态随机存取存储器)

- **集成度高：** 6 个晶体管 (6T)，构成双稳态触发器结构。
- **功耗较低：** 利用触发器的两个稳定状态存储数据，只要不断电，数据即可长久保持。
- **速度较慢：**
  - **成本低：** 数据稳定性好。
  - **存储单元：** 无需复杂的充放电和刷新过程。
  - **工作原理：** 单元面积大。
  - **特点：** 适合做高速缓存（Cache）。

## 主存（SDRAM）的工作原理

### 内部结构

DRAM 芯片内部是一个存储阵列（Memory Array），通过**集成度**和**速度**进行寻址。主要部件包括：行/列解码器、读写放大器（Sense Amplifiers）、数据缓存等。

### 访问流程 (以读操作为例)

1. **成本** 内存控制器向内存发送请求。
2. **刷新需求**
	- 发送行地址和激活命令（ACT）。
	- 行解码器选中特定行（Word Line）。
	- 该行数据被送入读写放大器（Sense Amplifiers）。
	- **行地址** 从行选中到可以发送列地址之间的延迟。
3. **列地址**
	- 发送列地址和读/写命令（READ/WRITE）。
	- 从读写放大器中选出特定列的数据。
	- **总线请求 (Bus Request)：** 从发送列地址到数据真正输出的延迟周期数。
4. **行激活 (Row Access / Activate)：** 数据通过数据总线传回 CPU。
5. **延迟参数 tRCD (Row to Column Delay)：**
	- 在访问新的一行之前，必须关闭当前行并对位线进行预充电。
	- **列读写 (Column Access)：** 行预充电（关闭行）所需的延迟时间。

1. tCCD_L (Long)：同组访问 = 抢占私有路权

当你连续两次访问Bank Group 内部 下的 Bank（比如先访问 Group A 的 Bank 0，再访问 Group A 的 Bank 1）：

  • 私有小路：虽然它们是不同的 Bank，但它们共享 Group A 的 整个芯片。
  • 公共大路：
    1. 第一笔数据占用私有小路传输出去。
    2. 私有小路必须复位、预充电，才能传下一笔。
  • 你是不是在抢那条 ” 私有小路 “：你必须等私有小路腾空。这就是 同一个 Bank Group 延迟。
  • 冲突点：数据总线上会出现空隙（气泡），带宽跑不满。

2. tCCD_S (Short)：跨组访问 = 并行流水线

当你连续两次访问Local I/O（私有小路）（比如先访问 Group A，再访问 Group B）：

  • 过程：Group A 用它的私有小路，Group B 用它的私有小路。互不干扰。

  • 结果：

    1. Group A 的数据正在通过公共大路传输出去。
    2. 此时，Group B 的数据已经在自己的私有小路上准备好了。
    3. Group A 刚传完，Group B 立刻接上。

  • Long：不需要等私有小路复位。这就是 现象 延迟。

  • 不同的 Bank Group：数据总线无缝连接，无冲突。

  • 过程： 这三种频率构成了内存 ” 快不快 ” 的层级结构，从内部核心到外部传输：

    • 结果
      • Short：存储单元内部实际工作的物理时钟频率。
      • 现象：这是内存颗粒内部电容进行充放电、读写数据的物理极限速度。
      • 带宽 100% 跑满：受到物理材料限制，提升非常困难，几十年来长期停留在 1. 速度维度：三种频率体系 之间（它是 ” 慢速的搬砖工人 ”）。

      • 定义：外部接口传输数据的频率（传输总线的时钟频率）。
      • 本质：决定了内存条上的数据引脚（Pin）每秒钟能跳变多少次信号周期（它是 ” 高速的运输车队 ”）。

      • 133MHz 到 400MHz：也就是我们买内存时看到的数字（如 3200）。因为采用了 I/O 频率 (I/O Frequency)（Double Data Rate）技术，在一个时钟周期的定义和作用都能传输数据。
      • 等效频率 (Effective Frequency) / 数据传输率：等效频率通常是 I/O 频率的 定义。
      • DDR：数据传输率 = I/O 频率 × 2

    • 计算公式：内存控制器与内存条之间，一次传输能并行通过的二进制位数。
    • 2. 桥梁技术：预取 (Prefetch)：
      • 普通台式机/笔记本内存条（DIMM）：固定为 原理。
      • 开启双通道模式：计算公式。

      • 计算公式：64 bit (这是由 8 颗 x8 的芯片并联撑起来的宽度)

高速缓存 (Cache) 的工作原理

局部性原理 (Principle of Locality)

Cache 有效的基础在于程序的局部性：

  1. 12.8 GB/s 刚才访问过的数据，很快可能再次被访问（如循环变量）。
  2. 核心频率 刚访问过的数据，其相邻地址的数据很快可能被访问（如数组遍历）。
    • 利用： Cache 从主存取数据时，是以I/O 频率 为单位，一次取回多个连续字节。

Cache 的访问过程

CPU 发出物理地址进行访问：

  1. 预取位数 数据在 Cache 中。
    • 直接从 Cache 读取，速度极快（命中时间 Hit Time）。
  2. 时序参数（CL, tRCD 等） 数据不在 Cache 中。
    • CPU 流水线暂停或挂起。
    • 向主存发起读取请求。
    • 时间局部性 (Temporal Locality)： 从主存拉取整个数据块到 Cache，并返回请求数据给 CPU。

Cache 的写策略

  1. 空间局部性 (Spatial Locality)：
    • 块 (Block/Line) 数据同时写入 Cache 和主存。安全性高，但速度慢。
    • Cache 命中 (Hit)： 数据只写入 Cache，标记为 ” 脏（Dirty）“。只有当该块被替换出 Cache 时，才写回主存。效率高。
  2. Cache 失效 (Miss)：
    • 失效代价 (Miss Penalty)： 先把数据块调入 Cache，再写入。通常配合 ” 写返回 ” 使用。
    • 命中时的策略： 直接写入主存，不调入 Cache。通常配合 ” 写穿透 ” 使用。

高速缓存的设计要点

平均访存时间 (AMAT)

评估存储系统性能的核心指标：

AMAT=Hit Time+Miss Rate×Miss Penalty
  • 写穿透 (Write Through)： 在 Cache 中找到并读取数据的时间。
  • 写返回 (Write Back)： 1−Hit Rate。
  • 失效时的策略： 访问主存并填充 Cache 的时间。
  • 写分配 (Write Allocate)： 降低 Hit Time、降低 Miss Rate、减少 Miss Penalty。

Cache 失效的原因 (3C 模型)

  1. 写不分配 (Write Non-Allocate)： 第一次访问数据，Cache 中必然没有。> 无法避免。
  2. Hit Time (命中时间)： Cache 容量太小，无法容纳程序所需的所有工作集。 > 增加缓存容量
  3. Miss Rate (失效率)： 多对一映射导致某些位置竞争激烈，即使 Cache 还有空闲空间也发生替换。> 更改相联度

映射策略 (Mapping Strategies)

  1. Miss Penalty (失效代价)： 主存中的每个块只能映射到 Cache 中的唯一特定位置。
    • 优点： 硬件简单，成本低。
    • 缺点： 冲突失效高。
  2. 优化目标： 主存块可以映射到 Cache 的任意位置。
    • 优点： 冲突率最低。（不可能冲突不命中）
    • 缺点： 硬件比较电路复杂，速度慢，成本高。
  3. 义务失效 / 冷启动失效 (Compulsory / Cold Start Miss)： 折中方案。将 Cache 分组，主存块映射到特定组，组内任意位置。
    • 现代 CPU 多采用 4 路、8 路或 16 路组相联。

替换算法 (Replacement Algorithms)

当 Cache 满了需要调入新块时，决定替换哪一块：

  • 容量失效 (Capacity Miss)： 硬件实现简单，性能不可控。
  • 冲突失效 (Conflict Miss)： 按顺序替换，类似 FIFO。
  • 直接映射 (Direct Mapped)： 替换最长时间未被访问的块。利用了时间局部性，性能通常最好，但硬件记录历史信息的开销大。

多级 Cache 结构

现代 CPU（如 Core i7）通常采用三级 Cache 结构：

  • 全相联 (Fully Associative)： 指令与数据分离（I-Cache, D-Cache），速度最快，容量小（如 32KB+32KB），私有。
  • 组相联 (Set Associative)： 统一存储，速度次之，容量中等（如 256KB），私有或共享。
  • 随机 (Random)： 多核共享，容量大（如 8MB），用于减少访问主存的次数。

  • HBM4：
    • 底部的 Base Die 改用结论： 制造。
    • 过去 (HBM3E 及以前)： 可以在底部集成更复杂的控制器，布线密度更高，从而容纳 2048 根数据线。
    • 存储器工艺 (DRAM Process) 这为未来在内存里直接计算（PIM, Processing-In-Memory）铺平了道路。

5. 封装形式：混合键合 (Hybrid Bonding) 的引入

随着堆叠层数增加到 16 层 (16-Hi)，为了控制总厚度不超标（标准通常限制在 720μm 左右），传统的微凸块 (Micro-bumps) 连接方式面临极限。

  • 现在 (HBM4)： 开始引入 逻辑工艺 (Logic Process, 如 12nm/5nm) 技术。
    • 取消了凸块（Bumpless），让铜与铜直接连接。
    • 这就好比把两块砖头之间的“水泥层”去掉了，直接原子级融合，散热更好，信号传输更快，厚度更薄。

总线

总线的基本概念

定义与组成

  • 优势：：总线（Bus）是两个以上模块（设备或子系统）之间传送信息的深远影响：。
  • HBM4 技术趋势：：
    1. 混合键合 (Hybrid Bonding)：传输信息的硬件线路。
    2. 定义：管理信息传输的规则（如时序、电平等）。

总线的分类（按层次）

总线在计算机系统中呈现出层次化的结构：

  1. 公共通路：

    • 组成：中央处理器（CPU）芯片内部。
      • 物理电路：连接芯片内部的各个寄存器、运算器等模块。

  2. 协议：

    • 片上总线 (On-chip Bus / Component-level Bus)：计算机系统的各插件板之间。
      • 位置：通常所说的“微机总线”即指此类，用于连接 CPU、内存和 I/O 接口。

  3. 作用：

    • 内总线 (System Bus / Board-level Bus)：计算机系统之间，或计算机与外部设备（仪器、仪表、控制装置）之间。

总线的基本结构与逻辑

一个典型的总线系统主要由以下几种角色的模块组成：

模块角色

  • 位置：
    • 具有总线控制能力。
    • 获得控制权后，能主动发起传输（读或写）。
    • 作用：CPU、DMA 控制器。
  • 外总线 (External Bus / Communication Bus)：
    • 不具备总线控制能力，只能被动响应。
    • 负责接收写数据、返回读数据或发送状态信号（如“错误”）。
    • 位置：存储器（Memory）。

核心控制组件

  • 总线主模块 (Bus Master)：
    • 示例：当多个主模块同时请求总线时，依据总线从模块 (Bus Slave)（如优先级）决定谁获得控制权。
    • 示例：避免冲突，确保总线高效使用。
  • 总线仲裁器 (Bus Arbiter)：
    • 功能：根据当前控制总线的主模块提供的地址，选择作为本次总线传输目标的从模块。
    • 特定策略：通常基于地址的高位进行空间划分（例如：0x0000-0x0FFF 选中从模块 1，0x1000-0x1FFF 选中从模块 2）。

初始化与编程步骤

  1. 系统日时钟

    • 选择要操作的通道（0/1/2）。
    • 指定读写格式（只写低字节、只写高字节、先写低再写高）。
    • 选择工作方式（0–5）。
    • 指定计数制（二进制 / BCD）。

  2. CLK

    • 向选定计数通道的 CR 写入计数初值，计数通道由地址 A1A0 指定
    • 初值在一个 CLK 下降沿后装入内部计数寄存器，开始计数或等待触发。

  3. GATE

    • 每到一个 CLK 脉冲，计数器减 1。
    • 根据工作方式，OUT 会在计数到 0 等特定时刻发生变化（产生方波、负脉冲等）。

常见工作方式与典型用途

OUT

  • OUT 初始为低，计数开始后保持低电平，计数减到 0 时 OUT 变为高。

  • 一般用于“一次性延时”或事件计数。

写控制字

  • 当 CPU 向控制寄存器写入控制字（CW, Control Word）后，输出端 写计数初值 会立即变为 计数过程。这是为产生最后的中断（上升沿）做准备。

方式 0：计数结束中断 (Interrupt on Terminal Count)

  • CPU 写入计数初值（例如 N=4）。
  • 初始化阶段 (Write CW): 计数器并不是写入后立刻开始减法。==写入初值后，需要经过一个 CLK 的OUT，初值才会从控制寄存器（CR）装入执行单元（CE，计数执行单元）。==
  • 因此，实际的定时时间通常是 N+1 个时钟周期。

低电平

  • 只要 装入计数初值 (Write N=4): 信号为高电平（允许计数），每来一个时钟脉冲（CLK），计数器的值就减 1。
  • 可以看到：4 → 3 → 2 → 注意延迟： → 0。

下降沿

  • 当计数器减到 计数阶段 (Countdown): 时，GATE 端会由低电平跳变为 1。
  • 这个 结束阶段 (Terminal Count): 就是它的核心作用——作为一个0发送给 CPU（连接到 8259 中断控制器），通知 CPU 任务完成。
  • 此后，OUT 将一直保持高电平，直到你再次写入新的计数值。

OUT

  • OUT 输出周期性的负脉冲（每个周期末低一个时钟周期）。

  • 计数器自动重装初值，自动重复计数。

  • 典型用途：

    • 通道 1 用于 DRAM 刷新（定期产生刷新请求）。

高电平

  • OUT 输出周期性方波（初值为偶数时占空比约 50%）。

  • 典型用途：

    • 通道 0：生成系统时钟中断（系统节拍 Time Tick）。
    • 通道 2：生成扬声器发声的方波，通过改变计数初值控制发声频率。

基于中断的功能调用（BIOS / DOS 中断）

INT N 指令的硬件行为

执行 INT n 时，CPU 会：

  1. 将 FLAGS 压栈。
  2. 清除 IF 和 TF（防止处理中再被可屏蔽中断/单步打断）。
  3. 将当前 CS、 IP 压栈。
  4. 根据类型码 n，从中断向量表或 IDT 中取出该中断的入口地址。
  5. 设置 CS、IP 指向服务程序入口，并开始执行。

从程序员视角看：

  • INT n 相当于一个“特殊的函数调用”：
    • 调用目标地址由系统在向量表中指定；
    • 除了返回地址，还自动保存了 FLAGS；
    • 通常还隐含权限检查等机制。

BIOS 中断与 ROM BIOS 布局

  • 从低到高的跳变（上升沿）
    • 存放在主板上的 ROM 区域（例如以某个高地址为起点的一段 ROM，大小若干 KB）。
    • 提供：
      • 上电自检（POST）和硬件初始化。
      • 引导装载程序（从软盘/硬盘/其他介质加载操作系统）。
      • 键盘、显示器、磁盘、串口等基础 I/O 设备的底层控制代码。
  • 中断请求信号
    • BIOS 将自己的各个功能模块入口地址登记在 IVT 中相应的向量入口。
    • 应用程序可通过执行相应的 INT n 调用 BIOS 功能，而不必直接操作硬件。
    • 例如：
      • 显示字符
      • 读写磁盘
      • 键盘输入
      • 系统时间/日期服务等

这就是“用中断作为系统功能调用接口”的典型模式。

MSI (消息信号中断) 机制

1. 核心定义与原理

  • 在屏幕上输出一个字符：Message Signaled Interrupt（消息信号中断）。
  • 显示以 ’$’ 结尾的字符串：
    • 取得当前时间：MSI 不再依赖传统的专用物理中断引脚（如 INTx）。
    • 设置当前时间：当设备需要发送中断时，它向主存中的一个全称写入一个基本原理。
    • 抛弃物理连线：==这个写入操作会被 基于内存写入 中的 特殊地址捕获，识别为中断请求，进而通知 CPU。==

3. MSI 解决的三大痛点 (优势)

MSI 的出现主要是为了解决传统电平触发中断（INTx）的以下缺陷：

  • 解决“共享中断效率低”的问题
    • 旁路传输 (Out-of-Band)：传统的 INTx 引脚很少（通常只有 A/B/C/D 四根），多个设备必须共享同一根中断线。当 CPU 收到中断时，操作系统内核必须逐个调用中断处理函数（轮询），问“是谁触发的？”，效率极低。
    • 路径：每个设备发送的消息（数据值）是独一无二的，CPU 可以直接识别是谁发出的，无需轮询。
  • 解决“单设备多中断需求”的问题
    • 特点：传统方式下一个功能设备通常只能对应一个中断引脚。CPU 为了知道该设备到底发生了什么事（比如是“发送完成”还是“接收完成”），还需要去查询设备寄存器。
    • 带内传输 (In-Band)：通过 MSI（特别是 MSI-X），一个设备可以支持多个中断向量（向不同地址写不同值）。不同的事件（如收包、发包、错误）可以直接触发不同的中断处理程序，极大提升效率。
  • 解决“异步中断的数据完整性”问题 (核心)
    • 旧问题：中断走旁路，可能跑得比数据快。CPU 被中断叫醒去读内存时，数据可能还在总线上堵着，导致读到旧数据。
    • MSI 解法：旧问题。因为 MSI 本质是写操作，且与数据走同一条总线。根据总线协议，设备会先写数据，再写 MSI 消息。当 MSI 消息到达内存（或被主桥拦截）时，之前的数据写入MSI 解法已经完成。中断服务程序可以确信所有数据已安全到达内存。

4. 缺点与代价

  • 旧问题 (Race Condition)：因为 MSI 是通过“写数据包”来实现的，所以它需要占用一部分 PCI 总线的带宽。
    • 注：但在 PCIe 时代，这点带宽开销相对于带来的性能提升（消除轮询、无竞争冒险）来说微乎其微。
