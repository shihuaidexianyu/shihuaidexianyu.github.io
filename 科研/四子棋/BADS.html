BADS 算法详解

BADS 通常指 Bayesian Adaptive Direct Search，中文可译为“贝叶斯自适应直接搜索”。它是一个面向黑箱、无梯度、可能带噪、且单次评估代价中等偏高的连续优化算法。它最典型的用途不是训练深网络那种大规模一阶优化，而是做参数拟合、最大似然估计、仿真模型校准这类问题：目标函数往往非凸、可能不光滑，梯度拿不到，评估一次却又不便宜。BADS 的核心设计是把 MADS 直接搜索框架 和 局部高斯过程贝叶斯优化 拼接起来：前者负责“保底的、模型无关的局部探索”，后者负责“利用代理模型更高效地猜下一步”。

先把它解决的问题写清楚。BADS 针对的是如下形式的优化：

x⋆=argx∈Xmin​E[f(x)]

这里，x∈RD 是待优化参数，X 是有界可行域，f(x) 是黑箱目标函数。若目标函数是确定性的，那么 E[f(x)] 就退化为普通的 f(x)；若目标函数带随机性，例如由模拟器近似、蒙特卡洛估计或随机采样产生，那么单次评估值只是随机观测，真正要找的是其期望意义下的最优点。BADS 从一开始就把这两种情形都纳入设计范围，这也是它比很多“默认假设无噪声”的局部优化器更实用的原因。

一、名字里的三个词分别是什么意思

1. Bayesian

这里的 “Bayesian” 不是指它在做完整的贝叶斯后验推断，而是指它使用了 贝叶斯优化 的核心工具：高斯过程（Gaussian Process, GP）代理模型。GP 用已经评估过的点去拟合一个“对目标函数局部形状的概率性近似”，从而在“均值低的地方”与“不确定性高的地方”之间做权衡，决定下一步评估哪里。

也就是说，BADS 里的 Bayesian，重点在于“用概率代理模型指导搜索”，而不是“输出参数后验分布”。这一点和 VBMC 之类以后验近似为目标的方法要明确区分。

2. Adaptive

“Adaptive” 有两层含义。

第一层是步长自适应：搜索成功时扩大尺度，失败时缩小尺度；

第二层是局部模型自适应：GP 的训练集、长度尺度、搜索协方差、轮询方向顺序都会随着当前局部地形和历史表现更新。也就是说，BADS 不是固定一个采样模板盲目试探，而是会根据“当前在哪儿、刚刚发生了什么、代理模型是否靠谱”不断换策略。

3. Direct Search

“Direct Search” 的意思是：它不依赖梯度，而是直接在参数空间里试探点并比较函数值。这里的“direct”不是“简单暴力穷举”，而是有结构的局部方向搜索。

BADS 继承的是 MADS 框架，所以它不是像随机搜索那样无方向，也不是像 CMA-ES 那样完全基于采样分布演化；它保留了一个网格/轮询式的确定性骨架，用来在 GP 不可靠时兜底。

二、BADS 的基本思想：为什么要把 BO 和 Direct Search 混在一起

单独看两类方法，各有短板。

传统贝叶斯优化的长处是样本效率高：评估代价很贵时，愿意多花算法开销去建立代理模型，从而少评几次真函数。问题在于，标准 BO 往往维护一个全局 GP，训练集越来越大时，代理建模和采集函数优化的开销会迅速变重；如果目标函数只是“中等昂贵”而非“极其昂贵”，这种额外开销就可能不划算。

纯直接搜索方法的长处是稳健：不需要代理模型，不怕模型失配，也不依赖梯度。但它们的问题是，若局部结构本来可以被学习到，它们就会显得“信息利用不足”，导致要靠更多真实函数评估才能找到好解。

BADS 的核心判断是：对于很多模型拟合问题，最合适的不是“全程相信 GP”，也不是“全程不建模”，而是把两者做成主从关系。于是它采用这样一种分工：

flowchart TD
    A["输入：目标函数 f、起点 x0、硬边界 LB/UB、可选合理边界 PLB/PUB"] --> B["初始化：坐标变换、初始设计、建立局部 GP"]
    B --> C["SEARCH 阶段：用局部 GP + 采集函数提出候选点"]
    C --> D{"是否得到足够改进？"}
    D -- "是" --> E["更新 incumbent，重新围绕新点做 SEARCH"]
    D -- "否" --> F["POLL 阶段：按 MADS 网格与方向集做保底搜索"]
    F --> G{"POLL 是否成功？"}
    G -- "是" --> H["更新 incumbent，并放大搜索尺度"]
    G -- "否" --> I["保持 incumbent，不移动，并缩小搜索尺度"]
    H --> J{"是否满足停止条件？"}
    I --> J
    E --> J
    J -- "否" --> C
    J -- "是" --> K["返回最优点；有噪声时按更保守准则选返回点"]

这张图里最关键的边界是：SEARCH 是“模型驱动的高效推进”，POLL 是“模型失效时的兜底纠偏”。二者不是并列装饰，而是职责不同、互相补位。

对 BADS 的概括是：它在 MADS 的 SEARCH 阶段里嵌入局部 BO，在 SEARCH 多次失败时切换到 POLL，由 POLL 提供稳健的模型无关探索，同时为下一轮 SEARCH 重新积累局部形状信息。

三、BADS 在优化什么对象：参数、边界、尺度与“合理边界”

BADS 的输入不只是一个函数和一个初始点。它特别强调两类边界。

第一类是 hard bounds，也就是硬边界 LB,UB。它们定义了绝对不允许越界的可行域。这个边界的作用是“数学上可行”，越出去直接不合法。

第二类是 plausible bounds，也就是 PLB,PUB。它们不是可行性边界，而是“你认为好解大概率会落在其中的区域”。这不是多余信息，而是 BADS 很重要的工程设计。算法会把变量线性变换到标准化盒子 [−1,1]D，其中这个标准盒子对应的恰好是 plausible bounds，而不是 hard bounds。这样做的逻辑是：BADS 主要在“合理区域”里组织局部 GP、步长和搜索尺度，避免因为硬边界过宽而把局部建模拉得过散。

这两类边界很容易混淆，但必须严格区分：

  • hard bounds 回答的是“这个参数取值允许不允许”；
  • plausible bounds 回答的是“这个参数取值值得不值得优先相信”。

例如，一个方差参数数学上可以在 (0,∞) 内变化，这是 hard bound 的层面；但根据问题背景，你可能知道它大概率落在 [10−2,10]，这就是 plausible bounds 的层面。BADS 会优先把搜索尺度适配到后者，而不会被前者的“无限或超宽范围”拖垮。

还有一个常被忽略的点：如果某个变量严格为正，且上界至少是下界的 10 倍，BADS 会自动把它转到对数空间再优化。这么做不是数学花活，而是因为正参数常常表现为“相对变化比绝对变化更自然”，在 log 空间里局部长度尺度更容易统一。

四、BADS 的骨架：MADS 到底提供了什么

BADS 的底层骨架来自 MADS，Mesh Adaptive Direct Search。理解 BADS，先要明白 MADS 的几个对象。

1. incumbent

xk​ 表示第 k 轮迭代中的当前最好点，也就是当前“驻点候选”。BADS 后续的局部建模、搜索候选生成、轮询方向试探，基本都围绕这个点展开。

2. mesh size 与 poll size

BADS 维护两个尺度参数：网格尺度 Δkmesh​ 和轮询尺度 Δkpoll​。前者控制点必须对齐到的细网格，后者控制轮询步长的大致尺度。一般默认在标准化空间里用

Δ0mesh​=2−10

以及

Δ0poll​=1

这意味着：一开始允许的轮询步已经足以跨越整个 plausible region 的量级，但底层网格却更细，方便把点吸附到一个统一离散结构上。

3. poll set

当 SEARCH 失败时，BADS 会构造一组轮询候选点：

Pk​={xk​+Δkmesh​v:v∈Dk​}

其中，Dk​ 是当前迭代采用的一组方向向量。它们不是任意乱选，而是 MADS 规则下可行的轮询方向。这个集合的意义是：以当前 incumbent 为中心，沿若干离散方向做局部试探。只要找到一个足够好的点，就可以提前终止这一轮 POLL，这叫 opportunistic poll。

4. 成功与失败如何改变尺度

MADS 的自适应逻辑很直接：

  • 若 SEARCH 或 POLL 成功找到改进点，则更新 incumbent；
  • 若成功来自 POLL，则通常放大尺度；
  • 若 SEARCH 与 POLL 都失败，则 incumbent 不动，尺度缩小。

BADS 默认使用 τ=2，也就是成功时尺度按 2 的倍数放大，失败时按 2 缩小。这个规则的本质不是“越来越大胆”或“越来越保守”那么简单，而是在局部地形上做一种分辨率调节：你能稳定改善，说明还没到精细打磨阶段，可以看更远；你怎么试都改不了，说明要么局部很平、要么模型不准、要么快到局部极值附近，此时就应降低尺度分辨局部结构。

五、BADS 的 Bayesian 部分：局部 GP 是怎么建的

这部分是 BADS 的“聪明”所在，但也是最容易被误解的地方。BADS 不是对所有历史点维护一个全局 GP；它维护的是围绕当前 incumbent 的局部 GP。这是它与标准 BO 的第一条关键边界。

1. 为什么只建局部 GP

全局 GP 的训练代价会随训练点数增加而快速上升，而且很多远处的旧点对当前局部推进帮助有限，反而可能污染局部近似。BADS 的做法是：从所有已经评估过的点组成的 cache 中，挑出一个靠近 incumbent 的子集作为训练集。这样做的目的有两个：

第一，控制代理模型开销，避免标准 BO 在中等预算下“建模比算函数还贵”。
第二，提高局部拟合针对性，因为真正需要判断的是“当前附近往哪儿走”，而不是整个大域的全局形状。

2. 训练集怎么选

默认规则是：先按到当前 incumbent 的“长度尺度归一化距离”排序，自动加入最近的 50 个点；然后再加入最多 10D 个、且满足一定局部半径条件的点。对 noisy objective，这个最小点数会提高到 100，最大值也会提高。

这里的“长度尺度归一化距离”不是普通欧氏距离，而是：

r2(x,x′)=d=1∑D​ℓd2​1​(xd​−xd′​)2

其中，ℓd​ 是第 d 个维度上的 GP 长度尺度。这个式子表达的是：如果某一维变化很敏感，ℓd​ 小，那么该维上一点点偏离都会被放大；如果某一维变化很迟钝，ℓd​ 大，那么同样的坐标偏差就不算远。这个距离的作用不是定义真实几何距离，而是定义“在 GP 看来，两个点是否属于同一个局部结构”。它解决的是各维尺度不一致的问题，但它本身并不能说明真实函数一定各向异性，只是当前代理模型这样认为。

3. 用什么核函数

BADS 默认使用 ARD 的 rational quadratic 核：

kRQ​(x,x′)=σf2​[1+2α1​r2(x,x′)]−α

其中，σf2​ 是信号方差，控制函数整体波动幅度；ℓ1​,…,ℓD​ 是各维长度尺度；α>0 是形状参数；r2(x,x′) 就是上一段的归一化距离。

这个核函数的意义是：当 x 与 x′ 很接近时，协方差大，说明函数值应当相似；当它们变远时，相关性下降。RQ 核相比常见的 squared exponential 核更“弹性”一些，可以看作多尺度平滑结构的混合，因此论文报告它和 BADS 的局部框架配合较好。这里要注意，核函数只是在表达“我们如何相信局部平滑性”，它不是在证明目标函数真的来自某个随机过程。

4. GP 超参数如何更新

BADS 不是每一步都把 GP 从零开始暴力重拟合。原始论文给出的做法是：对超参数使用经验贝叶斯先验，并通过 MAP 估计得到当前超参数；同时每隔大约 2D 到 5D 次函数评估才重新拟合一次，且在代理明显预测失真时会更频繁地重拟合。

这背后的逻辑很重要：GP 当然越精细越好，但 BADS 的目标不是“把代理模型本身拟合到极致”，而是“让代理足够好用、同时别拖慢整个优化”。所以它把 GP 当作实用工具，而不是主角。

六、采集函数：BADS 如何平衡“往低处走”和“去不确定处看”

有了 GP 之后，BADS 要决定下一次真实函数评估在哪个点。默认采集函数是下置信界（LCB）：

aLCB​(x)=μ(x)−νβt​​,s(x)

其中，μ(x) 是 GP 在 x 处的后验均值，表示当前预测的目标函数值；s(x) 是后验标准差，表示不确定性；ν 是调节探索强度的参数；βt​ 随评估次数 t 变化，用来控制探索项的规模。论文默认采用 ν=0.2 和 δ=0.1 对应的设定。

这个公式是为最小化问题写的，所以值越小越好。于是：

  • μ(x) 小，说明模型认为这里可能是好点；
  • s(x) 大，说明模型还不确定，这里值得探；
  • 减去 νβt​​,s(x)，就是把“未知但可能很好”的区域也拉进候选。

容易混淆的一点是：LCB 不是“越不确定越好”，也不是“只看均值”。它是在最小化一个均值减去不确定性奖励的量。若一个点均值很差，即使不确定性大，也未必值得去；若一个点均值很好但已被充分验证，也未必仍是最优采样点。它表达的是一种折中，而不是某个单一偏好。

七、SEARCH 阶段：BADS 真正“聪明推进”的部分

SEARCH 阶段是 BADS 最像贝叶斯优化的部分，但它不是“全局采集函数优化”，而是围绕 incumbent 的快速局部 BO。论文把 SEARCH 设计成一串激进、反复的小步代理搜索，每轮最多允许若干次不成功的 SEARCH；若一直找不到足够改进，就进入 POLL。默认允许的不成功 SEARCH 次数大约是

nsearch​=max{D,⌊3+2D​⌋}

1. 搜索候选点怎么生成

BADS 不是直接把采集函数交给一个昂贵的全局优化器去最小化，而是模仿 CMA-ES 的思路，在当前搜索焦点附近抽样一批候选点，再从中选取采集函数最优者。抽样分布大致写成：

x∼N(xs​,λ2(Δkpoll​)2Σ)

其中，xs​ 是当前 search focus，λ 是缩放系数，Σ 是搜索协方差矩阵。

这里每个符号的作用都很明确：

  • xs​ 决定“围着哪里搜”；
  • Δkpoll​ 决定当前搜索半径级别；
  • Σ 决定“往哪些方向更愿意伸展”；
  • λ 决定抽样范围比 poll 尺度更大还是更小。

2. 协方差矩阵不是固定的

BADS 在两类协方差之间切换。一类是由 GP 长度尺度导出的对角协方差，表示“按代理模型认定的各维敏感度独立伸缩”；另一类是已评估点的加权协方差，表示“从历史上表现较好的点分布中学一个方向结构”。BADS 再用 hedge 策略根据过往累计改进表现，在这两者之间概率性选择。

这一步非常体现 BADS 的混合风格。它并不完全相信 GP 的长度尺度，也不完全相信经验协方差，而是让两种几何感知竞争。

边界要讲清楚：这不是 CMA-ES 式完整的分布演化，因为 BADS 的目标仍然是服务于局部 BO；但它也不是纯 GP 采集优化，因为它显式利用了历史好点的几何形状。

3. 什么叫“SEARCH 成功”

BADS 不是一看到函数值下降一点点就宣布 SEARCH 成功。论文要求改进必须超过阈值，大约为 (Δkpoll​)3/2；若改进不足，就继续 SEARCH 或转入 POLL。

这个“足够改进”门槛的意义在于：防止算法被极小、可能不稳定的改善牵着走。特别是在代理模型有误差、函数有噪声或局部很平的时候，微小下降未必代表搜索方向真的对。这个判定标准本质上是在问：这次代理驱动的推进，是否足以支撑我重新围绕新 incumbent 建立局部局面。

八、POLL 阶段：为什么说它是 BADS 的“保底机制”

如果 SEARCH 多次失败，BADS 就进入 POLL。你可以把 POLL 理解为：先暂时别过度相信 GP 了，沿一组结构化方向，踏踏实实地试周围邻域。

POLL 的核心价值不只是“再试试看”，而在于它给 BADS 提供了三种保证。

第一，它是模型无关的。即使 GP 完全失配，POLL 仍然能工作。
第二，它给 MADS 带来局部收敛上的理论支撑，至少朝局部驻点的方向有保障。
第三，它会收集新的局部点，让下一轮 GP 的局部形状学习更可靠。

BADS 并没有把 GP 完全排除在 POLL 之外。它在 POLL 里做了两件“轻量借力”的事。

一是构造方向集 Dk​ 后，会按 GP 长度尺度对各维方向做重缩放。也就是说，POLL 依然保留“各维敏感度不同”的信息。
二是由于 POLL 可以 opportunistic 地提前停止，因此 BADS 会按采集函数给 poll set 排序，优先评估更有希望的方向。

所以 POLL 不是“完全不用 GP”，而是“不再让 GP 决定有没有资格搜索，只让它帮助排序和调方向”。这个边界很关键：BADS 在 POLL 中没有把采集函数当唯一决策依据，而是把它当启发式辅助。

九、带噪声时 BADS 为什么不能直接盯着最小观测值

这是 BADS 非常值得学的一点。很多人一看到 noisy objective，第一反应是“多评几次取平均”或“直接拿当前最小观测值”。BADS 认为这都不够稳健。

1. 噪声模型与额外保守设置

对于 noisy objective，论文给噪声标准差引入了对数正态型的超先验，并在工程上把训练集下限、初始设计大小、允许停滞次数都加大。例如，训练集最少点数从 50 增到 100，初始设计增到 20。这不是“随便调大更稳”，而是因为带噪时局部 GP 需要更多点去区分“结构”与“随机波动”。

2. 用 latent quantile 代替原始观测值

在 noisy 情况下，BADS 不直接用 yi​ 作为 SEARCH 与 POLL 的依据，而是使用 GP 的潜在分位数函数：

qβ​(x)=μ(x)+Φ−1(β)s(x)

其中，μ(x) 是 GP 后验均值，s(x) 是后验标准差，Φ−1 是标准正态分布的分位数函数，β∈[0.5,1)。

这个式子的含义要讲透。因为我们做的是最小化，若只看 μ(x)，就是只看平均表现；加上 Φ−1(β)s(x) 后，就会引入一个与不确定性相关的保守修正。当 β=0.5 时，Φ−1(0.5)=0，于是 qβ​(x)=μ(x)，只看均值；当 β 越接近 1，这个分位数越高，对不确定性的惩罚越重。也就是说，qβ​ 不是在估计“最可能观测值”，而是在评估“在给定保守程度下，这个点可能有多差”。

3. 为什么要维护 incumbent set

带噪时，当前 incumbent 不应该只由“这轮恰好观测最小”来定。BADS 因此维护一个 incumbent set，也就是保存历轮 incumbent，并在每次 POLL 结束后，基于新 GP 对这些 incumbent 重新计算 qβ​，再选出当前 active incumbent。

这个机制解决的是一个很现实的问题：某个点可能因为随机噪声显得特别好，但当新数据加入后，GP 会发现它其实没那么可靠。如果不重评历史 incumbent，就容易被噪声偶然值“骗走”。因此，在 noisy setting 下，BADS 不是“谁观测最小谁赢”，而是“谁在当前信息下最稳健谁当 incumbent”。

4. 返回点怎么选

论文还特别区分了“优化过程中使用的保守性”和“最终返回结果时的保守性”。优化过程中用的是 β=0.5，也就是均值层面，鼓励推进；最后返回结果时用的是更保守的 βend​=0.999，以稳健方式决定输出点。返回值也不是简单的最小观测 y，而是返回 μ(xend​) 或对 E[f(xend​)] 的无偏估计。

这条边界非常重要：noisy optimization 的目标不是找“历史上最幸运的一次采样”，而是找“真实期望表现最好”的参数。BADS 在这里的设计是相当严谨的。

十、把完整流程串起来：输入、前提、步骤、输出

现在把它整理成一个完整的流程性知识。

BADS 的输入包括：目标函数 f，初始点 x0​，硬边界 LB,UB，可选的合理边界 PLB,PUB，以及可选约束或噪声信息。算法先把变量转到标准化空间；必要时对严格正变量做 log 变换；然后构造初始设计，默认是 x0​ 加上若干 Sobol 点，并在标准化网格上对齐。若用户没有声明目标函数是否带噪，BADS 甚至会在 x0​ 做两次连续评估来初判。

之后，BADS 围绕当前 incumbent 建立局部 GP，必要时更新其超参数。接着进入 SEARCH 阶段：利用采集函数和局部采样协方差，提出一个候选点 xsearch​，评估真实函数后判断是否构成“足够改进”。若 SEARCH 成功，就把新点设为 incumbent，并围绕它重启 SEARCH；若多次 SEARCH 都不成功，则进入 POLL。

POLL 阶段会构造方向集和 poll set，按机会主义顺序逐点测试。一旦找到更好点，就更新 incumbent；若这一轮失败，则 incumbent 保持不变，并缩小 mesh 与 poll 尺度。若 POLL 成功，则扩大尺度。整个循环持续到满足停止条件：函数评估预算耗尽、poll 尺度缩到阈值以下、或长时间停滞。默认预算大致是 500D 次函数评估，默认还会在停滞若干轮后提前停下。

最终输出的是原始坐标系下的最优参数点。若目标函数无噪声，通常返回当前最小目标值对应的点；若有噪声，则按前面讲的保守分位数准则返回更稳健的点。

十一、BADS 为什么常常比“纯 BO”更实用

一个常见误解是：既然 BADS 里面有 GP，那它不过是“换皮的 BO”。不对。BADS 与标准 BO 的本质差异至少有三条。

第一，BADS 只做局部 GP，而标准 BO 通常更接近全局 GP。局部 GP 让 BADS 的建模成本更可控，也更适合中等预算下的连续局部推进。

第二，BADS 有 MADS 的 POLL 兜底。标准 BO 若代理模型很差，常常只是继续依据不靠谱的采集函数行动；BADS 在 SEARCH 反复失败后会切到更稳健的模型无关机制。

第三，BADS 的工程目标是“适度开销下的强鲁棒性”。论文报告在基准中，BADS 的算法开销大约在每次函数评估额外 0.03 到 0.15 秒量级，而对照的“vanilla BO”因为全局训练集增长，平均可能达到每次评估约 8 秒额外开销。这个数量级差异解释了为什么 BADS 特别适合“单次评估不是天价、但也不便宜”的模型拟合任务。

十二、它和几类常见优化器的边界

1. BADS 与标准 Bayesian Optimization

两者都用 GP 和采集函数，但标准 BO 更偏向“用全局代理在很少次评估内做全局决策”，BADS 更偏向“用局部代理加快局部推进，并用直接搜索做稳健补偿”。所以 BADS 不是全局 BO 的替代品，而是更适合中低维、局部推进、模型拟合型黑箱优化。

2. BADS 与 MADS

BADS 不是另起炉灶的全新框架，而是把 BO 植入 MADS 的 SEARCH 阶段。MADS 负责提供可收敛的直接搜索骨架，BADS 负责把 SEARCH 变聪明。因此，BADS 可以看作“带局部 GP 的 MADS”，而不能简单理解成“加了个轮询步骤的 BO”。

3. BADS 与 CMA-ES

CMA-ES 也不需要梯度，也很擅长连续黑箱优化，但它主要通过演化采样分布的协方差来适应地形；BADS 则显式构建 GP 代理，用不确定性来指导评估。BADS 的 SEARCH 里借鉴了 CMA-ES 式局部高斯采样，但本质上它仍是“代理模型驱动 + 直接搜索兜底”的混合算法。

4. BADS 与 Nelder-Mead

Nelder-Mead 依赖单纯形变换，结构更轻，但在不光滑、带噪、边界复杂时往往不够稳健，也没有 GP 代理提供的信息利用。BADS 的代价更高，但在困难模型拟合问题上通常更可靠。论文的基准测试中，BADS 与多种常见优化器比较时表现通常不差，很多设定下更优。

十三、什么时候适合用 BADS，什么时候不适合

官方文档给出的推荐场景相当明确：==目标函数粗糙或不光滑，可能因为数值近似或噪声导致；单次函数评估至少“中等昂贵”，例如超过 0.1 秒；梯度不可得；维度大约在 D≤20 的范围内。==

这个推荐区间背后的逻辑是：

  • 若评估极便宜，BADS 的 GP 与管理开销可能不值；
  • 若维度太高，局部 GP 和黑箱采样都会变得困难；
  • 若梯度质量很好、目标又比较光滑，成熟的一阶或二阶方法通常更划算；
  • 若你的真正目标是参数不确定性或后验分布，而不是点估计，那么 BADS 也不是终点方法。

因此，BADS 最典型的位置可以概括为一句话：它不是最便宜的问题上的最快方法，也不是最高维问题上的通用解，而是“中低维、黑箱、无梯度、局部复杂、评估不太便宜”的那段甜蜜区间里的强工具。

十四、一个简短但完整的直觉总结

你可以把 BADS 想成这样一个优化器：

它先在你认为“参数大概靠谱”的区域里，把变量标准化；然后围绕当前最好点，取一小批附近历史点建一个局部 GP；接着用 LCB 这类采集函数在附近快速挑下一个最值得试的点，这就是 SEARCH。若这个代理驱动的动作屡试不灵，BADS 就承认“现在别太信模型”，转而沿着 MADS 的结构化方向做 POLL。POLL 一方面继续寻找改进，另一方面也补充了新的局部形状信息，供下一轮 GP 使用。成功则放大尺度，失败则缩小尺度；带噪时不盯着最小观测值，而用更稳健的潜在分位数和 incumbent set 来决定当前点与最终返回点。

所以，BADS 的精髓不在于某一个公式，而在于这套分工：

用 GP 提高局部搜索效率，用 MADS 保证代理失效时仍能稳健前进。
