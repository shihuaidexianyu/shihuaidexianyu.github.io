Summary

这篇笔记带你进入概率图模型的世界，理解有向图（贝叶斯网络）与无向图（马尔可夫随机场）如何用图结构编码高维随机变量之间的依赖关系。读完之后，你应该能读懂 D-分离规则、掌握三种基本局部结构（链式、分叉、汇聚）的条件独立性，并理解变量消除与信念传播如何利用图结构降低推断复杂度。

概率图模型：用图的语言描述概率世界

概率图模型以图结构表示高维随机变量之间的联合分布，通过图中的节点、边与局部因子，把“概率分布的表示”“条件独立关系的编码”“推断与学习的计算过程”统一到同一框架之中。它既是对概率分布结构的压缩表达，也是对复杂概率计算可行化的重要方法。

Intuition

想象你要描述一个班级里 50 个学生的各科成绩联合分布。如果不加任何结构，你需要列举所有成绩组合的概率，状态空间大到无法想象。但如果你知道“数学成绩好的学生，物理成绩通常也好”，就可以引入一条边把数学和物理连起来，从而把庞大的联合分布拆成许多小块。概率图模型就是干这件事的。

概率分布的基本规则与图模型的出发点

概率图模型建立在若干基本概率规则之上。对随机变量的联合分布、边缘分布与条件分布，常用的基本关系包括边缘化规则、乘法公式、链式法则与贝叶斯公式。

离散情形下，边缘化规则可写为

p(x1​)=x2​∑​p(x1​,x2​)

连续情形下，相应地写为

p(x1​)=∫p(x1​,x2​)dx2​

乘法公式写为

p(x1​,x2​)=p(x1​∣x2​)p(x2​)=p(x2​∣x1​)p(x1​)

对任意多个随机变量，链式法则写为

p(x1​,x2​,…,xp​)=i=1∏p​p(xi​∣x1​,…,xi−1​)

贝叶斯公式写为

p(x1​∣x2​)=p(x2​)p(x2​∣x1​)p(x1​)​

这些公式在理论上始终成立，但当变量维度很高时，直接表示和计算联合分布会迅速变得困难。若不引入额外结构，联合分布需要描述全部变量组合的概率，参数规模与计算复杂度都会随维度急剧增长。概率图模型的核心思想正是在这一点上展开：通过对变量间依赖关系作结构化约束，把原本庞大的联合分布分解成若干局部项，并把这些局部项之间的关系用图表示出来。

这种约束通常体现为条件独立性。若变量集合被划分为 A、B、C 三部分，且在给定 XC​ 的条件下，XA​ 与 XB​ 相互独立，则记为

XA​⊥XB​∣XC​

条件独立性并不是对联合分布信息的简单删减，而是对依赖结构的明确说明。概率图模型的作用就在于用图把这种结构编码出来，并据此得到更紧凑的表示、更高效的推断以及更清晰的建模假设。

概率图模型的基本组成

概率图模型通常包含三个相互关联的层面：表示、推断与学习。

  • 表示：讨论如何用图结构编码概率分布及条件独立关系。常见的表示形式包括有向图模型、无向图模型以及更一般的因子图。
  • 推断：讨论如何由模型计算感兴趣的概率量，例如边缘概率、条件概率以及最大后验估计。
  • 学习：讨论如何从数据中估计模型参数与图结构。参数学习在完备数据情形下较为直接，而在存在隐变量时，通常需要使用迭代方法（如 EM 算法）。若连图结构本身也未知，则进一步构成结构学习问题。

有向图模型：贝叶斯网络

基本定义与因子分解

贝叶斯网络是一个有向无环图（DAG）。图中的每个节点对应一个随机变量，每条有向边表示一种直接依赖关系。若记节点 i 的父节点集合为 pa(i)，则整个联合分布可以分解为

p(x1​,x2​,…,xp​)=i=1∏p​p(xi​∣xpa(i)​)

Intuition

这个分解非常直观：每个变量只直接依赖于它的父节点。只要图是有向无环图，就总能按照某个拓扑顺序写出这样的分解。真正起作用的关键不只是分解本身，而是分解背后的局部独立性假设：每个变量在给定其父节点后，与其非后代变量条件独立。

三种基本局部结构

贝叶斯网络中的条件独立关系可先从三个最基本的局部结构理解。

链式结构

graph LR
    A["A"] --> B["B"]
    B --> C["C"]

其联合分布为

p(A,B,C)=p(A)p(B∣A)p(C∣B)

在给定中间节点 B 后，A 与 C 条件独立，即

A⊥C∣B

Intuition

这就像一条信息传递的链条：A 影响 B，B 再影响 C。一旦你知道了 B 的取值，A 对 C 的额外解释能力就消失了。比如，“下雨”→“路面湿”→“车祸多”。如果你已经知道路面湿不湿，那么“是否下雨”对“车祸多不多”就没有额外信息了。

分叉结构

graph LR
    B["B"] --> A["A"]
    B --> C["C"]

其联合分布为

p(A,B,C)=p(B)p(A∣B)p(C∣B)

在给定共同原因 B 后，两个结果变量相互条件独立，即

A⊥C∣B

Intuition

这表示由共同原因引起的相关性，在原因已知后会被解释掉。比如，“季节是夏天”同时导致“冰淇淋销量高”和“溺水事故多”。如果你不知道季节，可能会误以为冰淇淋销量和溺水事故有关；但一旦你知道了季节，这两者就条件独立了。

汇聚结构

graph LR
    A["A"] --> B["B"]
    C["C"] --> B["B"]

其联合分布为

p(A,B,C)=p(A)p(C)p(B∣A,C)

在未给定 B 时，A 与 C 彼此独立，即

A⊥C

但一旦对 B 进行条件化，A 与 C 一般不再独立，即通常有

A⊥C∣B

Intuition

汇聚结构是贝叶斯网络中最重要的特殊结构，也叫“解释消除”（explaining away）。比如，A 表示“电池没电”，C 表示“油箱空了”，B 表示“车发动不了”。平时电池和油箱是独立的；但如果你已知车发动不了，再知道电池没电，就会降低油箱空了的概率——因为“车发动不了”已经被解释掉了一部分。

Trap

若进一步观测到 B 的后代，这条路径同样会被“打开”。因此，汇聚结构与链式、分叉结构在条件独立行为上根本不同。在判断 D-分离时，汇聚节点及其后代是否在条件集合中至关重要。

D-分离与全局条件独立

局部结构说明了最基本的独立关系，但实际模型通常远比三个节点复杂。贝叶斯网络中的全局条件独立由 D-分离（d-separation）刻画。

对图中的一条路径，若其中存在一个非汇聚节点落在条件集合 C 中，则该路径被阻断；若其中存在一个汇聚节点，而该节点及其任何后代都不在 C 中，则该路径也被阻断。只有当路径上所有非汇聚节点都不在 C 中，且所有汇聚节点本身或其某个后代在 C 中时，该路径才是激活的。

若从集合 A 到集合 B 的所有路径都被 C 阻断，则称 A 与 B 在给定 C 下 D-分离，并由此推出

XA​⊥XB​∣XC​

D-分离把局部结构推广到整个有向图，使得“从图上读出条件独立关系”成为可能。它是贝叶斯网络表示能力的核心，也是推断算法能够只关注局部相关部分的重要依据。

马尔可夫毯

贝叶斯网络中，单个节点的条件分布不需要依赖全图所有变量。若记除 xi​ 之外的所有变量为 x−i​，则有

p(xi​∣x−i​)=p(xi​∣xMB(i)​)

其中 MB(i) 表示节点 i 的马尔可夫毯，它由三部分组成：该节点的父节点、子节点以及这些子节点的其他父节点。马尔可夫毯之外的变量，在给定马尔可夫毯后不再为 xi​ 提供额外信息。

从因子分解角度看，xi​ 的局部条件分布只与包含 xi​ 的那些因子有关，因此有

p(xi​∣x−i​)∝p(xi​∣xpa(i)​)j∈ch(i)∏​p(xj​∣xpa(j)​)

其中 ch(i) 表示 i 的子节点集合。这个公式说明，对单个变量做局部更新时，只需考虑与之直接相关的局部项，而无须扫描整个联合分布。

Intuition

马尔可夫毯就像是每个节点的“社交圈”：只要知道了它的父母、孩子和孩子的其他父母，圈外的任何人和事都与它无关。

典型有向图模型

贝叶斯网络是一类统一框架，许多经典模型都可视为其特殊情形。

  • 朴素贝叶斯模型假设在给定类别变量 y 后，各个观测特征条件独立：
p(y,x1​,…,xp​)=p(y)i=1∏p​p(xi​∣y)
  • 高斯混合模型引入离散隐变量 z 表示样本属于哪个成分：
p(x)=k=1∑K​πk​N(x∣μk​,Σk​)
  • 马尔可夫链刻画时间序列中相邻时刻的局部依赖：
p(x1:T​)=p(x1​)t=2∏T​p(xt​∣xt−1​)
  • 隐马尔可夫模型在此基础上再引入观测层：
p(z1:T​,x1:T​)=p(z1​)t=2∏T​p(zt​∣zt−1​)t=1∏T​p(xt​∣zt​)

无向图模型：马尔可夫网络

基本定义

马尔可夫网络又称马尔可夫随机场，用无向图表示随机变量之间的对称依赖关系。与有向图不同，它不强调“谁生成谁”或“依赖方向”，而强调“哪些变量在结构上彼此相邻”。边的存在表示直接相互作用，边的缺失则与某种条件独立结构相关。

无向图中的条件独立关系通常从三个层面刻画：

  • 全局马尔可夫性质：若节点集合 C 在图上把 A 与 B 分隔开，则有 XA​⊥XB​∣XC​。
  • 局部马尔可夫性质：对任一节点 i，在给定其邻居集合 N(i) 后，该节点与其余非邻居节点条件独立。
  • 成对马尔可夫性质：若节点 i 与节点 j 之间没有边，则在给定其余所有节点后，有 Xi​⊥Xj​∣XV∖{i,j}​。

在分布满足严格正性条件时，这三种表述是等价的。

团、势函数与因子分解

无向图要真正构成概率模型，还需要给出联合分布的因子分解。为此引入团（clique）的概念。团是图中一组两两相连的节点集合；若该集合已经不能再加入更多节点而保持两两相连，则称为最大团。

设图中的团集合为 C，则联合分布可写为

p(x)=Z1​c∈C∏​ϕc​(xc​)

其中 ϕc​(xc​) 是定义在团上的势函数，要求取正值；Z 是配分函数，用于保证概率归一化。离散情形下，

Z=x∈X∑​c∈C∏​ϕc​(xc​)

连续情形下，上式中的求和应改为对全空间的积分。

Intuition

势函数并不一定本身就是概率分布，它只需要刻画某一局部配置的相对“相容程度”或“偏好强度”。比如，在图像去噪中，相邻像素如果颜色相近，势函数就给出较高的“相容分”；如果颜色差异很大，势函数就给出较低的分数。多个势函数相乘后经由 Z 归一化，才得到合法的联合分布。

Gibbs 分布与能量形式

若把势函数写成指数形式

ϕc​(xc​)=exp(−Ec​(xc​))

则整个分布可写为

p(x)=Z1​exp(−c∈C∑​Ec​(xc​))

其中 Ec​(xc​) 称为局部能量函数。局部能量越低，对应的配置越容易出现；局部能量越高，对应的配置越不受偏好。这种写法称为 Gibbs 分布或玻尔兹曼分布形式。

Note

对正分布而言，Hammersley-Clifford 定理表明：满足图上的马尔可夫性质与能够按图的团分解为势函数乘积，这两件事是等价的。

有向图与无向图之间的转换

道德图

在很多情况下，需要把有向图转换为无向图，以便应用无向图上的统一推断框架。贝叶斯网络转化为无向图时，首先把每个节点的父节点两两相连，然后去掉所有边的方向。得到的无向图称为道德图（moral graph）。

graph LR
    A["A"] --> B["B"]
    C["C"] --> B["B"]

Intuition

对上面的汇聚结构，若直接去掉箭头，会丢失由共同子节点 B 诱发的 A 与 C 之间的依赖关系。因此需要先把 A 与 C 相连，再忽略边方向，得到一个三角形结构。所谓“道德化”，正是指让共同指向同一子节点的父节点“成婚”。

道德图的作用不是保留原有向图中全部精细语义，而是为后续的无向图因子分解与精确推断提供一个可操作的结构基础。

因子图

因子图是一种更细粒度的表示方式。它把变量节点与因子节点显式地区分开来，用二部图表示联合分布的乘积分解。若

p(x)∝s∏​fs​(xs​)

则因子图中每个变量对应一个变量节点，每个局部因子 fs​ 对应一个因子节点，因子节点与其所依赖的变量节点相连。

Intuition

因子图的优点在于：它直接表达“哪个局部函数依赖哪些变量”，因此比单纯的有向图或无向图更接近实际计算结构。消息传递算法在因子图上具有特别清晰的表达。

推断问题

推断的目标

推断的核心任务是在给定模型后计算感兴趣的概率量。常见目标包括：

  • 边缘概率：某个变量或变量子集本身的分布；
  • 条件概率：在给定观测后未观测变量的后验分布；
  • 最大后验估计（MAP）：最可能的隐变量配置或参数取值。

若记观测证据为 E，则最大后验问题可写为

x^=argxmax​p(x∣E)

精确推断与近似推断

精确推断方法要求得到严格正确的概率结果。典型方法包括变量消除、信念传播以及 Junction Tree。它们的共同思想是利用图结构把原本在高维空间中的求和、积分或最大化变成局部计算，但计算代价仍然会受到图复杂度，特别是树宽的显著影响。

近似推断方法在计算成本过高或图中存在复杂环路时使用。常见方法包括有环图上的 Loopy Belief Propagation、重要性采样、马尔可夫链蒙特卡罗方法以及变分推断。它们通过随机采样、分布近似或迭代消息传递来换取可计算性。

变量消除

变量消除是最基本的精确推断方法。它的思想是：只要某个变量在最终目标中不需要保留，就可通过局部求和或积分把该变量“消去”，并把中间结果保存为新的局部因子。

以链式结构 a→b→c→d 为例，若要求 p(d)，则从联合分布出发有

p(d)=a,b,c∑​p(a,b,c,d)

利用链式分解可写为

p(d)=a,b,c∑​p(a)p(b∣a)p(c∣b)p(d∣c)

进一步按消除顺序嵌套整理，可得

p(d)=c∑​p(d∣c)b∑​p(c∣b)a∑​p(b∣a)p(a)

Intuition

变量消除并不是对整个联合分布一次性求和，而是把高维求和重写为一系列低维局部计算。就像剥洋葱一样，一层一层地消去不需要的变量。每次消去一个变量，都会产生一个新的中间因子，供后续计算继续使用。

变量消除的效率高度依赖于消除顺序。不同顺序会导致中间因子的维数差异很大，从而显著影响时间与空间复杂度。寻找最优消除顺序一般是困难问题，其本质与图的树宽控制有关。

信念传播

基本思想

变量消除虽然能重用部分中间计算，但若对多个查询重复执行，仍可能产生大量重复工作。信念传播进一步把局部计算组织成沿边传递的消息，使得树形结构上的边缘概率计算可以高效复用。

在成对势函数形式的无向树上，若节点势为 ϕi​(xi​)，边势为 ϕij​(xi​,xj​)，则从节点 j 传向节点 i 的消息定义为

mj→i​(xi​)=xj​∑​ϕj​(xj​)ϕij​(xi​,xj​)k∈N(j)∖{i}∏​mk→j​(xj​)

其中 N(j) 表示节点 j 的邻居集合。该式表示：节点 j 把自己收到的所有其他邻居的信息先整合，再通过与 i 相连的边势投射给 i。

当所有相关消息都已收集后，节点 i 的边缘信念可写为

bi​(xi​)∝ϕi​(xi​)j∈N(i)∏​mj→i​(xi​)

在树结构上，若消息从叶节点向内收集，再从根向外分发，则每条边只需在两个方向上传递一次消息，就可得到所有节点的精确边缘分布。

Example

想象一棵家族树，你想知道每个人的姓氏分布概率。信念传播就像是一场“家族会议”：每个成员先听听父母和孩子传来的消息，再把汇总后的消息传回去。因为家族树没有循环（无环），所以消息只要来回传一次，每个人就能算出准确的概率。

适用条件与局限

标准信念传播在树图上给出精确结果。若图中存在环，则直接使用同样的消息更新得到的是 Loopy Belief Propagation。此时算法不一定收敛，即使收敛也不一定得到精确边缘分布，但在许多实际问题中仍可能表现良好。

Max-Product 算法

当推断目标不是求边缘分布，而是求最可能的变量配置时，可将信念传播中的求和替换为最大化，得到 Max-Product 算法。对边 j→i，其消息写为

mj→i​(xi​)=xj​max​ϕj​(xj​)ϕij​(xi​,xj​)k∈N(j)∖{i}∏​mk→j​(xj​)

该算法沿图传递的不是“所有可能配置的总权重”，而是“在给定边界变量取值时，内部子图所能达到的最大权重”。在树结构上，它可用于高效求解 MAP 配置。若在序列模型中使用这一思想，就得到经典的 Viterbi 算法。

Note

实际求解最大后验配置时，除了保留最大值本身，通常还需要记录使最大值达到的自变量取值，即回溯指针。仅有最大值而没有对应的回溯信息，无法在最后恢复出完整的最优配置。

概率图模型的整体理解

概率图模型的统一性体现在三个方面：

  1. 它把高维联合分布转化为图上的局部分解，使原本不可处理的概率对象获得结构化表达；
  2. 它把条件独立关系显式编码在图中，使“哪些信息相关、哪些信息可屏蔽”成为可读、可证明的结构性质；
  3. 它把推断与学习建立在局部计算之上，使边缘化、后验计算、最大后验估计以及参数学习都能够围绕相同的图结构展开。

Summary

有向图模型适合表达生成顺序、因果方向与逐层条件化结构；无向图模型适合表达对称相互作用、整体兼容性与能量约束；因子图则进一步把局部因子的计算结构显式化。无论采用哪种表示形式，其核心目标都不是图本身，而是利用图来控制高维概率问题的复杂度，并在表示、推断与学习之间建立统一的数学联系。
