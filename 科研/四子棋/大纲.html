Expertise Increases Planning Depth in Human Gameplay

0. 这篇文章到底做了什么

作者设计了一个复杂度足够高、但规则足够简单的新棋类任务 ” 四连棋 “（4×9），并提出一个 ” 启发式价值函数 + best-first 树搜索 + 剪枝 + 注意力遗漏（feature dropout）+ 噪声/失误 ” 的计算认知模型。模型不仅能预测人的落子，还能在单局层面预测反应时与注视位置分布；通过学习实验、时间压力实验、移动端大样本数据、图灵测试、记忆重建实验等，得到一致结论：水平越高/经验越多，规划（planning）越深，注意力遗漏越少；在实验室样本里 ” 启发式权重是否更正确 ” 并未随学习改善（甚至略降），但在移动端大样本里启发式质量会随经验上升（起点更差，提升空间更大）。

1. 任务：Four-in-a-row（4×9 四连棋）

1.1 规则与界面行为

  • 棋盘：4 行 × 9 列（共 36 格）。
  • 两名玩家轮流在空格落子（黑/白）。
  • 任一方先达成 ” 连续四子 “（横/竖/对角线）即胜。
  • 棋盘填满且无人四连则为和局。
  • 在线可游玩（作者提供网页版本）。

1.2 复杂度

  • 作者强调该任务拥有约 1.2×10^16 个非终局状态，复杂度远高于常见认知科学规划任务，因此不会轻易 ” 顶到天花板 “，更适合研究 ” 高手是否真的规划更深 “。

2. 计算认知模型：启发式评估 + Best-first 搜索（principal variation）+ 剪枝 + 注意力遗漏 + 噪声 + 失误

2.1 总体结构（每一步落子如何生成）

模型假设：在同一实验 session 内，一个人每一步落子由同一组参数生成；且每一步选择相互独立（在建模层面这么处理）。

每次轮到模型落子，它执行：

  1. 是否 ” 走神/失误 “(lapse)：以概率 λ 直接随机下一个合法空格。

  2. 注意力遗漏（feature dropout）：对启发式特征的 ” 每个具体实例 “，以概率 δ 独立丢弃；被丢弃的特征在本次落子整棵搜索树里都不会参与任何节点的价值计算。

  3. 迭代式树搜索：反复 ” 选点 → 扩展 → 回传 (minimax)” 直到：

    • 根节点的胜/负/和已经被 ” 确定 “（determined with certainty），或
    • 发生随机停止：每次迭代后以概率 γ 停止（从而总迭代次数服从几何分布）。

  4. 在根节点所有子节点中选 value 最大 的那个子节点对应的落子：

    • 形式上就是 argmax_{child of root} child.val。

2.2 价值函数（启发式评估）：用少量 ” 棋形特征 ” 快速估值

2.2.1 总体形式（” 自己特征 ” 减 ” 对手特征 ”）

作者以 ” 特征加权和 ” 的方式近似棋局价值。直观含义：值越大越利于黑方胜利。

一个简化表述是：

  • 价值函数由若干特征 φ_i 的加权和构成；
  • 以 ” 自己（self）” 与 ” 对手（opponent）” 的特征差来形成相对优势。

2.2.2 具体用了哪些特征（5 类）

模型包含 五个特征类型（并在图中把对称/同色的特征权重做了绑定约束）：

  1. centre（中心偏好）：越靠近棋盘中心越好。
  2. connected two-in-a-row（相邻二连）：两个相邻棋子，且周围还有足够空位使其有潜力扩展成四连。
  3. unconnected two-in-a-row（隔一格二连）：同一条长度为 4 的连续线段里出现两个不相邻棋子，另外两格为空（典型 ”..X.X..” 嵌入到四格线段的那种）。
  4. three-in-a-row（三连带一空）：长度为 4 的连续线段里已有 3 子 + 1 空；这是 ” 立即致胜威胁 “（轮到该方时可以直接补空成四连）。
  5. four-in-a-row（四连）：只有在已经胜利的状态里才出现。

这些棋形特征会在三种方向上计数：横、竖、对角。每个 ” 特征实例 ” 对应一个具体位置 (x,y) 和方向 o，是一个二值函数：出现为 1，不出现为 0。

2.2.3 Centre 特征的精确定义（公式层面）

中心项写成 ” 离中心越近贡献越大 “，具体是 距离的倒数求和差：

  • x_center 是棋盘中心坐标；
  • ||·|| 是欧氏距离。

则：

  • Vcentre​(s)=Σx∈Pieces(s,black)​∣∣x−xcenter​∣∣1​−Σx∈Pieces(s,white)​∣∣x−xcenter​∣∣1​

（注意这里写的是 ” 倒数距离 “，不是线性距离。）

2.2.4 Pattern 特征项 + 主动/被动缩放（active scaling）

除了 centre 外，其他棋形特征项会根据 ” 当前轮到谁走 ” 做缩放：

  • 当一个玩家 ” 马上要走子 “（active）时，某些特征（尤其是三连威胁）价值会更高；
  • 作者用一个缩放常数 C 来乘到 ” 轮到走子的那方 ” 的特征权重上（centre 特征不缩放）。

在公式里用 c_black, c_white 表示这次轮到谁走：

  • 若轮到黑走：c_black = C, c_white = 1
  • 若轮到白走：c_black = 1, c_white = C

2.2.5 节点价值噪声（Gaussian Value noise）

在树搜索过程中，每个节点的 V(s) 还会加上一个均值 0、方差 1 的高斯噪声项：

  • + N(0,1)
    这让模型能产生 ” 人类式波动 ” 和偶发次优选择。

2.3 树搜索：best-first（沿 Principal Variation 反复扩展）

2.3.1 节点结构

每个节点保存：

  • 一个状态 s（棋盘局面）
  • 该状态轮到的 active player
  • 当前对 s 的价值估计 val

节点初始值来自上面的 V(s)（带噪声、且考虑 feature dropout 之后的特征集合）。

2.3.2 选点（selection）：principal Variation

  • 使用 best-first 思路：每次沿着当前树里 ” 双方都走最优 ” 的路径（principal variation）从根一路走到叶子；
  • 选中这条路径上的叶子进行扩展；
  • 因为回传会改写节点值，principal variation 会动态改变，因此搜索会在不同分支间来回切换。

2.3.3 扩展（expansion）+ 剪枝（pruning）

对选中的叶子节点：

  1. 为该局面所有合法落子建立子节点（每个子节点对应一个落子后的新状态）。

  2. 初始化这些子节点的值为 V(s_child)（此时并不立刻判定终局）。

  3. 剪枝：对 ” 明显差 ” 的子节点直接删掉。规则是：

    • 令最佳候选子节点的值为 V_best；
    • 若某个子节点与 V_best 的差值超过阈值 θ，则被剪掉。

  4. 终局赋值在剪枝之后才做：

    • 黑胜：赋 V = 10000
    • 白胜：赋 V = -10000
    • 和局：赋 V = 0

一个关键后果（作者明确指出）：

  • 如果 w_four-in-a-row 不够大，或关键位置的 ” 四连 ” 特征实例刚好被 dropout 丢掉，模型可能在 ” 剪枝发生时 ” 没意识到某一步是立刻取胜/立刻防守，从而把真正的制胜步剪掉，出现很糟但 ” 很像人类 ” 的低级失误。

2.3.4 回传（backpropagation）：minimax（但只沿 PV 高效更新）

  • 使用 minimax 规则：
    • 若节点轮到黑走：节点值取子节点最大值
    • 若轮到白走：节点值取子节点最小值
  • 为提高效率，只更新 principal variation 上的节点，并按从叶到根的顺序回传。
  • 回传后，节点值被解释为：在当前已展开的树中，对 ” 双方最优应对 ” 结果的最好估计。

2.3.5 停止规则（stopping）

  • 若根节点已经能确定胜/负/和，则停止；
  • 否则每次迭代后以概率 γ 停止；因此迭代次数呈几何分布。

作者还提到：当把模型当作对弈 AI 时，会把迭代次数 N 映射成 ” 思考时间 ”：

  • t = a * sqrt(N) + b，其中 a = 4s，b = 0.5s。

2.4 模型参数（主模型 10 个参数）

主模型参数一共 10 个：

  1. 剪枝阈值 θ

  2. 停止概率 γ

  3. lapse 概率 λ

  4. feature drop 概率 δ

  5. active scaling 常数 C

  6. 五个特征权重：

    • w_centre
    • w_connected-two-in-a-row
    • w_unconnected-two-in-a-row
    • w_three-in-a-row
    • w_four-in-a-row

作者强调：没有把 ” 价值噪声方差 σ²” 当作独立参数，因为把噪声从 N(0,1) 改成 N(0,σ²) 的效果，等价于把 θ 和全部 w 同时按 1/σ 缩放，会导致 σ、θ、w 不可识别。

2.5 用模型计算 ” 博弈论意义上的真值 “（game-theoretic value）

作者用 best-first 搜索在 ” 无噪声、无剪枝、默认权重 ” 的设置下，逼近从某局面开始双方完美下法的结果（胜/负/和）：

  • 理论上迭代次数 → ∞ 时，根节点值收敛到博弈论真值。
  • 实际上作者经验上认为 200,000 次迭代对几乎所有局面足够。
  • 若 200,000 次仍无法得到 ” 确定结果 “，则把该局面真值设为 0（和局）。

2.6 替代模型（用于验证 ” 哪些组件必不可少 ”）

作者比较了 22 个替代模型，三大类：

2.6.1 Lesion（删组件：把某参数固定为常数）

  • No centre：w_centre = 0
  • No connected two-in-a-row：对应权重为 0
  • No unconnected two-in-a-row：对应权重为 0
  • No three-in-a-row：对应权重为 0
  • No four-in-a-row：对应权重为 0
  • No feature drop：δ = 0
  • No active scaling：C = 1
  • No pruning：θ = 20000（足够大，几乎永不剪枝）
  • No tree：γ = 1（只做 1 次迭代，相当于只展开根节点；此时剪枝也不起作用，所以 θ 不再是参数）

2.6.2 Modifications（替换某组件的实现方式）

  • Fixed iterations：停止改为固定迭代次数 N。
  • Fixed depth：改为把每个分支都探索到固定深度 D。
  • Fixed branching：不按阈值剪枝，而是每个节点保留 K 个最优子节点（白走时保留 ” 最小值 ” 那侧的 K 个）；子节点少于 K 则不剪。
  • Square dropping：去掉 feature drop，改为扩展时对每个子节点以概率 ε 随机剪掉（在价值剪枝之前），形成 ” 格子/走法层面 ” 的随机漏看。
  • Optimal weights：把 {w_i} 限制成一个固定向量（不因人而异），该向量通过最大化 tanh(V(s)/20) 与博弈论真值之间的 Pearson 相关得到。

2.6.3 把搜索改成 MCTS（Monte Carlo Tree Search）

  • 评估节点不再直接用 V(s)，而是做一次 rollout（从 s 开始模拟一局）。
  • rollout 中双方采取 ” 近视策略 ”：
    • 在状态 s′，黑选使 V(s′+m) 最大的落子；白选使其最小的落子。
  • rollout 结果映射为值：
    • 黑胜：1；白胜：0；和：1/2
  • 因为 V 本身含噪声，rollout 和近视策略也随机。
  • 回传不用 minimax，而是对后代 rollouts 的平均（让中间节点值等于其后代 rollout 结果均值）。
  • 选点规则从 argmax child.val 改成 UCB：
    • m = argmax_{c∈children(root)} [ c.val + C_exp * sqrt( log(n.N_rollouts) / c.N_rollouts ) ]
    • n.N_rollouts：节点 n 及其后代一共做过多少次 rollout
    • c.N_rollouts：子节点 c 及其后代一共做过多少次 rollout
    • C_exp 控制 ” 探索 vs 利用 ” 权衡
  • 最终落子：选根节点子节点中 N_rollouts 最大者。

2.6.4 Extensions（增加新参数/新特征）

  • Orientation-dependent weights：对竖向/对角向的特征权重分别乘以缩放常数 c_vert、c_diag。
  • Orientation-dependent dropping：让横/竖/对角三种方向的 drop rate 各不相同。
  • Type-dependent dropping：让 drop rate 随特征类型不同而不同。
  • Triangle model：加入一个 ” 统计某组 3 子图案出现次数 ” 的新特征。
  • Opponent scaling：加入 c_opp，对 ” 属于对手的特征权重 ” 做统一缩放；作者强调这与 active scaling 不同：前者不随轮到谁走而变，后者是 ” 谁走谁更重要 ” 的自适应缩放。

4. 实验 1：人 - 人对战（无时间限制）+ 模型落子预测

4.1 过程细节（人 - 人对战）

  • 40 人两两配对：
    • 一起签同意书、一起听任务说明；
    • 之后被分到不同房间，通过在线界面对弈；
    • 50 分钟结束并完成最后一局后填写问卷，然后当场现金支付（12 美元），最后离开。
    • 参与者只在对弈前后有社交互动，对弈过程中不互动。
  • 对弈设置：
    • 每局后双方交换先后手/颜色。
    • 每局后弹窗显示双方名字、当前比分和 ” 继续 ” 按钮；只有双方都点击继续才进入下一局。
    • 每次任一方落子，界面会播放轻微点击音效。
    • 允许 ” 求和 ”：
      • 玩家可在自己回合不落子而提出和局；
      • 对手屏幕弹窗选择接受/拒绝；
      • 接受则立即结束；拒绝则弹窗消失，提出者继续落子；
      • 不限制提出次数（甚至同一步可多次），但实际提出很少。
    • 全程无时间限制。

4.2 拟合与结果（落子预测）

  • 对每个参与者单独估计参数（包括特征权重、feature drop rate、决策树规模、剪枝阈值、噪声水平等），采用 五折交叉验证。
  • 模型对 out-of-sample 选择预测准确率：40.8% ± 1.4%（跨参与者均值±SEM）；显著高于随机猜空格（检验统计量 t39=26，p<0.001）。
  • 图中还给出：
    • 单个样例局面：模型对下一步落子形成概率分布（热度/红色阴影），并与真实选择对比。
    • 每人准确率排序柱/点图；每人平均试次数量标注为 n = 137.1 ± 9.6（平均±SEM）。
  • 作者还做了大量模型规格对比与组件消融，结论是：特征型价值函数 + 树搜索 + 注意力遗漏机制对预测人类选择是关键组件；主模型被选为该类模型中更简洁的代表。

5. 实验 2：泛化实验（对电脑 + 2AFC + 局面评估）

5.1 实验流程

  • n=40，个人完成：

    1. 与电脑对战 30 分钟
    2. 84 次 2AFC：给定局面与两种候选落子，点选更偏好的落子（无时间限制）
    3. 84 次局面评估：给定局面，用 7 点量表评估自己的胜率/赢面（无时间限制）
      结束后解释与付款（12 美元）。

5.2 人 - 机对战界面差异

  • 与人 - 人对战界面基本一致，但：
    • 局间弹窗不显示名字与比分
    • 移除 ” 求和 ” 按钮

5.3 电脑对手的生成与匹配（非常细）

  • 电脑算法 ” 类似行为模型 “，但有三处修改：

    1. 剪枝规则使用 fixed branching 模型那套（保留 K 个最佳子节点）。
    2. 加入 ” 对手特征缩放 “（opponent scaling）与 ” 方向依赖权重 “（orientation-dependent weights），但不包含 “active vs passive 的另一种缩放形式（no active scaling 模型里那种对照项）”。
    3. 使用了一个 ” 略不同 ” 的特征集合（与人类拟合用的特征集略有差别）。

  • 为了让电脑 ” 像在思考 “，每一步电脑落子加入一个人为延迟：延迟时间随该步搜索迭代次数单调增加，使得简单局面下得更快、困难局面下得更慢（但不改变其决策）。

电脑对手参数族如何构造（从人类拟合出发）

  1. 先把行为模型拟合到人 - 人对战里的每位人类参与者，得到一组参数向量。

  2. 对每个参数向量再造 8 个变体：对以下三个量做 ” 翻倍/减半 ” 并组合：

    • 平均树大小（tree size）翻倍
    • 价值噪声减半
    • feature drop rate 减半
    • 以及它们的任意组合

  3. 用这些参数向量跑 ” 全员互搏 ” 的电脑 - 电脑锦标赛，用 Elo 排序。

  4. 从 Elo 轴上挑 30 个对手，让它们强度覆盖从 ” 略弱于最弱人类 ” 到 ” 略强于最强人类 ” 的区间，并尽量均匀。

  5. 把 30 个对手分成 6 个 level，每级 5 个对手。

  6. 用 one-up one-down 阶梯法匹配受试：

    • 从 level 3 开始；
    • 每局在当前 level 的 5 个对手中随机抽一个；
    • 结果好则升一级、差则降一级（阶梯细节按经典 one-up one-down）。

5.4 2AFC 与评估任务的 ” 刺激选择 “（信息论挑题）

  • 2AFC：
    • 目标：既让人觉得难，也让模型预测/参数约束更强。
    • 作者在 ” 人类可能参数集合 ” 内，选择能最大化 ” 所选落子与模型参数之间互信息 ” 的局面与候选走法组合。
    • 同时计算候选走法的 ” 客观/博弈论价值 “，并让不同 ” 走法真值组合类型 ” 均衡出现（例如：两步都赢、一个赢一个和、两步都和……），每一类型出现 14 次。
    • 对每个受试呈现相同的一组局面，顺序随机。
  • 局面评估：
    • 7 点按钮，首/中/末分别标注 ” 输/均势/赢 ”。
    • 局面挑选方式与 2AFC 类似，但最后筛选时确保呈现局面的博弈论真值在 ” 赢/输/和 ” 三类上均匀分布：各 28 个。

5.5 泛化结果

  • 用 ” 人 - 机对战阶段的落子数据 ” 拟合每个受试参数，然后去预测其：
    • 2AFC 选择：58.6% ± 1.0%（高于 50%），t39=8.3，p<0.001。
    • 局面评估：预测值与真实评估的相关 ρ = 0.377 ± 0.039，t39=9.6，p<0.001。
  • 作者还展示：模型能超过一个 ” 总是下客观正确步、随机打破平局 ” 的 oracle（说明模型捕捉了个体偏好，而非只追求客观最优）。

6. 图灵测试：人能不能分辨 ” 真实人类棋局片段 ” Vs ” 模型生成片段 “

6.1 设计（两天两 session）

  • 第一天：受试与电脑对战 60 分钟。电脑对手直接使用 ” 主模型 + 人 - 人对战中某个真实参与者的拟合参数 ”。
  • 第二天：完成 180 个分类 trial：
    • 每 trial 看一段棋局视频（平均长度 9.38 步）。
    • 视频来源二选一：
      • 人 - 人对战真实两人棋局片段
      • 两个模型代理（参数取自对应人类玩家拟合值）从同一起始局面继续下出来的片段
    • 受试点击 ” 播放 ” 开始；播放速度固定 1.8 秒/步；之后用滑条判断：
      • 左端：” 肯定是电脑 ”
      • 中间：” 没把握 ”
      • 右端：” 肯定是人 ”
    • 每次回答后给即时反馈：“Correct!” 或 “Incorrect”。

6.2 视频片段如何抽样（非常具体）

  • 对每盘人 - 人对战棋局：

    1. 起始步号从几何分布抽样，rate = 0.15；丢弃之前的所有步，片段从该步开始。
    2. 片段最大长度从另一几何分布抽样，rate = 0.1；从起点不断添加后续步，直到超过最大长度或棋局结束。

  • 对应的电脑 - 电脑片段：

    • 从同一个起始局面开始；
    • 用相同的最大长度阈值；
    • 让两个模型代理继续对弈生成后续步，直到达到长度或结束。

  • 因为起始局面来自真实棋局、播放速度恒定且不给反应时线索，受试只能依据 ” 走法序列本身 ” 判断。

  • 最终挑：

    • 90 盘棋用于 ” 人类片段 ”
    • 90 盘棋用于 ” 模型片段 ”
    • 合计 180 段视频。

6.3 结果（两层结果一起给）

  • 主文报告：整体区分准确率 55.4%（接近随机 50%，说明模型 ” 像人 ”）。
  • 扩展数据里更细：
    • 对 “1 步视频 “（共有 8 个）受试基本在随机水平；
    • 准确率显著超过 50% 主要发生在 长度 > 10 步 的片段；
    • 混合效应线性回归估计：每多观察一步，准确率仅增加 0.33% ± 0.10%；
    • 在 180 个视频中，没有任何一个视频能让 全部 30 名观察者一致判断；
    • 有相当一部分视频（63/180）甚至 ” 多数观察者判断错误 “。

7. 过程数据验证：反应时（RT）与眼动

7.1 反应时预测（从 ” 树的结构 ” 预测 RT）

  • 做法：
    • 先用 choice 数据拟合参数；
    • 在 best-first 搜索上再加一个 “early-stopping 规则 “：当继续迭代不太可能改变最终决策时提前停（用于更贴近人的停止策略）；
    • 用每步构建出来的 ” 决策树 ” 作为 RT 的预测变量。
  • 分析细节：
    • 合并人 - 人对战数据做 RT 分析时，排除棋盘上棋子数 <6 或 >30 的局面。
    • 以对数空间比较预测与真实 RT。
  • 结果：
    • 跨参与者预测 - 真实 RT 的平均相关：ρ = 0.351 ± 0.029，t39=12，p<0.001。

7.2 眼动实验（n=10）

设备与显示

  • EyeLink 1000（远程红外视频眼动），采样率 1 kHz，精度约 0.01°。
  • 眼动软件设置：Heuristic filter ON。
  • 显示器：21 英寸 Sony GDMF520 CRT；分辨率 1280×960；刷新率 100 Hz。
  • 头托：眼 - 屏距离约 57 cm。
  • 只记录事件（fixation / saccade / blink），即数据是这些事件的时间序列。

校准（两层校准）

  • 每 session 先做系统自带 9 点校准。
  • 另外在 ” 对电脑对战 ” 环节前后各加一次自定义校准：
    • 显示空棋盘，在左下角格子放一个白棋，上面叠一个注视十字；
    • 受试注视稳定后按空格；
    • 十字 + 白棋向右移动一格；重复；
    • 跨 4 行移动，最终获得每行的 9 个注视坐标与按键时间戳。

” 人类注意分布 ” 如何从眼动构造

  • 把注视轨迹用高斯滤波做卷积得到空间分布；
  • 截断到 ” 未落子的空格 ”；
  • 再按时间平均，得到该 trial 的 ” overt attention 在空格上的概率分布 ”。

” 模型注意分布 ” 如何定义

  • 用模型搜索过程中 ” 访问到的格子/节点 ” 形成一个对应分布（本质上是模型搜索的访问轨迹在棋盘空格上的投影）。

结果

  • 人的注意分布与模型搜索访问分布高度相似：
    • 跨参与者平均相关：ρ = 0.535 ± 0.024，t9=21，p<0.001。
  • 更细的机制解释：
    • 这种相关性由 ” 树中分支偶尔能到达约 7 步深 ” 所驱动；
    • 线性回归分析显示：用模型 principal variation 在各深度包含的格子分布去预测人的注意分布时，回归系数在 深度 ≤ 7 显著大于 0，且在更接近根部（深度靠近 1）时最大。
    • 还挑出一些局面：若去掉 feature drop（No feature drop 模型）会给人的真实落子很低概率；这些局面里，受试在思考时往往根本不看 ” 模型认为应该下的格子 “，支持 ” 人确实漏看了关键四连相关特征实例 ” 的解释。

8. 专长/经验如何改变规划：学习实验 + 时间压力实验 + 指标定义

8.1 学习实验（n=30，5 次 session，间隔 ≤2 天）

结构

  • 共 5 次 session，要求相邻 session 间隔不超过 2 天。
  • 第 1/3/5 次：
    • 与电脑对战 30 分钟
    • 60 次 2AFC + 60 次局面评估
  • 第 2/4 次：
    • 与电脑对战整整 60 分钟
  • 2AFC/评估局面挑选仍按 ” 信息最大化 ” 思路，但：
    • 先挑出 180 个局面，分成 3 组各 60；
    • 不同受试在第 1/3/5 天看到的 3 组顺序做了反平衡（counterbalanced）。
  • 支付：每次 12 美元；完成 5 次后额外 12 美元完成奖金。

实力度量（Elo）

  • 用 Elo 量化每个 session 的水平，且把所有实验的 Elo 放到同一基准上（靠 Bayeselo + 统一数据库/对手校准）。

主要结果（随 Session 增长）

  • Elo 随 session 上升：线性回归 β = 21.6 ± 4.6（p<0.001）。
  • 作者把每次拟合参数映射到三个 ” 可解释指标 “（见 8.3），并观察：
    • 规划深度上升：β = 0.255 ± 0.061（p<0.001）。
    • 注意力遗漏下降（feature drop rate）：β = −0.0119 ± 0.0028（p<0.001）。
    • 启发式质量不升反降一点：β = −0.0067 ± 0.0020（p=0.0012）。
  • session 1 到 session 5 的总 Elo 增益：90 ± 26；分解为：
    • 规划带来 +36 ± 11
    • 注意力改善带来 +46 ± 12
    • 启发式质量变化带来 −6.6 ± 3.5
  • 另外核查：后期并不是下得更慢导致 ” 看起来规划更深 “，反而总体更快（因此 ” 规划深度上升 ” 不是靠拖延时间堆出来的）。

8.2 时间压力实验（n=30；每步限时 5/10/20 秒）

结构与界面细节（非常具体）

  • 与电脑对战 50 分钟；流程与泛化实验一致，但每局有固定时限：
    • 每步限时为 5 秒、10 秒或 20 秒
    • 同一局内时限固定，不同局之间随机切换
    • 人和电脑都遵守时限
    • 超时立即判负、结束该局
  • 电脑 ” 思考时间 ” 显示为界面暂停，确保电脑最多用到其分配时间的 80%（但强调：电脑决策本身只需极短时间，暂停不影响落子选择）。
  • 时间提示 UI：
    • 棋盘右侧显示彩色条形计时器，随思考时间线性缩短；下方显示文本倒计时（秒）。
    • 20 秒条件：条初始高度等于棋盘高度；0~10 秒剩余为绿；剩 10 秒变蓝；剩 5 秒变红。
    • 还会在剩余 2、1、0 秒分别播放短蜂鸣音，音高依次升高。
    • 10 秒条件：计时条从 “20 秒条件过了 10 秒后的状态 ” 开始（蓝色、半棋盘高度）。
    • 5 秒条件：从 “20 秒条件过了 15 秒后的状态 ” 开始（红色、四分之一棋盘高度）。
    • 电脑思考时，在棋盘左侧显示同样的计时器。
  • 超时发生很少：仅 1.87%（33 / 1,766 局）因超时判负。

结果

  • 规划深度整体低于学习实验，但随允许时间更长而上升：β = 0.042 ± 0.018（p=0.019）。
  • 但 Elo 没有随更长时限提升：β = −2.0 ± 1.6（p=0.21）。
  • 模型给出的解释：最宽松时限下，人反而更容易 ” 漏看特征/注意力走神 ”：
    • feature drop 随时限放宽上升：β = 0.0027 ± 0.0010（p=0.009）
    • 漏看的代价抵消了更深搜索的收益。
  • 启发式质量随时限变化不显著：β = 0.00086 ± 0.00056（p=0.13）。
  • 另一个验证：时限操控确实显著改变了受试反应时（平均只用掉可用时间的一部分，但可用时间更长时反应时仍显著上升）。

8.3 三个 ” 派生指标 “（derived metrics）：把 10 参数压缩成 3 个更可解释量

作者把每个拟合得到的 10 参数映射为：

  1. Planning depth（规划深度）

    • 定义：模型决策树里 principal variation（双方都走当前树里最优步的那条路径）从根到叶的长度（叶的深度）。
    • 不是直接从每盘棋实时树里读，而是：
      • 在一组固定 probe positions 上跑模型模拟；
      • probe positions 取自人 - 人对战实验中出现过的全部局面，共 5,482 个；
      • 每个局面做 10 次模拟落子；
      • 对所有局面与模拟取平均，得到该参数向量对应的 planning depth。
    • 解释：它是 ” 模型（以及被解释的人）大致往后想了多少步 ” 的近似量。

  2. Feature drop rate（注意力遗漏率）

    • 直接等于参数 δ。

  3. Heuristic quality（启发式质量）

    • 做法：
      • 在同一组 5,482 probe positions 上计算 V(s,w)；
      • 取 tanh(V(s,w)/20)（把值压到更稳定的范围）；
      • 与该局面的 ” 博弈论真值 ” 做 Pearson 相关；
    • 注意：只依赖 w 与 C（与停止/剪枝等搜索参数无关）。

作者还给出派生量与原参数的依赖关系（在大量人类拟合参数向量上统计）：

  • planning depth 主要由停止概率 γ 决定：
    • 与 γ 的 Spearman 相关 ρ = −0.87（p<0.001）
    • 与剪枝阈值 θ 也有小幅相关 ρ = −0.21（p<0.001）
  • heuristic quality 与 w_three / w_connected 有相关：
    • Spearman ρ = 0.55（p<0.001），但作者强调这种关系不算非常强，其他权重也会影响。
  • 总结：三个派生指标把 10 参数 ” 切开 ” 了：
    • planning depth 主要对应（θ, γ）
    • feature drop rate 对应 δ
    • heuristic quality 对应（w, C）

9. 大规模移动端数据（Peak 合作）

9.1 数据收集与伦理

  • 平台：Peak 移动端应用（视觉上更丰富、棋盘竖向显示）。
  • 用户注册时同意隐私政策，其中包括：可能把聚合且匿名化的数据共享给大学等第三方。
  • NYU IRB 判定该研究为 “exempt”，无需额外同意流程。

9.2 数据规模与对手匹配

  • 共收集 10,874,547 局，来自 1,234,844 名唯一用户。
  • 用户永远先手（先下）。
  • 用户与电脑对手对弈；电脑实现主模型的一个版本，参数来自实验室多项实验拟合并做了适配。
  • 电脑对手生成流程与实验室的人 - 机实验一致，但因电脑永远后手进行了重新校准。
  • 电脑对手分成 7 个强度类别，并根据用户的胜负 ” 战绩轨迹 ” 动态匹配对手强度。

9.3 分析子样本与分块

  • 出于计算限制，从全体中随机抽取 1,000 名至少下过 100 局的用户。
  • 把每位用户的前 100 局分成 5 个 block，每 block 20 局，以对齐实验室学习实验的经验量级。

9.4 移动端结果（随经验增长）

  • Elo 上升：β = 1.13 ± 0.04（p<0.001）
  • planning depth 上升：β = 0.0108 ± 0.0010（p<0.001）
  • feature drop rate 下降：β = −2.58×10^−4 ± 4.7×10^−5（p<0.001）
  • heuristic quality 上升：6.12×10^−4 ± 4.2×10^−5（p<0.001）
  • 也做了 ” 不是因为变慢才显得更深 ” 的检验：planning depth 上升不由更慢的下法解释。
  • 作者还指出一个跨场景差异（按文中原句的数值顺序记录）：
    • 移动端前 20 局的 heuristic quality 被描述为 ” 明显低于 ” 实验室首个 session，但括号里给出的两数是 0.5301 ± 0.0098 vs 0.4788 ± 0.0044（t999=4.7，p<0.001），并由此推断移动端起点更差、提升空间更大，而实验室可能接近 ” 启发式质量天花板 “。

10. ” 规划深度 4–6 步 ” 到底意味着什么（作者专门澄清）

  • 模型对个体选择最匹配时，planning depth 常在 4 到 6。
  • 作者强调这不代表人对未来每一步都同等具体：
    • 因为模型沿树分支累积价值噪声，越往后越 ” 松散/不确定 ”；
    • 前几步更具体，后几步更模糊。
  • 模型的停止规则是随机几何停止，因此经常会 ” 继续搜索但最终决策没变 “；人类停止更策略化、更接近资源理性意义上的近最优。
  • 加入 early-stopping 规则后，模型估计的 planning depth 会更低；但 ” 专长导致更深规划 ” 的核心结论对多种停止阈值都稳健。

11. 记忆与重建实验：高手是否更像 ” 模式识别机器 “

11.1 受试分组

  • 两组各 19 人（共 38）：
    • ” 有经验组 “：曾完成学习实验，且在 4 周内；
    • ” 无经验组 “：没玩过该游戏，被告知任务是记忆 ” 方格与圆点图案 “。

11.2 单 Trial 流程与操作

  • 展示某个局面 10 秒
  • 空白棋盘 1 秒
  • 进入重建界面（无时间限制）：
    • 右键：放置/移除黑子
    • 左键：放置/移除白子
    • 点击 submit 表示完成
  • 提交后反馈：36 格（含空格）中有多少比例重建正确。

11.3 刺激（96 个局面）如何构造

  • 每人重建同一组 96 个局面，顺序随机，整个实验约 1 小时。

  • 96 分为两套各 48：

    1. 来自人 - 人对战真实棋局的局面：

      • 控制局面棋子数从 11 到 18
      • 每个棋子数随机抽 6 个局面（因此 8 个棋子数量 × 6 = 48）

    2. 程序生成的随机局面：

      • 精确匹配 ” 棋子数量分布 ”
      • 近似匹配 ” 各格子被占用的边际分布 “（marginal distribution of occupied squares）

11.4 结果要点（含具体数值）

  • 错误类型（专家 vs 新手）：
    • extra piece：专家略差（β = 0.0071 ± 0.0031，p = 0.049）
    • missed piece：专家明显更好（β = 0.037 ± 0.006，p < 0.001）
    • wrong color：专家明显更好（β = 0.019 ± 0.003，p < 0.001）
  • 重建时间：
    • 专家平均用时更长（β = 2.73 ± 0.57，p < 0.001）
    • 作者因此提醒：总体错误率优势可能掺杂速度 - 准确性权衡；但进一步分析显示，专家对 ” 与游戏相关的特征 “（例如三连威胁）确实在相近时间下更准确。
  • 例子局面（底行存在一个三连威胁）：
    • 新手正确重建该三连的概率：42.1%
    • 专家：84.2%
  • 总体解释：人的棋盘表征更像 ” 以游戏相关特征为单位 ” 的表征；专家在这类特征上更精准，从而支持 ” 更快/更可靠的局面评估 → 每单位时间可评估更多节点 → 规划更深 “。

12. 统一 Elo 标尺：Bayeselo 的用法（跨实验对齐强度）

  • 作者用 Bayeselo 实现 Elo（并把实验室人类与电脑对手放进同一体系）。
  • 具体做法：
    • 建一个数据库，包含：
      • 全部 ” 人 - 机对战 ” 棋局
      • 一个模拟的 ” 机 - 机锦标赛 “：每个电脑对手与其他所有电脑对手（包括自己）各下 1 局
    • 锦标赛里包含：
      • 泛化/学习/时间压力实验里用的全部 AI 对手
      • 移动端 app 用的 AI 对手
  • 这样 Elo 在不同实验中是同一刻度。

13. 模型拟合：为什么这么贵、怎么拟合

13.1 为什么解析似然不可得

要算精确 log-likelihood，需要对多个隐变量做边缘化，作者认为不可行，包括：

  • 哪些特征实例被 dropout 丢掉
  • 树搜索的迭代次数（随机停止）
  • 每个节点的价值噪声实现

因此不能限制在 ” 可解析似然 ” 的模型族里，否则噪声结构等会被迫简化到很不现实。

13.2 似然估计与优化

  • log-likelihood 用 inverse binomial sampling 估计：本质上用模型生成的模拟数据与真实数据对比来估计似然；估计无偏但有噪声。
  • 因为无法获得梯度，用 multilevel coordinate search 做无梯度优化。
  • 用 五折交叉验证比较模型以减轻过拟合。

13.3 计算成本与硬件

  • 拟合单个参与者、单个模型约需 10^14 次浮点运算量级。
  • 在 NYU 高性能集群上跑（Intel Xeon E5-2690v2，3.0 GHz）：
    • inverse binomial sampling 做了并行实现，使用 20 核。
    • 单人单模型拟合大约 1 小时。

14. 扩展数据的关键内容（模型验证/稳健性/可解释性）

14.1 模型比较（交叉验证 Loglik per move）

  • 作者把替代模型分为：
    • 删组件（lesions）
    • 替换实现（modifications）
    • 增加组件/参数（extensions）
  • 用交叉验证的 “log-likelihood per move” 做比较，并在多个数据集上分别报告：
    • 汇总（N=330）
    • 人 - 人对战（N=40）
    • 泛化（N=40）
    • 眼动（N=10）
    • 学习（N=150）
    • 时间压力（N=90）
  • 结论导向：主模型在各数据集上总体表现最好或接近最好；删掉树搜索/删掉注意力遗漏/删掉关键特征会明显变差，从而支持主模型选择。

14.2 用 ” 总结统计量 ” 验证（不只靠逐步预测）

  • 作者把对局按棋子数进程来算一系列 summary statistics（对比：人类 vs 主模型 vs 随机落子），包括但不限于：
    • 落子到棋盘中心的距离
    • 自己棋子平均距离、对手棋子平均距离
    • 自己/对手的 ” 质心（center-of-mass）距离 ”
    • 自己/对手的相邻棋子数量（neighbouring pieces）
    • 造成的威胁数量（threats made）
    • 防守/化解的威胁数量（threats defended）
  • 这些统计量上，人类显著偏离随机；主模型能贴近人类曲线。

14.3 个体差异也能预测（主模型不仅拟合总体平均）

在 ” 同一批统计量 ” 的个体层面散点图中，主模型预测与真实个体差异高度相关（每点一个参与者），并报告了 Pearson ρ：

  • 到对手质心距离：ρ=0.85

  • 对手相邻棋子数：ρ=0.83

  • 威胁防守数：ρ=0.89

14.4 组件 ” 到底解释了什么局面 “（用例局面展示）

作者挑了两类 ” 主模型与消融模型预测差异最大的局面 ”：

  • 去掉树搜索（No tree）会与主模型在某些局面上产生巨大分歧：通常是需要往后算很多步才知道某个选择是输/和/赢；例如作者指出某一局面里 No tree 喜欢的步是必败，而主模型选择能达成和局，但这依赖一个必须深入计算才能发现的 10 步强制序列。
  • 去掉 feature drop（No feature drop）解释不了人类某些 ” 漏看一手制胜/漏防一手致命 ” 的选择倾向，feature drop 机制正是为了抓住这种 ” 选择性注意力漏检 “。

14.5 实力相关性与反应时（更细汇总）

  • 在学习 + 时间压力数据中：
    • Elo 与 planning depth 正相关：ρ=0.62（p<0.001）
    • Elo 与 feature drop rate 负相关：ρ=−0.73（p<0.001）
    • Elo 与 heuristic quality 的相关不显著：ρ=0.11（p=0.088）
  • 学习实验中，反应时随 session 变化呈 ” 略变快 ” 的趋势（因此 planning depth 上升不是因为更慢）。
  • 时间压力实验中，时限确实把反应时拉长（尽管平均只用了部分可用时间）。

15. 讨论与解释框架（作者的核心解释）

  • 更强的玩家之所以更强，模型层面主要体现在：

    1. 更深的树搜索（更深规划）
    2. 更少的注意力遗漏（更少 feature dropout）

  • 在实验室数据里，玩家一开始就带有 ” 相对正确的特征归因/归纳偏置 “，因此 heuristic quality 可能接近上限，学习中并未再提升；移动端用户起点更差，因此能观察到 heuristic quality 的提升。

  • 关于 ” 更深规划 vs 更强模式识别 ” 的经典争论：

    • 作者认为二者并不矛盾：记忆/重建实验显示专家更擅长以 ” 游戏相关特征 ” 编码棋盘；
    • 这会提升处理速度：单位时间可做更多局面评估，因此也能推动更深规划；
    • 因此 ” 模式识别的提升 ” 可以通过 ” 更快的评估 → 更深的搜索 ” 间接表现为 ” 更深规划 “，而不仅仅是 ” 直接替代搜索 “。

16. 推广、限制与未来方向（作者明确写出的）

  • 推广猜想：在更复杂游戏或更自然的规划任务中，专长可能同样提升注意力与搜索；复杂游戏里还可能存在 ” 需要长期经验才能学到的非显然特征 “，从而 heuristic quality 的提升会更显著。
  • 特征发现：在复杂棋类中，可以考虑用训练好的神经网络内部表征来挖掘候选特征，再塞进类似的认知模型做检验。
  • 学习机制建模：未来可用 ” 深度强化学习 + 前向规划 ” 类算法来解释人类随经验的变化，并计划在更大规模（百万级用户）的移动端数据上检验。
  • 任务范围限制：当前模型只覆盖确定性两人零和博弈；随机或多人博弈可能需要额外机制。
  • 神经层面的预测（作者提出的假设型观点）：
    • 未来状态价值可能与奖励/价值相关脑区的神经活动相关；
    • 当玩家思考时，神经活动的时间过程可能反映搜索迭代中 ” 根节点价值随迭代更新 ” 的动态轨迹。
