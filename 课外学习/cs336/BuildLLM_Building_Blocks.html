Basic Building Blocks

Parameter Initialization

在深度学习框架中，” 层 ” 的本质是对状态（权重）和计算（前向传播）的封装。手动构建模块时，必须严谨地管理这些状态的生命周期。

  • 系统实现机制：

    • 注册机制 (Auto-Registry)：自定义层必须继承 torch.nn.Module 并首行调用 super().__init__()。这不仅仅是 Python 的继承语法，更是为了激活 PyTorch 的树状管理机制，使得 model.parameters() 能够递归地扫描到所有子模块中的 nn.Parameter，从而让优化器（Optimizer）能够更新它们。
    • 参数包装：必须使用 torch.nn.Parameter 包装原始 Tensor，将其标记为 ” 需要计算梯度 ” 的叶子节点。

  • 初始化分布 (Initialization Strategy)：

    为了防止训练初期的梯度消失或爆炸，不同类型的层需要特定的统计分布初始化：

    • Linear Weights (线性层)：使用截断正态分布 (Truncated Normal)。标准差设为 σ=din​+dout​2​，通常在 [−3.0,3.0] 之间截断。这保证了输入输出方差的一致性。
    • Embedding (嵌入层)：使用截断正态分布，σ=1.0。
    • RMSNorm (归一化层)：其缩放参数 γ (Gain) 初始化为 1，意味着初始状态下不改变输入的幅度。

Linear Module

这是神经网络中最基础的 ” 突触连接束 ”。

  • 认知视角：

    Linear 层本身不包含 ” 神经元 “（即非线性激活），它只负责信息的线性重组。如果说神经网络是乐高城堡，Linear 层就是积木块的物理形状，决定了下一层能接收什么维度的信息。

  • 架构演进：

    • 无偏置 (No Bias)：遵循 PaLM 和 Llama 等现代 LLM 的设计哲学，去除 Bias 项。这不仅节省了参数，还被证明对模型性能无负面影响，甚至能提升稳定性。

  • 内存布局 (Memory Ordering)：

    • 直觉 vs 实现：数学上我们常写 y=Wx（列向量乘法），但在计算机内存中，数据通常是行优先存储的 (Batch,Seq,Din​)。
    • 权重形状：为了符合直觉并避免不必要的转置操作，我们将权重 W 存储为 (in_features, out_features)。
    • 计算流：执行 X@W（矩阵乘法）。这里利用了矩阵乘法的结合律，直接将最后一维 (Din​) 映射为 (Dout​)。

Embeding Module

这是将离散的符号世界（Symbolic）转换为连续的向量世界（Connectionist）的桥梁。

  • 数学本质：

    它等价于一个 One-Hot 向量与权重矩阵的矩阵乘法。

y=OneHot(id)×W

  • 系统优化：

    • 查表 (Lookup)：由于 One-Hot 向量极其稀疏（只有一位是1，其余全0），做矩阵乘法是极大的算力浪费。
    • 实现：工程上通过数组索引 (Array Indexing) 实现，即 output = self.weight[token_ids]。这实际上是一次内存的随机访问（Random Access）。

Pre-Norm Transformer Block

现代 LLM 普遍采用 Pre-Norm 结构，即 x = x + Layer(Norm(x))。 相比于原始 Transformer 的 Post-Norm，Pre-Norm 创造了一条清晰的梯度高速公路 (Residual Stream)。梯度可以直接从输出层回传到嵌入层，极大地缓解了深层网络的训练不稳定性。

Root Mean Square Layer Normalization

LayerNorm 的一种简化变体。研究发现，LayerNorm 中的“减均值”操作（Centering）对于现在的 LLM 并不重要，重要的是“除以标准差”（Scaling）。

  • 数学定义：
RMSNorm(x)=RMS(x)x​⋅γ RMS(x)=d1​i=1∑d​xi2​+ϵ​

  • 认知视角 (个体内归一化)：

    RMSNorm 计算的是单个 Token 内部所有特征维度的能量平均值。它像是一个“音量调节旋钮”，不管输入的声音（特征数值）多大或多小，都强制将其总能量拉回到一个标准范围，防止数值爆炸。它不依赖 Batch 中的其他样本（这点对变长序列生成至关重要）。

  • 工程细节：

    • Upcast：在计算平方和 (x2) 时，必须先将数据类型强制转换为 float32。因为 FP16 的动态范围太小，平方运算极易导致溢出 (Overflow)。
    • 维度选择：使用 mean(-1)。这里的 -1 代表 Feature Dimension。

Position-Wise Feed-Forward Network

如果 Self-Attention 是“团队开会”（交换上下文信息），那么 FFN 就是“闭关消化”（独立思考）。FFN 对序列中的每一个 Token 独立进行非线性变换。

  • 激活机制：SwiGLU

    这是目前 Llama 3、Qwen 等前沿模型的标配，性能优于传统的 ReLU。

    • GLU (Gated Linear Unit)：引入了“门控”机制，允许网络学习选择性地通过信息。

    • 公式：

FFN(x)=Wdown​(SiLU(Wgate​x)⊙Wup​x)

  • 这里涉及三个矩阵：

    1. Wgate​: 门控投影。
    2. Wup​: 升维投影。
    3. Wdown​: 降维投影。

  • 维度设计：

    • 中间层维度 dff​ 通常设为 38​dmodel​。
    • 硬件对齐：需确保该维度是 64 或 128 的倍数，以充分利用 GPU 的 Tensor Cores 进行矩阵乘法加速。

Relative Positional Embeddings

RoPE 不向 Token 添加位置向量，而是通过旋转 Token 的 Embedding 向量来注入位置信息。如果 Token A 在位置 m，Token B 在位置 n，它们的内积（Attention Score）将自然包含 m−n 的相对位置信息。

1. 频率计算 (The Gears - 齿轮系统)

RoPE 设计了一组不同频率的 ” 齿轮 ”。

  • 公式：θj​=10000−2j/d。
  • 物理意义：
    • 低频维度（j 小）：波长很短，旋转极快，用于捕捉相邻词的微观位置关系。
    • 高频维度（j 大）：波长很长，旋转极慢，用于捕捉长距离的宏观位置关系。
  • 实现：计算出的 inv_freq 是固定值，应注册为 Buffer，不参与反向传播更新。

2. 外积与广播 (Outer Product via Einsum)

我们需要计算每一个位置 m 在每一个频率 θj​ 下的旋转总角度。

  • 操作：位置向量 × 频率向量。

3. 旋转矩阵的微观构造 (The Micro-Slice)

虽然理论上 RoPE 是一个巨大的块对角矩阵，但我们不需要显式构建它。我们在微观层面（2x2）模拟旋转：

  • 构建行：利用 cos 和 sin 构建 [cos, -sin] 和 [sin, cos]。
  • 堆叠：利用 torch.stack 形成 (…, 2, 2) 的张量。

4. 维度配对策略 (Pairing Strategy - 关键考点)

这是 RoPE 实现中最容易出错的地方：如何将 d 维向量拆分为复数对？

  • Llama/HuggingFace 风格：将向量一分为二，前半部分与后半部分配对。

  • 原始论文/作业风格 (Interleaved)：相邻元素配对。

  • 注意：作业的测试用例通常基于论文定义。如果使用切片法会导致位置编码错乱（除了位置 0），必须使用 view 进行相邻重塑。

5. 执行旋转 (Einsum Execution)

最终的旋转操作可以看作是无数个 2x2 矩阵乘法的并行执行。

  • Einsum 魔法：
    • …: 自动处理 Batch 和 Sequence 维度（SIMD 思想）。
    • ij, j -> i: 在最底层的 2D 空间内，执行标准的旋转矩阵乘法。

通过这种方式，RoPE 优雅地将绝对位置的算术加法（位置 m+1），转化为了复数平面上的几何旋转（角度 θ），从而完美保留了相对位置信息。

Scaled Dot-Product Attention

核心定义：

这是 Transformer 的计算引擎，负责计算 Token 之间的相关性。它本身不包含可学习参数，完全依赖输入的 Q,K,V 进行运算。

数学公式：

Attention(Q,K,V)=softmax(dk​​QKT​)V

关键实现细节：

  1. 缩放 (Scaling)：
    • 操作：在做 Softmax 之前，将 QKT 的结果除以 dk​​ (即 head_dim 的平方根)。
    • 原因：当 dk​ 很大时，点积结果的方差会变得很大，导致 Softmax 函数进入梯度极小的饱和区（梯度消失）。缩放将其拉回梯度敏感区。
  2. 因果掩码 (Causal Masking) - 避坑指南：
    • 目标：确保位置 t 的 Token 只能看到 0 到 t 的信息（过去和现在），不能看到 t+1 之后的信息（未来）。
    • 逻辑定义：
      • 保留 (Keep)：下三角区域 (Lower Triangular)。
      • 遮挡 (Block)：上三角区域 (Upper Triangular)。
    • 代码实现：
      • 推荐使用 torch.tril 生成一个下三角为 1 (True) 的矩阵。
      • 在 masked_fill 时，逻辑是：scores.masked_fill(mask == 0, float('-inf'))。
      • 注意：之前的错误在于混淆了 ~ 取反和 triu/tril 的关系。标准做法是：掩码为 True/1 的地方是允许被 Attention 的。
  3. 数值稳定性：
    • Softmax 计算应在最后一个维度 (dim=-1) 进行。
    • 填充值必须是 -inf，这样 Softmax 后对应的权重才会是纯 0。

Causal Multi-Head Self-Attention

核心定义：

通过多头机制（Multi-Head），让模型能够同时关注不同子空间（Subspaces）的信息。例如，一个头关注语法结构，另一个头关注指代关系。

数据流与维度变换 (Pipeline)：

  1. 线性投影 (Linear Projections)：
    • 输入 X 形状：(Batch, Seq, D_model)。
    • 通过 Wq​,Wk​,Wv​ 投影。注意，为了计算效率，通常将所有头的投影合并在一个大矩阵中计算，然后再切分。
  2. 切分多头 (Split Heads)：
    • 变形：(Batch, Seq, D_model) -> (Batch, Seq, n_heads, head_dim)。
    • 注意：这里不需要转置，保持 Seq 在前是为了方便后续应用 RoPE（如果 RoPE 实现支持广播）。
  3. 相对位置编码 (RoPE Injection) - 核心难点：
    • 应用时机：必须在切分多头之后，计算 Attention Score 之前。
    • 作用对象：仅作用于 Query (Q) 和 Key (K)。Value (V) 不进行旋转。
    • 维度策略 (Fold Heads Strategy)：
      • 为了避免 einsum 的广播维度错误，最稳健的做法是将 Batch 和 Heads 合并。
      • Reshape：将 Q,K 转置并重塑为 (Batch * n_heads, Seq, head_dim)。
      • 应用 RoPE：此时 RoPE 看到的是 3D 张量，无需处理复杂的 Head 广播。
      • 还原：旋转后再变回 (Batch, n_heads, Seq, head_dim)。
    • 配对方式：必须使用相邻配对 (Interleaved) [-x1, x0] 模式，即 x.view(…, d//2, 2)，而非切片模式。这是通过测试的关键。
  4. 计算注意力 (Attention Calculation)：
    • 转置：确保输入给 scaled_dot_product_attention 的形状是 (Batch, n_heads, Seq, head_dim)。
    • 错误警示：千万不要在 RoPE 还原后，再次对 Q,K 做错误的 transpose(1, 2)。确保维度含义是 (Batch, Heads, Seq_Q, Seq_K)。
  5. 输出投影 (Output Projection)：
    • Attention 输出形状：(Batch, n_heads, Seq, head_dim)。
    • 合并头：转置回 (Batch, Seq, n_heads, head_dim) 然后 contiguous().view(Batch, Seq, D_model)。
    • 通过 Wo​ 进行最后一次线性变换。

The Full Transformer LM

宏观架构：

这是一个基于 Decoder-only 架构的生成式模型（类似 Llama 2/3 或 GPT-2/3）。

组件堆叠顺序：

  1. Embedding Layer (嵌入层)：
    • 将 Token ID 映射为向量。
    • 注意：由于我们使用了 RoPE，这里不需要添加绝对位置编码（Learned Positional Embeddings）。位置信息将在每一层的 Attention 中动态注入。
  2. Transformer Blocks (堆叠层)：
    • 采用 Pre-Norm 结构（这是现代 LLM 的标准配置）。
    • 残差连接 1：h = x + Attention(RMSNorm(x))。
    • 残差连接 2：out = h + FFN(RMSNorm(h))。
    • 关键点：Norm 放在子层输入之前，残差路径保持干净（Clean Residual Stream），这有助于深层网络的梯度传播。
  3. Final Norm (最终归一化)：
    • 在所有层通过后，输出 Logits 之前，必须再加一层 RMSNorm。这是为了稳定最后的特征分布。
  4. Language Modeling Head (LM Head)：
    • 一个线性层，将 d_model 维度映射回 vocab_size 维度。
    • 权重绑定 (Weight Tying)：通常将 Input Embedding 的权重矩阵复用于 LM Head（即 lm_head.weight = embeddings.weight）。这不仅减少了大量参数，还被证明能提升效果。
  5. 损失函数 (Loss Function)：
    • 使用 Cross Entropy Loss。
    • Shift Labels：计算 Loss 时，输入是 t 时刻的输出，标签是 t+1 时刻的 Token。代码中通常需要 logits[:, :-1, :] 和 labels[:, 1:] 进行错位对齐。

总结构建流程：

Input IDs → Embedding → Block 1 → … → Block N → RMSNorm → LM Head → Logits → Softmax → Next Token Probabilities。

Training a Transformer LM

Cross-entropy Loss

1. 核心定义

在语言模型（如 Transformer）中，交叉熵损失函数的核心任务是衡量模型预测的概率分布与真实结果之间的差异。

简单来说，它的逻辑是：

  • 输入：模型对下一个词的原始打分（Logits）。
  • 目标：让正确答案对应的概率尽可能大（接近 1）。
  • 惩罚机制：
    • 如果模型给正确答案的概率很高（比如 0.99），Loss 就接近 0（几乎不惩罚）。
    • 如果模型给正确答案的概率很低（比如 0.01），Loss 就会非常大（重罚）。

2. 数学原理分解

根据论文片段，计算过程分为四个步骤。假设我们有一个输入向量（Logits）o 和正确答案的索引 y（即 target）。

第一步：从分数到概率 (Softmax)

模型输出的是原始分数 oi​（Logits），范围可正可负。我们需要通过 Softmax 将其归一化为概率分布：

p(x)=Softmax(o)=∑j​eoj​eox​​
第二步：聚焦正确答案

虽然模型输出了整个词表的概率，但计算 Loss 时，我们只关心正确答案 y 对应的那一个概率 p(y)。

第三步：计算负对数似然 (Negative Log-Likelihood)
Loss=−log(p(y))

这就是标准的交叉熵公式。取对数是因为概率是相乘的关系，转成对数变成相加，计算更方便；取负号是因为 log(概率) 是负数，我们需要一个正数的 Loss 来进行最小化。

3. 工程实现的挑战：数值稳定性

在实际编程（PyTorch）中，直接照搬上面的公式会遇到**数值溢出（Overflow）**的问题。

问题来源

如果你先算 Softmax，计算机需要计算 ex。

  • 如果 x=1000，则 e1000→∞（无穷大）。
  • 这将导致程序报错或出现 NaN / inf。
解决方案：Log-Sum-Exp 技巧

为了解决这个问题，我们在数学上对公式进行变形，把 ” 先除后取对数 ” 转化为 ” 先减后取对数 ”。

推导过程：

Loss​=−log(∑j​eoj​eoy​​)=−(log(eoy​)−log(j∑​eoj​))=log(j∑​eoj​)−oy​​

最终公式：

Loss=LogSumExp(o)−otarget​

这个公式的好处：

  1. 避免除法：不需要显式地算出概率。
  2. 数值稳定：PyTorch 的 logsumexp 函数内部使用了 ” 减去最大值（Shift Trick）” 的技巧，即使输入是 1000 也能算出正确结果，绝对不会溢出。

5. 常见误区警示 (Debug 总结)

在学习过程中，以下三个错误最容易出现：

  1. 混淆索引与数值：
    • 错误写法：loss = logsumexp - targets
    • 后果：你减去了 ” 答案的编号 “（如 50），而不是 ” 答案的分数 “（如 2.5）。会导致结果出现负数。
    • 正确：必须用 inputs[…, targets] 把分数取出来再减。
  2. 先 Softmax 后 Log：
    • 错误写法：loss = -torch.log(softmax(inputs))
    • 后果：当输入数值较大（如 1000）时，Softmax 会溢出导致 inf。必须使用 logsumexp 或 PyTorch 自带的 F.cross_entropy（它内部合并了这两步）。
  3. 忽略 Batch 维度处理：
    • 在取值时如果没有配合 torch.arange，可能会导致取值形状错误或无法正确对应每个样本的答案。

Perplexity

虽然 Cross-Entropy Loss 对于**训练（优化参数）是足够的，但在评估（Evaluation）**模型好坏时，我们通常汇报 Perplexity (PPL)。

定义与公式

对于一个长度为 m 的序列，如果我们计算出的交叉熵损失分别是 ℓ1​,ℓ2​,…,ℓm​。

那么该序列的困惑度定义为平均损失的指数：

Perplexity=exp(m1​i=1∑m​ℓi​)

直观理解

困惑度可以理解为：” 模型在预测下一个词时，平均有多少个不确定的选项？”

  • PPL = 1：完美模型。模型对每一次预测都 100% 确定且正确（Loss=0,e0=1）。
  • PPL = 100：相当于模型在做选择时，像是在 100 个词里随机瞎猜。
  • PPL = Vocab Size：模型完全未经训练，相当于在整个词表中随机扔骰子。

结论：Perplexity 越低越好。

The SGD Optimizer

  • 基本定义：
    • SGD (Stochastic Gradient Descent) 是最基础的基于梯度的优化器。
    • 核心思想：从随机初始化的参数 θ0​ 开始，在每一步 t，从数据集 D 中采样一个随机批次 (Batch) Bt​，计算损失函数 L 的梯度，并反向更新参数。
    • 更新公式：
θt+1​←θt​−αt​∇L(θt​;Bt​) 
其中 $\alpha_t$ 是学习率（Learning Rate），$|B_t|$ 是批次大小。

  • PyTorch 实现细节：

    • 继承类：需要继承 torch.optim.Optimizer。
    • __init__ 方法：
      • 接收 params（待优化的参数集合）和超参数（如 lr）。
      • 必须调用父类的 super().__init__(params, defaults)，将参数存储在 self.param_groups 中。
    • step 方法：
      • 在 loss.backward() 之后调用。
      • 遍历 self.param_groups 中的每个参数张量 p。
      • In-place 更新：直接修改 p.data。利用 p.grad（损失关于参数的梯度）进行更新。
      • 状态管理：如果需要实现学习率衰减（如文档示例中的 1/t+1​ 衰减），需要利用 self.state[p] 来存储迭代次数 t。

  • 训练循环的标准结构：

    1. opt.zero_grad(): 清空过往梯度。
    2. loss = …: 前向传播计算损失。
    3. loss.backward(): 反向传播计算梯度。
    4. opt.step(): 更新参数。

AdamW

  • 概述：

    • 现代语言模型（如 GPT-3, LLaMA）通常使用 AdamW 而非 SGD。
    • AdamW vs. Adam：AdamW 是 Adam 的变体，其核心改进在于将权重衰减 (Weight Decay) 与梯度更新步骤解耦。这能提供更好的正则化效果。

  • 算法机制：

    • 有状态 (Stateful)：不同于无状态的基础 SGD，AdamW 为每个参数维护两个额外的状态张量：
      1. 一阶矩 (First Moment, m)：梯度的指数移动平均。
      2. 二阶矩 (Second Moment, v)：梯度平方的指数移动平均。
    • 超参数：
      • α (Learning Rate)。
      • β1​,β2​：控制矩估计的衰减率。典型值为 (0.9, 0.999)，但训练 LLM 时常设为 (0.9, 0.95)。
      • ϵ：数值稳定性项（如 10−8），防止除零。
      • λ：权重衰减系数。
    • 更新步骤 (伪代码逻辑)：
      1. 更新一阶矩：m←β1​m+(1−β1​)g
      2. 更新二阶矩：v←β2​v+(1−β2​)g2
      3. 偏差修正（Bias Correction）：调整学习率 αt​ 以适应初始阶段的偏差。
      4. 参数更新：θ←θ−αt​v+ϵ​m​
      5. 权重衰减（独立应用）：θ←θ−αλθ

  • 资源消耗核算 (Resource Accounting)：

    • 内存占用 (Memory)：
      • 假设使用 float32 (4 bytes)。
      • 静态内存 (Static Memory)：每个参数需要存储 4 份数据：参数本身 (θ)、梯度 (g)、一阶矩 (m)、二阶矩 (v)。
Static Memory≈16 bytes×参数量 P 
- **动态内存 (Activations)**：取决于 Batch Size ($B$)、序列长度 ($L$)、层数 ($N$) 和隐藏维度 ($h$)。反向传播需要保存前向传播的激活值（如 Attention Scores, FFN 输入等）。

  - 对于 GPT-2 XL 级别模型，激活值内存通常占据主要部分。例如在 80GB 显存上，若不使用 Checkpointing，Batch Size 可能被限制在个位数（如 3）。
  • 计算量 (Compute/FLOPs)：
    • 优化器步：AdamW 是逐元素操作 (Element-wise)，不涉及矩阵乘法。计算量仅与参数量成线性关系，约为 12 FLOPs / 参数。
    • 模型训练步：包含前向和反向传播。近似计算量为 6PBL (假设反向传播计算量是前向的 2 倍)。
    • 吞吐量 (Throughput)：A100 GPU 的 FP32 理论峰值较低 (19.5 TFLOPS)，实际训练通常依赖 Tensor Core (TF32/BF16) 以获得更高吞吐量。若纯跑 FP32，训练时间将极其漫长。

Learning Rate Scheduling

  • 目的：
    • 固定的学习率通常不是最优的。
    • 策略：开始时使用较大的学习率快速收敛，训练后期减小学习率进行微调。
  • Cosine Annealing with Warmup (余弦退火加预热)：
    • 这是训练 Transformer（如 LLaMA）的标准策略。
    • 三个阶段：
      1. Warm-up (预热期) (t<Tw​):
        • 学习率从 0 线性增加到最大值 αmax​。
        • 公式：αt​=Tw​t​αmax​
      2. Cosine Annealing (退火期) (Tw​≤t≤Tc​):
        • 学习率按余弦函数曲线从 αmax​ 衰减到 αmin​。
        • 公式涉及 cos(Tc​−Tw​t−Tw​​π)。
      3. Post-annealing (后退火期) (t>Tc​):
        • 保持在最小值 αmin​ 不变。

Gradient Clipping

  • 问题背景：
    • 训练过程中可能会遇到某些样本产生极大的梯度（Exploding Gradients），这会导致参数更新过猛，破坏模型的稳定性。
  • 解决方案：

    • 梯度裁剪 (Gradient Clipping)：限制梯度的范数（Norm）。

    • 算法逻辑：

      1. 计算所有参数梯度的全局 L2​ 范数：∣∣g∣∣2​。

      2. 设定一个阈值 M (Max Norm)。

      3. 判断：

        • 如果 ∣∣g∣∣2​≤M，梯度保持不变。
        • 如果 ∣∣g∣∣2​>M，则对梯度进行缩放：
g←g×∣∣g∣∣∗2+ϵM​ 
     (其中 $\epsilon$ 为极小值，如 $10^{-6}$，用于数值稳定)。

4. 结果：缩放后的梯度范数将略小于或等于 $M$。
  • 实现：通常在 loss.backward() 之后、optimizer.step() 之前进行。操作是 In-place 的。

Checkpointing

在深度学习实验中，**检查点（Checkpointing）不仅是保存训练成果的手段，更是实现训练任务可恢复性（Resumability）和训练动力学研究（Training Dynamics）**的基础。

1. 为什么需要检查点？

  • 容错与恢复：在训练大型模型（如 LM）时，任务可能因硬件故障、系统超时或资源抢占而中断。拥有检查点可以从最后一次保存的状态恢复，避免重新训练。
  • 动力学分析：通过保存训练过程中的中间模型，可以研究模型在不同阶段的采样效果或权重变化。
  • 学习率调度：为了维持 Learning Rate Scheduler 的连续性，必须记录当前的迭代步数（Iteration/Step）。

2. 检查点的三大核心要素

一个完整的检查点应该包含 ” 复活 ” 训练所需的所有有状态对象：

要素来源作用
模型权重 (Model Weights)model.state_dict()恢复神经元的所有可学习参数（Weights & Biases）。
优化器状态 (Optimizer State)optimizer.state_dict()记录动量（Momentum）和二阶矩（如 AdamW 的动量估算），确保恢复后梯度更新逻辑一致。
元数据 (Metadata)iteration / epoch记录当前的训练进度，用于计算学习率策略及循环计数。

3. PyTorch 序列化机制

PyTorch 提供了简洁的 API 来处理这些状态：

  • state_dict()：返回一个 Python 字典，将每一层映射到其对应的参数张量。注意：它只包含参数，不包含模型结构。
  • torch.save(obj, path)：利用 Python 的 pickle 模块将对象序列化到磁盘。
  • torch.load(path)：将序列化的对象加载回内存。
  • load_state_dict()：这是一个 原地（In-place） 操作，将保存的张量复制到当前模型或优化器的参数中。

3. 训练循环的核心机制

这是脚本最复杂的部分，涉及多个提升效率和稳定性的技术：

A. 梯度累积 (Gradient Accumulation)

当显存不足以支撑大 batch_size 时，通过 grad_accum_steps 实现：

  • 在 grad_accum_steps 次微步（micro-steps）中，计算损失并反向传播，但不清空梯度。
  • 计算 micro_loss = micro_loss / grad_accum_steps 以保证梯度的平均值正确。
  • 最后进行一次 optimizer.step()。
B. 混合精度训练 (Mixed Precision)

使用 torch.amp (Automatic Mixed Precision) 提高训练速度并节省显存：

  • Autocast: 在 with autocast_context 块内，算子会自动选择合适的精度（如 float16 或 bfloat16）。
  • GradScaler: 解决 float16 梯度下溢问题。脚本在反向传播前对损失进行缩放（scaler.scale(loss).backward()），并在更新前视情况取消缩放（unscale_）。
C. 学习率调度 (LR Scheduling)
  • Warmup & Decay: 通过 learning_rate_schedule 在训练初期线性增加学习率，后期可能采用余弦退火（Cosine Decay）。
  • 在每个 step 中手动设置 optimizer.param_groups[0]["lr"] 以实现动态调整。

4. 健壮性与可观察性

  • 梯度裁剪 (Gradient Clipping): 限制梯度的范数（Norm），防止由于大梯度导致的训练崩溃（Loss Spiking）。
  • 检查点序列化 (Checkpointing): 定期保存模型权重和优化器状态（save_checkpoint）。如果训练中断，可以通过 --resume-from 恢复状态。

  • Local: 使用 ExperimentLogger 记录到 CSV 文件。
  • Remote: 集成 Weights and Biases (W&B)，实时可视化损失曲线和硬件利用率。

Generating Text

原理

1. 自回归解码 (Autoregressive Decoding)

语言生成的核心是一个重复的预测过程。模型接收一个序列 x1…t​，并输出下一个 token xt+1​ 的概率分布。

  • 数学表达: P(xt+1​∣x1…t​)=Softmax(Transformer(x1…t​)).
  • 过程:
    1. 输入当前上下文（Prompt）。
    2. 模型计算最后一个位置的 Logits（未归一化的分数）。
    3. 应用 Softmax 得到概率分布。
    4. 从分布中采样得到下一个 token。
    5. 将新 token 拼接到上下文末尾，重复上述步骤直到遇到 <|endoftext|> 或达到最大长度。

2. 采样策略 (Sampling Strategies)

直接采样（贪婪搜索，即每次取最大概率）往往会导致重复或枯燥的文本。为了生成更有趣且连贯的文本，引入了随机性，并通过以下策略控制：

A. 温度缩放 (Temperature Scaling)

通过参数 τ (Temperature) 调整概率分布的 ” 尖锐 ” 程度。

  • 公式:
Pi​=∑j​exp(vj​/τ)exp(vi​/τ)​
  • 原理:
    • τ→0 (低温): 分布变得极其尖锐，高概率词的优势被放大，接近贪婪搜索（确定性强，创造力低）。
    • τ→1: 标准 Softmax。
    • τ>1 (高温): 分布变得平坦，低概率词被选中的机会增加（创造力高，但容易产生胡言乱语）。
B. 核采样 (Top-p / Nucleus Sampling)

为了解决 Top-k 采样（只取前 k 个）中 k 值难以固定的问题，Top-p 动态截断概率分布的尾部。

  • 机制: 只保留累积概率之和达到阈值 p 的最小集合 V(p)。
  • 公式:
x∈V(p)∑​P(x)≥p
  • 效果: 在分布平坦时（模型不确定），候选词多；在分布尖锐时（模型确定），候选词少。这比固定的 Top-k 更自适应。
