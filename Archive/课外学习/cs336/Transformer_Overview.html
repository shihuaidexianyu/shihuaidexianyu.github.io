我们可以通过一个生动的类比来理解这两个子层在 ” 处理语言 ” 时各自扮演的角色。

你可以把处理一个句子（比如 “The animal didn’t cross the street because it was too tired” ）想象成一群学生（单词）在进行一次课堂活动。

  1. 自注意力机制 (Self-Attention)：像 ” 小组讨论 ”

核心作用：搞清楚 ” 我 ” 跟 ” 谁 ” 有关。

  • 情境：在没有自注意力之前，每个单词只知道自己的字面意思，” 孤立 ” 地站着。

  • 发生了什么：自注意力机制就像老师让大家 ” 自由讨论 ”。

    • 单词 “it”（它）感到很困惑，不知道自己是指 “animal”（动物）还是 “street”（街道）。

    • 通过自注意力机制，“it” 能够去 ” 看 ” 句子里的其他单词。它发现 “tired”（累）这个词跟 “animal” 关系很紧密，而跟 “street” 没啥关系。

    • 于是，“it” 决定更加关注 “animal”，将一部分 “animal” 的含义融入到自己的理解中。

  • Note

这个可以在后面的注意力机制解释。

  • 关键点：这一步解决了依赖性（Dependencies）。它让模型能够理解上下文，知道这个词在当前语境下到底意味着什么。

  1. 前馈神经网络 (Feed Forward Neural Network)：像 ” 独立作业 ”

核心作用：消化吸收，整理信息。

  • 情境：小组讨论（自注意力）结束了，每个单词都已经通过和别人交流，更新了自己的理解（比如 “it” 现在知道自己是一只累坏了的动物）。
  • 发生了什么：现在，老师要求所有学生回到座位上，独立完成一道相同的问答题。
    • 每个位置的单词都通过完全相同的神经网络进行计算。
    • 虽然大家用的 ” 解题公式 “（权重）是一样的，但因为每个单词刚刚在讨论中获得的信息不同，所以算出来的结果也不同。
    • 在这个过程中，单词之间不再互相交头接耳（没有依赖关系），大家各算各的。
  • 关键点：这一步是**并行（Parallel）**的。因为每个词的处理路径是独立的，互不干扰，所以计算机可以同时处理所有单词，效率非常高。

在 GPT 类模型的语境下，Causal Self-Attention 和 Masked Self-Attention 指的是同一个东西。

这两个词只是侧重点不同：

  • Causal（因果）：描述的是目的或性质。即遵守因果律，当前时刻 t 只能由 t 之前的信息决定。
  • Masked（掩码）：描述的是实现手段。即通过一个掩码矩阵（Mask Matrix）把 ” 未来 ” 的信息遮住。

Q：什么是 Causal Multi-Head Self-Attention？它位于架构的什么位置？它是如何工作的？

1. 概念拆解

它是生成式大模型（GPT 类）的**解码器（Decoder）**核心组件，主要由三个特性组成：

  • Self-Attention (自注意力)：计算序列内部元素间的关联（全连接图）。
    • 心理学类比：**工作记忆（Working Memory）**中的语义联想激活。
  • Multi-Head (多头)：将高维向量切分为多个子空间独立计算，最后拼接。
    • 心理学类比：并行分布式处理（Parallel Distributed Processing），大脑不同区域同时处理语法、语义、情感等不同特征。
  • Causal / Masked (因果/掩码)：强制模型在时刻 t 只能关注 [0,t] 的信息，屏蔽 t+1 及其之后的信息。
    • 核心目的：维持**自回归（Autoregressive）**特性，防止信息穿越。

2. 架构位置

  • 位置：Transformer Decoder Block 的最底层（第一层）。
  • 作用：作为数据流的第一道关卡，在进行任何非线性变换（FFN）或外部交互（Cross-Attention）之前，先锁定当前时刻的有效上下文范围。

3. 数学实现 (Masking Mechanism)

通过在点积注意力（Dot-Product Attention）中引入一个**下三角掩码矩阵（Lower Triangular Mask）**来实现：

Attention(Q,K,V)=softmax(dk​​QKT+M​)V

其中矩阵 M 为：

Mij​={0−∞​if i≥j (过去与现在)if i<j (未来)​

  • 原理：利用 softmax(−∞)=0 的特性，物理上阻断了对未来 Token 的注意力权重。

Q：Mask 机制似乎只有在训练时才有效？因为推理时 ” 未来 ” 本来就不存在。

1. 核心结论

是的。显式的 Mask 矩阵运算主要用于 ” 训练加速 “。

2. 训练阶段 (Training)：并行作弊 (Teacher Forcing)

  • 场景：我们拥有完整的 Ground Truth 句子（例如 “A B C D”）。
  • 系统行为：为了利用 GPU 的并行计算能力，我们将整个句子一次性输入模型。
  • Mask 的作用：如果不加 Mask，计算 Token “A” 时就能看到显存里存在的 “B”。Mask 强行遮挡了已知的答案，模拟了 ” 未知 ” 的状态。
  • 本质：用空间换时间。通过 Mask 矩阵，在并行计算中模拟了串行的时间流逝。

3. 推理阶段 (Inference)：串行生成与 KV Cache

  • 场景：逐个生成 Token（输入 “A”，生成 “B”；输入 “AB”，生成 “C”）。
  • 系统行为：
    • 物理因果：在生成 t 时，t+1 在物理内存中尚不存在，因此不需要 Mask 来遮挡。
    • 性能优化 (KV Cache)：为了避免重复计算，我们缓存过去的 Key/Value 状态。
    • 计算逻辑：当前 Token 的 Query 只与**缓存中（过去）**的 Keys 进行运算。这在逻辑上天然符合因果律。
  • 本质：真正的串行过程。Mask 机制在这里隐式存在于数据流的先后顺序中。

4. 认知类比总结

补充笔记：词嵌入 (Word Embeddings)

1. 基本概念

  • 文本向量化：和大多数 NLP（自然语言处理）应用一样，Transformer 不能直接理解文本。因此，首先需要使用嵌入算法（Embedding Algorithm）将每个输入的单词转换成一个向量。

2. 向量维度 (Dimension)

  • 大小：在该文档的示例中，每个单词被嵌入为一个维度为 512 的向量。
  • 统一标准：这也是整个模型中向量流动的标准尺寸。所有的编码器（Encoders）都设计为接收和输出大小为 512 的向量列表。

3. 位置与流动

  • 仅限底层：词嵌入操作只发生在最底层的编码器（Bottom-most encoder）之前。
  • 层级传递：
    • 最底层编码器：直接接收词嵌入向量作为输入。
    • 上层编码器：接收的是它正下方那个编码器的输出向量，而不是原始的词嵌入。
  • 数据流向：输入序列中的单词被嵌入后，会依次流经编码器的两个子层（自注意力层和前馈网络层）。

4. 序列长度

  • 列表大小：编码器接收的是一个向量列表。这个列表的长度是一个我们可以设置的超参数（Hyperparameter），通常设定为训练数据集中最长句子的长度。

将词嵌入（Embedding）乘以权重矩阵（WQ,WK,WV），在数学和几何意义上，确实完全等同于将该向量 ” 线性投影 ” 到了由这些矩阵所定义的特定空间（Subspace）中。

以下是关于这一概念的详细解析：

1. 数学视角：线性变换与降维

  • 操作：输入向量 x（维度 512）乘以矩阵 W（维度 512×64）。
  • 几何意义：这是一个线性变换 (Linear Transformation)。它将向量从原本广阔的 512 维通用语义空间，映射（或压缩）到了一个更低维度的 64 维子空间中。
  • 目的：通过降维，模型强迫向量在这个新空间里只保留最关键的特征。

2. 语义视角：三个不同的 ” 平行宇宙 ”·

这一步之所以重要，是因为 Q,K,V 代表了看待同一个词的三种不同视角（即三个不同的空间）：

  • WQ 投影出的空间 (Query Space)：
    • 意义：这个空间代表**” 提问 “**的视角。
    • 例子：当 “it” 被投影到这个空间时，它的向量形态主要表达：” 我在找我的指代对象，谁是名词？谁是动物？”
  • WK 投影出的空间 (Key Space)：
    • 意义：这个空间代表**” 索引 “或” 标签 “**的视角。
    • 例子：当 “animal” 被投影到这个空间时，它的向量形态主要表达：” 我是一个名词，我是动物实体。“（这样正好能跟 “it” 的 Query 匹配上）。
  • WV 投影出的空间 (Value Space)：
    • 意义：这个空间代表**” 内容 “**的视角。
    • 例子：这是该词原本含义的精华保留，用于最后被提取和融合。

3. 多头注意力的进一步印证

多头注意力给了模型**” 多个表示子空间 (representation subspaces)”** 。

  • 如果我们有 8 个头，我们就训练了 8 组 WQ,WK,WV 矩阵。
  • 这意味着我们将同一个词嵌入，投影到了 8 个不同的空间 里去观察。

那问什么定义这几个矩阵叫做 Q,K,V，它们的作用是如何被决定的？或者说是如何被限制的？

简单来说，我们没有直接编写代码告诉 WQ 去 ” 提问 “，也没有告诉 WK 去 ” 回答 ”。

确定它们起到这些作用的，是**” 以结果为导向 ” 的反向传播（Backpropagation）训练过程以及数学结构的强制约束**。

我们可以从以下三个层面来理解：

1. 训练目标的倒逼：如果不配合，就无法得分

Transformer 的唯一目标就是减少预测错误（Loss Function）。

  • 情境：模型需要处理 “The animal didn’t cross the street because it…” 并预测下一个词是 “was” 或理解 “it” 的含义。
  • 失败的情况：如果 WQ 投影出的向量在 ” 胡言乱语 “（没有发出寻找指代的信号），或者 WK 没有把 “animal” 标记为可被指代的对象，那么 Q 和 K 的点积（Score）就会很低。
  • 后果：“it” 就无法关注到 “animal”，导致对 “it” 的理解缺失，最终翻译错误或预测失败。这会产生巨大的损失（Loss）。
  • 修正：反向传播算法会告诉模型：” 你搞砸了。为了下次能对，你需要调整 WQ 让它更像是在 ’ 找名词 ‘，调整 WK 让它更像是在 ’ 标记自己是名词 ’。”
  • 结果：经过亿万次的训练，这些矩阵被迫进化出了这种 ” 提问 - 索引 ” 的配合模式，因为只有这样才能把 Loss 降下来。

2. 数学结构的强制：点积只奖励 ” 相似 ”

自注意力机制的计算公式中，核心是 点积（Dot Product）：Score=q⋅k。

  • 数学特性：点积是衡量两个向量相似度或对齐程度的工具。只有当 q 和 k 在向量空间中指向相似的方向时，得分才会高。
  • 物理约束：这种结构在物理上强制要求 WQ 和 WK 必须将原本不同的词（如 “it” 和 “animal”）映射到同一个公共空间里，并且在这个空间里它们必须 ” 这就好比两块拼图，只有形状契合（数学上的高点积）才能拼在一起。
  • 因此，Q 必须变成一把 ” 锁 ”，K 必须变成一把 ” 钥匙 “，否则这一层的数学机制就无法让信息流动。

3. 视觉验证：Attention Map 是铁证

虽然我们不能直接看懂矩阵里的浮点数，但我们可以通过**注意力可视化图（Attention Map）**来验证它们是否起到了作用。

  • 证据：正如之前的图所示，当处理 “it” 时，我们看到它与 “animal” 之间的连线非常粗（权重高）。
  • 结论：这证明了 WQ 确实生成了一个能 ” 钩住 ” WK（来自 animal）的向量。这种高权重连接就是矩阵发挥作用的 ” 犯罪现场证据 “。

总结

你无法在初始化时确定它们的作用（那时它们只是随机数）。

但是，架构的设计（点积关注机制） 搭建了舞台，训练的目标（最小化误差） 编写了剧本。经过训练，WQ,WK,WV 必须分别扮演好查询、索引和内容的角色，否则这台戏（模型预测）就演不下去。

  • 将 Softmax 分数乘以对应的 Value 向量。

1. 这一步的 ” 含义 ” 是什么？

——这是一种 ” 信息筛选 ” 或 ” 音量调节 ” 机制。

  • 直觉：你可以把 Softmax 分数看作是**” 音量旋钮 ”** 。
  • 操作：
    • 对于你认为重要的单词（例如分数 0.88 的 “animal”），你把它的音量调大（乘以 0.88），这样它的声音（信息）就能被清晰地保留下来。
    • 对于你认为不相关的单词（例如分数 0.001 的 “street”），你把它的音量调到几乎静音（乘以 0.001），这样它的声音就被**” 淹没 ” (drown-out)** 了，不会干扰主旋律。

2. 为什么是乘 Value 向量？在这里 Value 的作用是啥？

——Value 向量是 ” 实际内容的载体 ” (Payload)。

为了理解 Value 的特殊作用，我们需要把它跟 Query 和 Key 区分开：

这是一个 ” 查字典 ” 的过程

  • Query (Q)：你想要查的**” 问题 ”**（比如：” 谁是名词？”）。
  • Key (K)：字典里每个词条的**” 索引标签 ”**（比如：” 我是名词 ”、” 我是动词 ”）。
  • Value (V)：字典里该词条下的**” 详细解释/内容 ”**。

过程解析：

  1. 匹配 (Q · K)：你拿着 Q 去扫所有的 K。如果匹配成功（点积得分高），说明你找到了对的词条。
  2. 提取 (Multiply V)：既然找到了对的词条，你想要的是这个词条下的内容 (Value)，而不是它的标签 (Key)。
    • 如果我找到了 “animal” 这个词条，我想要把它的语义内容（Value 向量）提取出来，融入到我对 “it” 的理解中。
    • 如果我乘的是 Key 向量，那就相当于我找到了书页，却只把页码剪下来贴在作业里，而没有把书里的知识（Value）抄下来。

总结 Value 的作用： Value 向量代表了该单词在被转化为最终输出前，模型认为最有价值传递给下一层的信息特征。Q 和 K 只是为了确定 ” 找谁 “，V 才是真正的 ” 货物 ”。

那这里我有一个疑问：为什么不开辟新的维度用于存储原始的数据，而是要用原始的数据和变化量相加进行传递？

这是一个非常棒的工程架构问题。你提出的“开辟新维度”（即拼接/Concatenation）确实是深度学习中另一种常用的特征融合方式（比如在 DenseNet 或 UNet 中）。

但在 Transformer（以及 ResNet）这类通过堆叠很多层来构建的深度模型中，选择相加 (Addition) 而不是拼接，主要是为了控制维度爆炸和维持接口统一。

以下是具体的分析：

1. 维度爆炸问题 (Dimensionality Explosion)

如果采用“开辟新维度”的方式，模型的参数量会呈指数级或线性级增长，导致无法训练。

  • 相加 (Addition)：
    • X 是 512 维，Z 是 512 维。
    • X+Z 依然是 512 维。
    • 结果：无论堆叠多少层（6层、12层甚至100层），数据流动的“管道粗细”始终保持不变。文档明确指出，嵌入和编码器的输入/输出向量维度都是 512。
  • 拼接 (Concatenation)：
    • 第一层输出：X (512) + Z (512) = 1024 维。
    • 第二层输出：1024 + 1024 = 2048 维（或者更多）。
    • 后果：如果我们堆叠 6 个编码器，向量维度会迅速膨胀。这会导致后续的权重矩阵 W 变得极其巨大，显存瞬间爆炸，计算速度也会越来越慢。

2. 接口的一致性 (Interface Consistency)

Transformer 的设计哲学是模块化堆叠。

  • 提到，编码器是**“堆叠 (stack)”**在一起的。
  • 这意味着 Encoder #1 的输出，必须能直接作为 Encoder #2 的输入。
  • 文档指出，所有编码器接收的都是大小为 512 的向量列表 。
  • 如果使用拼接，每一层的输出维度都不一样，那么每一层编码器的结构（权重矩阵的大小）就必须单独设计，无法做到“所有编码器结构相同” 。通过使用相加，每一层的输入输出维度被锁定在 512，保证了模块的可复制性。

3. 数学意义：是“修正”不是“归档”

从语义理解上，残差连接的目的不是为了“把过去所有的历史都存下来”（那是拼接做的事），而是为了“在原有基础上进行修正”。

  • 拼接 (Concat)：意思说“这是旧的 X，这是新的 Z，我都给你，你自己看着办。”这更像是档案归档。
  • 相加 (Add)：意思说“原本是 X，经过思考后，我觉得应该往 Z 方向调整一点。”这更像是迭代进化。通过 X+Z，向量在保持 512 维空间位置的同时，逐渐向更准确的语义方向移动。

总结

选择相加是因为它是一种**零成本（不增加维度）**的信息保留方式。它让模型能够在保持 512 维“身材”不变的前提下，通过 6 层堆叠不断“提纯”信息，而不是背负着越来越重的历史包袱前行。

7. 解码过程

Note

在编码器到解码器器之前，我们需要知道编码器到编码器的传递内容是什么。

1. 数量一致：n 进 n 出

  • 机制：编码器接收一个由 n 个向量组成的列表（对应输入句子中的 n 个单词）。
  • 结果：经过自注意力层和前馈层的处理后，输出的仍然是一个包含 n 个向量的列表。序列的长度（单词数量）在经过编码器层时保持不变。

2. 尺寸一致：维度保持不变

  • 维度设定：词嵌入向量的维度是 512。编码器接收的向量列表，其中每个向量的大小也是 512。
  • 残差连接的要求：这是尺寸必须一致的硬性物理原因。因为每个子层后都有一个残差连接操作 x+Z。
    • x 是输入向量。
    • Z 是子层处理后的输出向量。
    • 在数学上，只有当两个向量的维度完全相同时，它们才能进行逐元素相加。

总结：

一层编码器就像一个**“等宽管道”**：它接收 n 个 512 维的向量，经过内部处理，输出 n 个 512 维的向量。这种一致性保证了编码器可以像乐高积木一样被无限堆叠。

对于解码器同理

编码器到解码器的传递

这一步是通过线性投影（Linear Projection）和修改版的注意力计算来实现的。

简单来说，就是把编码器（Encoder）整理好的“资料”投影成“索引（Key）”和“内容（Value）”，然后让解码器（Decoder）拿着自己的“问题（Query）”去查阅。

以下是详细的步骤解析：

1. 准备阶段：生成 K 和 V

  • 来源：数据直接来自编码器栈的顶层输出（Top Encoder Output）。假设这个输出矩阵为 M（包含了对整个输入句子的深度理解）。
  • 操作：解码器的“编码器-解码器注意力”层拥有两个独立的权重矩阵：WK 和 WV。
  • 计算：
    • K=M×WK
    • V=M×WV
  • 结果：这样就得到了一组 Key（用于被检索的特征）和 Value（实际的语义内容）。这组 K 和 V 会被复用于解码过程的每一步。

2. 准备阶段：生成 Q

  • 来源：数据来自解码器当前层的前一子层（通常是 Masked Self-Attention 的输出）。
  • 操作：该层还有一个权重矩阵 WQ。
  • 计算：将解码器当前的输入向量乘以 WQ，生成 Query 向量。
  • 含义：这个 Q 代表了解码器“当前正在翻译什么”以及“下一步想查询什么” 。

3. 交互阶段：注意力计算 (The Cross-Attention)

现在的场景是：解码器手里拿着 Q，编码器提供了 K 和 V。接下来的计算逻辑与自注意力完全一致：

  1. 匹配 (Matching)：计算 Q×KT。看看解码器当前的“问题”与编码器提供的哪个“索引”最匹配（点积得分高）。
  2. 归一化 (Softmax)：将得分转换为概率。比如，对于生成 “student” 这个词，原文中 “étudiant” 对应的 Key 可能会得到 0.9 的高分。
  3. 提取 (Extraction)：将概率乘以 V。把高分对应的 Value（内容）提取出来，融合进解码器的当前状态中。

总结

这一步的本质是**“借用”**。

解码器并没有重新去读一遍输入句子，而是直接拿到了编码器已经整理好的、高度抽象的特征矩阵（M）。它只需要用自己的权重矩阵 (WK,WV) 对这个特征矩阵做一次线性变换（投影），就能得到它需要的“字典索引” (K) 和“字典内容” (V)，然后随时查阅。

Masked Self-Attention

这部分内容解释了 Transformer 在训练和生成阶段如何防止“作弊”，确保模型严格按照时间顺序生成文本。

我们可以从“**为什么需要”**和“如何实现”两个角度来深入理解：

1. 为什么需要掩码 (The “Why”)

——防止看到“未来”的答案。

  • 生成过程是线性的：解码器在生成翻译时，是一个接一个词生成的。比如生成 “I am a student”，它是先生成 “I”，再利用 “I” 生成 “am”，以此类推。
  • 训练时的矛盾：在训练阶段，为了效率，我们是一次性把整个正确答案（如 “I am a student”）全都喂给解码器的。
  • 如果不加掩码：当模型在学习如何预测 “am” 时，如果不加限制，自注意力机制会“偷看”到后面的 “a student”。
  • 后果：这就像学生考试时偷看了后面的标准答案。虽然训练时正确率极高，但一旦到了真正的应用场景（没有后面单词的时候），模型就什么都不会了。
  • 规则：因此，解码器中的自注意力层只被允许关注输出序列中较早的位置。

2. 如何实现 (The “How”)

——利用 Softmax 的数学特性进行“物理屏蔽”。

输出

这一步是将模型内部的“数学抽象”转化为人类可理解的“自然语言”的关键桥梁。如果没有这一步，Transformer 输出的只是一堆毫无意义的浮点数。

1. 线性层 (Linear Layer):

  • 输入是什么？ 解码器栈（Decoder Stack）处理完后，会吐出一个浮点向量（例如 512 维）。这个向量是模型对“下一个词应该是什么”的抽象表示 。
  • 线性层做什么？ 线性层是一个全连接神经网络 。它的作用是将这个 512 维的向量**投影（Project）**到一个超级长的向量上。
  • 这个长向量有多长？ 它的长度等于模型输出**词汇表（Output Vocabulary）**的大小。
    • 如果模型认识 10,000 个单词，这个向量就有 10,000 个格子（单元） 。
    • 每个格子对应一个固定的单词（例如：索引 0 是 “a”，索引 4 是 “student”，索引 5 是 “<eos>”） 。
  • 结果是什么？ 输出的这个长向量被称为 Logits 向量。
    • 向量中的每个数字代表该对应单词的原始得分（Raw Score）。
    • 比如，“student” 对应的格子分数很高，而 “book” 对应的格子分数很低 。

2. Softmax 层:

  • 为什么需要它？ Logits 向量里的分数也就是实数，可能是正无穷也可能是负数，不直观。我们需要知道“有多少把握”是某个词。
  • Softmax 做什么？ 它将 Logits 向量中的所有分数转换为概率 。
    • 特性 1：所有概率都是正数。
    • 特性 2：所有概率加起来严格等于 1.0。
  • 结果是什么？ 现在我们得到了一个概率分布图。
    • 例如：“student” 的概率是 0.98，“a” 的概率是 0.01，其他词接近 0 。
  • 最终输出： 模型选择概率最高的那个格子（Argmax 操作），找到其对应的单词（“student”），这就成了这一步的最终翻译结果 。

参数更新

训练的核心目标是让 Loss 变小。这通过反向传播 (Backpropagation) 来实现。

  1. 初始化：

    一开始，模型的所有权重矩阵（WQ,WK,WV, 前馈网络权重等）都是随机初始化的。此时模型输出是乱猜的。

  2. 前向传播 (Forward Pass)：

    输入数据流经整个模型，最后经过 Softmax 得到一个概率分布（例如预测 “am” 的概率是 0.02，而正确答案是 1.0）。

  3. 计算损失 (Calculate Loss)：

    使用交叉熵损失 (Cross-entropy loss) 6来衡量“预测分布”和“真实分布（One-hot）”之间的距离。

    • 距离越大，Loss 越大。

  4. 反向传播 (Backward Pass)：

    这是数学上的链式法则应用。计算 Loss 对每一个权重参数的梯度 (Gradient)。

    • 直觉：“如果要让 Loss 变小，我这个权重参数应该变大一点还是变小一点？”

  5. 优化器更新 (Optimization)：

    优化器（如 Adam）根据梯度，微调模型中成千上万个参数。

    • 更新公式简述：Wnew​=Wold​−learning_rate×gradient
    • 这个过程就是“微调” 。

搜索策略

在推理阶段，模型每次输出的是一个概率分布（词汇表中每个词都有概率）。如何决定选哪个词？

  1. 贪婪解码 (Greedy Decoding)：
    • 策略：眼光短浅。每一步只拿概率最高的那个词。
    • 缺点：如果在第一步选错了（比如选了概率第二高但更适合整句的词），后面就回不去了。
  2. 束搜索 (Beam Search)：
    • 策略：眼光长远。每一步保留前 N 个（例如 2 个）最好的候选序列（Partial Hypotheses）。
    • 过程：
      • Step 1: 可能是 “I” (0.6) 和 “My” (0.3)。保留这两个。
      • Step 2: 分别预测 “I” 后面接什么，“My” 后面接什么。计算累积概率。
      • 筛选: 在所有组合中，再只保留最好的 2 个。
    • 结果：这能找到整体概率更高的句子，因为在这个例子中，Beam Size 为 2，Top Beams 也为 2，意味着我们会返回两个翻译结果。
